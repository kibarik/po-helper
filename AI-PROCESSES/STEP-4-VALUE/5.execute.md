## 🎯 Цель фазы для Step 4

Провести эксперименты по **четырём методам валидации** (smoke / лендинг / fake door / пилот), поднимая **Confidence Point** динамически по мере поступления данных, с явным различением **«оффер сработал» ≠ «ценность подтверждена»**. Цикл повторяется до подтверждения или осознанного pivot; каждый цикл = event-based обновление узлов Нексуса продукта.

Это **execute-фаза Und-Id-Ex** для ценностных гипотез [AI-TRANSFORMATION 4.3]. Пилот — **не MVP**: цель проверить гипотезу, а не запустить продукт.

> Ключевой принцип PAF: **цикл валидации повторяется, пока не найдено подтверждение.** Ни одна гипотеза не подтвердилась — это данные, не провал. Далее **pivot** — осознанная смена компонента бизнес-модели с возвратом на соответствующий этап Discovery.

---

## 👤 Роль

- **Product Engineer** — выбирает метод валидации под CP-профиль гипотезы; фиксирует **pre-experiment statement** ДО запуска (гипотеза, метрика, порог, выборка, критерии «подтверждена/не подтверждена»); принимает решение go / pivot / следующий цикл.
- **Product Ops** — настраивает event-based мониторинг CP: поступление сигнала → пересчёт CP → триггер следующего gate или alert.

> **Зона человека**: стратегическое решение о pivot — какой компонент бизнес-модели менять (потребность / сегмент / рынок / цена / формат) и на какой этап Discovery возвращаться. Решение об инвестициях, не делегируется агенту.

## 🤖 Кортекс-агент

**Агент-валидатор экспериментов** [AI-TRANSFORMATION 4.3]:
1. **Кортекс-генерация лендингов/офферов** — по VP-node и Offer-node генерирует варианты лендинга (Hero/Problem/Solution/Trust/Pricing/FAQ) и 2–3 варианта объявления для A/B формулировок оффера.
2. **Авто-запуск** smoke/таргета/fake door с UTM-метками, цели в аналитике.
3. **Парсинг интервью** — ASR → кодирование → авто-валидация → кодирование цитат в узлы Нексуса.
4. **Динамический CP**: каждый сигнал (CTR, конверсия, цитата, повторное использование) пересчитывает CP по колесу Гилада. Растёт от данных, падает при срабатывании рисков.
5. **Отличает «оффер сработал» от «ценности подтверждена»**: CTR/клик = реакция на оффер; повторное использование/оплата = ценность. Агент не даёт поднять CP до уровня need/value fit только по CTR.
6. **Оценка стоимости риска** перед масштабированием метода (smoke → лендинг → пилот).
7. **4 сценария интерпретации**: A (подтверждена) / B (не подтверждена) / C (неоднозначно) / D (неожиданно подтверждена) → триггер следующего действия.

### Четыре метода валидации (от простого к глубокому)
| Метод | Что проверяет | Метрика | Когда выбирать |
|---|---|---|---|
| **1. Smoke / Реклама / Брошюра** | Реакцию на **оффер** (не на ценность!) | CTR (порог 1,5–3%) | Быстро/дёшево; нет продукта и лендинга; A/B формулировок |
| **2. Лендинг / Сбор контактов** | Готовность к **следующему шагу** (email/заявка) | Конверсия в целевое действие (8–12%) | ЦП требует объяснения; качественная обратная связь |
| **3. Fake Door / Пустышка** | Спрос у **существующих юзеров** | CTR по элементу (5–15% активных) | Есть продукт с активными юзерами; тест новой функции без разработки |
| **4. Пилот / Образец** | **Реальную ценность** (не только интерес) | Конверсия + повторное использование + качественная реакция | Другие методы неоднозначны; ЦП сложно без демонстрации |

### Диагностика уровня проблемы при отрицательном результате
| Уровень | Признак | Куда возвращаться |
|---|---|---|
| Оффер | Низкий CTR, но в интервью «хотел бы попробовать» | остаёмся в Stage 4 (переформулировать оффер) |
| Канал | Хороший CTR, нулевая конверсия — нецелевая аудитория | остаёмся в Stage 4 (таргетинг/канал) |
| ЦП (узкий) | Конверсия есть, но уходят после первого контакта | остаёмся в Stage 4 |
| Потребность | В интервью потребность не «горящая» | → Stage 2 |
| Сегмент | Реагирует другая ЦА | → Stage 2 |
| Бизнес-модель | ЦП нравится, но цена отпугивает | остаёмся в Stage 4 |
| Рынок | Конкуренция слишком высока | → Stage 3 |

## 🧠 Человек (метапознание)

- **Документировать ДО запуска**: pre-experiment statement обязателен, иначе соблазн интерпретировать любой результат как подтверждение.
- **Различать декларируемое и реальное поведение**: слова «куплю» дешевле денег. Нет оплаты/предзаказа → тест не пройден, даже если все в интервью говорили «да».
- **Диагностировать уровень проблемы** при отрицательном результате (матрица выше), и только потом объявлять pivot.
- **Veto** на premature «go»: один слабый сигнал ≠ подтверждение. Минимум 2 разных источника (количественный + качественный) в одном направлении.
- **Pivot ≠ провал**: осознанная смена компонента бизнес-модели с возвратом на соответствующий этап Discovery.

## 📦 Артефакт = узлы Нексуса продукта (+CR)

- **Experiment-node (+CP-дельта)** — метод, гипотеза, метрика, порог, выборка, срок, pre-experiment statement, результаты (количественные + качественные), сценарий интерпретации, CP до/после.
- **Insight-node** — новые знания о потребителе/рынке из каждого эксперимента (даже отрицательного).
- **Pivot-node** (если сработал) — что меняется, на какой этап возвращаемся.
- **Context Ripeness** по подшагу: целевое ≥75% (при подтверждённой гипотезе).

## 🚦 CP-гейт (HARVEST = гейт Need/Value Fit)

Stage-gate с **динамическим CP** между методами валидации:
- smoke → лендинг: CP вырос от CTR + стоимость риска приемлема.
- лендинг → пилот: CP вырос от конверсии в целевое действие, оффер и ценность разведены.
- пилот → 6.harvest (фиксация): **need/value fit в зоне видимости** — повторное использование/оплата/готовность продолжать при достаточной выборке.

CP не поднимается без реальных данных (симуляция ≠ a posteriori). Осознанный отказ — равноправный исход gate. Финальный gate шага — на HARVEST: 🟧 **Need/Value Fit**.

## 🛡️ Анти-workslop

- **Главный риск**: принять сработавший оффер за подтверждённую ценность. Кортекс-агент явно разделяет: CTR/email = оффер; повторное использование/оплата/retention = ценность.
- **Статистическая гигиена**: минимум 30–50 конверсий для вывода о конверсии; <10 конверсий — иллюстрация, не доказательство. (Полная гигиена A/B α/β/MDE — на HARVEST, метод 4.6.)
- Pre-experiment statement обязателен — защищает от подгонки интерпретации.
- Кортекс-лендинги проходят workslop-фильтр: нет «воды», каждая секция несёт конкретный профит.
- Запрещённые синонимы: НЕ «приоритизация», НЕ «go/no-go чекбокс» → **CP-gate с оценкой стоимости риска** (Принцип 4: риски, не потери).
