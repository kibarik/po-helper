# 5.execute — EXECUTE / Интервью и кодирование (Step 2: Customer)

> Фаза Sprint: **EXECUTE** (тяжёлая). Источник: [[operating-model]] (Discovery: валидация гипотез), [S1] VI.5. Термины — по [[sa_documentation/naming_conventions|naming_conventions]].

---

## 🎯 Цель фазы для Step 2

Discovery-исполнение: провести глубинные интервью (CustDev/JTBD) по Банчу гипотез из SCOUT, защищённые **The Mom Test**, и насытить Нексус потребителя через цикл **ASR → кодирование → авто-валидация → event-driven обновление CP**. Каждое интервью — **Harvesting-микроинкремент** контекста в Нексус [2.3, 2.4 instruction].

Ключевые принципы:
- Интервью ведёт **человек**; на ранней зрелости Кортекса ИИ — **только кодировщик транскрипта, не собеседник** [reformation 2.9, 2.4].
- Насыщение данных (data saturation) — после ~12 интервью на сегмент ответы повторяются [2.3].
- Валидация в реальном времени: респонденту **не показывают** список гипотез; исследователь мысленно соотносит рассказанное со своими гипотезами [2.4].

## 👤 Роль

- **Product Engineer** — определяет профиль «правильного респондента» (обязательные/желательные/стоп-критерии); ведёт интервью (на ранней зрелости) или модератирует; интерпретирует кодировки; защищает от «вежливых» ответов.
- **Product Ops** — настраивает Рекрутёр-агента (каналы, реестр статусов), ASR-пайплайн, JTBD-RAG, триггеры авто-валидации, флаги увядания узлов.
- **Growth Engineer** — добавляет в скрининг вопросы для mNSM-валидации (готовность платить); следит за mNSM-сигналами в кодировках.
- **Зона человека:** решение «pivot/продолжать» при провале гипотез — стратегический вызов, **не передаётся ИИ** [S2] III.4.

## 🤖 Кортекс-агенты

**1. Рекрутёр-агент (Recruiter Agent):** [[AI-TRANSFORMATION/RB-STEP-2-CUSTOMER-VALIDATION/2.3 instruction]]
- **Вход:** профиль респондента + план выборки (≥12 на сегмент — насыщение данных).
- **Действие:** рассылает по каналам (личные контакты → знакомые знакомых → тематические чаты → соцсети); проводит скрининг (2–5 мин, закрытые вопросы); ведёт реестр статусов (✅ подтверждён / ⏳ в контакте / 🔵 квалификация / ❌ отказ).
- **Выход:** назначенные интервью в календаре + квалифицированный список.

**2. Скрипт-билдер (Script Builder Agent):** [[AI-TRANSFORMATION/RB-STEP-2-CUSTOMER-VALIDATION/2.3 instruction]]
- **Вход:** Банч гипотез из SCOUT + узлы Нексуса.
- **Действие:** строит скрипт с **маппингом вопрос→гипотеза→узел**; направления PAF (альтернативные пути / триггеры / проблемы+причины / цели / мотивация); применяет **The Mom Test** (3 принципа: прошлый опыт, не будущие намерения; меньше говорить, больше слушать; избегать якорения).
- **Выход:** скрипт интервью + матрица покрытие гипотез.

**3. Кодировщик транскриптов (Transcript Coder Agent):** [[AI-TRANSFORMATION/RB-STEP-2-CUSTOMER-VALIDATION/2.4 instruction]]
- **Вход:** аудио/видео интервью → ASR (транскрипция).
- **Действие:** кодирует реплики в единицы знания о поведении (Действие / Ситуация-Триггер / Цель / Причина-Мотивация / Проблема / Источник: интервью #N / Частота: все/большинство/некоторые); соотносит с гипотезами (✅ подтверждена / ❌ опровергнута / 🆕 новая); извлекает цитаты-доказательства; **авто-валидирует** (≥70% подтверждают / ≤30% опровергают); **event-driven** пересчитывает CP узлов; триггерит Harvesting-микроинкремент в Нексус.
- **Выход:** обновлённые узлы Нексуса с новым CP + карточка интервью + новые гипотезы-кандидаты.

### The Mom Test — 3 принципа (Роб Фитцпатрик, [2.3])
1. Спрашивай про прошлый реальный опыт, не про будущие намерения. «Расскажи, как в последний раз…» — не «стал бы пользоваться…?».
2. Меньше говори, больше слушай (80% времени — молчи и уточняй; выдерживать паузу).
3. Избегай якорения: никаких наводящих («Вам же неудобно…?») и гипотетических («А если бы было…?»).

### Цикл обработки (event-based, [S1] VI.2)
```
аудио → ASR-транскрипт → кодирование в единицы знания
  → соотнесение с гипотезами-узлами (маппинг из Скрипт-билдера)
  → авто-валидация (≥70% / ≤30%)
  → event: «CP узла изменился» → пересчёт CR Нексуса
  → если сработал риск → CP падает (Кортекс/инженер)
  → Harvesting-микроинкремент: узел закоммичен в Нексус
```

## 🧠 Человек (метапознание)

- **Чекпоинт:** кодировка не «натянута» на гипотезу (галлюцинация соответствия); трассировка до цитаты.
- **Veto-право:** переопределить авто-статус гипотезы, если респондент сказал «вежливое да» (Mom Test); veto на ИИ-интервьюера.
- **«Провал всех гипотез — ценный сигнал, а не катастрофа»** [2.4]: решение о pivot — за человеком.
- **Разделять поведение и мнение/пожелание:** «хотел бы оценку» ≠ «оценивает» → отдельный узел, низкий CP.
- **JTBD-RAG:** кодировщик опирается на индекс транскриптов в формате «Когда я <ситуация>, я хочу <цель>, чтобы <результат>» [2.4].

---

## 📦 Артефакт = узлы Нексуса потребителя (+CR)

- Единицы знания о поведении → узлы (действия, триггеры, проблемы+причины, мотивация).
- Цитаты-доказательства привязаны к узлам (`источник-интервью[N]`, `частота`).
- CP узлов обновлён event-driven; новые гипотезы-кандидаты добавлены в Банч; статус увядания отслеживается.
- Context Ripeness растёт по мере насыщения данных.

## 🚦 CP-гейт

- **Критерий выхода из EXECUTE:** насыщение данных (после ~12 интервью ответы повторяются); CP фокус-гипотез ≥ gate.
- **Стоимость риска:** «Сколько ресурсов потратим, если валидируем на ненасыщенной выборке или с искажённой кодировкой?»

## 🛡️ Анти-workslop (критический для EXECUTE)

- **Правило «персона ≥4 интервью»** вступает в силу при сборке сегментов (BUNCH): узел Сегмент принимается только при ≥4 подтверждающих интервью с цитатами [2.6].
- **Защита от AI-сгенерированных персон:** на ранней зрелости ИИ — только кодировщик, **не интервьюер** (искажение, потеря невербалики, «дофаминовая петля» — респонденту приятнее говорить с ИИ → рост когнитивной нагрузки, не снижение [S2] III.7).
- ASR-транскрипт хранится как первоисточник; кодировка трассируется до цитаты.
- **Запрет в сообщении рекрутинга** упоминать продукт/идею/«тестирую приложение» — меняет поведение респондента.
- **Лучшие респонденты — бесплатные** (цель поделиться знанием); платные часто «говорят то, что хотят услышать» [2.3].
- **Суррогатная валидация** (форумы/отзывы/Reddit через Кортекс) — только дополнение/triangulation, **не замена** живого интервью; CP от суррогата ниже.
- Плохие практики: наводящий вопрос · гипотетический вопрос · платные респонденты · меньше 12 на сегмент · радикальная смена скрипта (обнуляет сравнения) · обработка «потом» вместо event-driven.
