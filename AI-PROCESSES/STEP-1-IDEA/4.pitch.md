## 🎯 Цель фазы для этой вехи

**Pitching of Trust** — **stage-gate** [S1] VI.3, VI.4 цикла: синтез **Vision (BIG Idea Statement)** из доказанной части Банча и защита его перед командой, где команда «заваливает» аргументы → назначается **final Confidence Point** и принимается **Go-Decision** с resource commitment.

В Step 1 PITCH — **тяжёлая фаза**. Три действия:

1. **Vision-Synth** — синтез BIG Idea Statement (Vision [S1] базовая логика) из Confirmed фич Банча + `Bet` + `mNSM-numeric` + `JTBD-Opportunity`. Vision — узел Нексуса продукта, не слоган.
2. **Assumption-Audit** — пометка допущений без evidence (контр-челендж Кортекса).
3. **Pitching of Trust** — 10-min pitch (Context/BIG Idea/Bunch/Validation) + Q&A «try to kill it» (Red-Team) + голосование команды за CP каждой фичи (CP < порога dropped, CP ≥ порога approved) + resource commitment [S1] VI.7.

Gate тем легче проходит, чем выше CP (рост контекста = рост скорости процессов [S1] VI.3).

## 👤 Роль

- **Product Engineer** — формулирует финальный Vision (зона человека [S2] III.4); защищает Банч; честно признаёт unknowns; принимает final CP. Проходит Assumption-Audit перед защитой.
- **Команда (Product Pod)** — задаёт скептические вопросы; голосует за CP; «заваливает» слабые аргументы [S1] VI.4. Размер пода может быть из 1 человека (весь цикл [S1] III.2), тогда Red-Team — единственный «оппонент».
- **Product Ops** — ведёт Red-Team Agent; фиксирует final CP в Узлах; поддерживает версионирование Vision.
- **Portfolio Manager** — подтверждает resource commitment (если Банч проходит gate); сверяет Vision с Картой Целей; разрешает конфликты каннибализации.
- **Зона человека:** голосование за CP и решение о resource commitment — за командой людей (синхронизация взаимодействия сотрудников — зона человека [S2] III.4). ИИ аугментирует (Red-Team), не решает.

## 🤖 Кортекс-агент фазы

Два агента в связке:

**Vision-Synth Agent** (режим Scouting Agent [reformation] §1.5, синтез + челендж):
- Синтез Vision из Узлов: `Bunch-v1` + `Bet` + `mNSM-numeric` + `JTBD-Opportunity` + `Monetization-Hypothesis`.
- Генерация по шаблону исходника 5.1 (For/Who/The/That/Unlike/We + validate by + mNSM + revenue).
- **Челендж-режим:** каждое утверждение Statement без evidence-узла помечается; отсев vanity-mNSM, «Unlike» без конкурента, количественные обещания без NPV-ребра.
- **Output:** `Vision-v1`-узел + `Assumption-Audit`-узел.

**Red-Team Agent** ([reformation] §1.5):
- Скептические вопросы «kill it» для каждой фичи: «Что если [риск]?», «Почему не [альтернатива]?», «Какое evidence?», «Какой worst case?».
- Симуляция «kill it» из Узлов Нексуса (где evidence слабкий, где `Bet` рискованный).
- Асинхронная поддержка: подсказывает контр-контр-аргументы из Узлов (финальное слово — за человеком).
- **Output:** `Red-Team-Questions`, обновлённый **final CP** фич, `Pitching-Decision` (approved/dropped), `Resource-Commitment`.

- **Input:** `Bunch-v1` (3.bunch) + Узлы 3 Нексусов + `Presentation-v1` (5.2 из исходника).
- **Output:** `Vision-v1`, `Assumption-Audit`, `Red-Team-Questions`, final CP, `Pitching-Decision`, `Resource-Commitment`.

## 🧠 Человек (метапознание)

- **Чекпоинт (Vision):** Statement — топ-зона workslop («красивый слоган без измеримой основы» [S2] III.7). Инженер проходит Assumption-Audit: какие допущения без evidence?
- **Чекпоинт (Pitching):** команда не должна **поддакивать Кортексу** (дофаминовая петля ИИ [S2] III.7). Red-Team — инструмент, финальное суждение — за людьми.
- **Veto-право:** «achieve $500k ARR» без `mNSM-numeric` и NPV-ребра → отсев обещания или понижение CP Vision; любая фича с CP < порога после final → dropped **или** осознанный отказ от всего Банча [S1] VI.3.
- **Вопрос:** «Финальное CP — это голос людей или одобрение Кортекса?» · «Red-Team указал на evidence-брешь в фиче X — есть ли реальный ответ (не от Кортекса), или это drop?»

## 📦 Артефакт = узлы Нексуса (+ Context Ripeness)

- **Создаются:** `Vision-v1`-узел (BIG Idea Statement + Supporting Details) с рёбрами к `Bunch-v1`, `Bet`, `mNSM-numeric`, `JTBD-Opportunity`, `Competitor`; `Assumption-Audit`-узел (допущения без evidence, ранжированные по стоимости ошибки); `Red-Team-Questions` (привязанные к фичам); `Presentation-v1` (Context/BIG Idea/Bunch/Validation, 10-min); `Pitching-Decision` (approved/dropped per feature); `Resource-Commitment` (бюджет/команда/timeline).
- **Метаданные:** `источники` (Узлы), `CP-Vision`, `assumptions` (без evidence), `final-CP` фич, `decision` (approved/dropped), `red-team-бреши`, `commitment`, `timestamp`, `статус увядания`.
- **Bunch-v2:** формируется из Confirmed фич (final CP ≥ порога) — замена Bunch-v1 после голосования.
- **Прирост CR:** +5% (final CP + бреши + Bunch-v2 — инкремент контекста).

## 🚦 Confidence Point (гейт к след. фазе)

- **Это и есть stage-gate** ([reformation] §1.6). Критерий: Confirmed фичи с final CP ≥ порога → approved; CP < порога → dropped / осознанный отказ. Resource commitment — если Банч проходит.
- **Стоимость риска:** «Какой бюджет мы committing? Какой NPV-потенциал Банча оправдывает этот бюджет?» — в `Resource-Commitment`.

## 🛡️ Анти-workslop

- **Workslop-риск:** Statement как слоган без evidence; количественные обещания без NPV-ребра; «Unlike» без конкурента; vanity-mNSM; команда поддакивает Кортексу (Red-Team как театр — вопросы есть, CP не падает).
- **Признак:** утверждение без ребра к Узлу; Assumption-Audit отсутствует; Vision опирается на Needs-More фичи; ни одна фича не dropped после Pitching (подозрительно); CP вырос без новых evidence; commitment без NPV-обоснования.
- **Правило верификации:** Assumption-Audit обязателен; каждое утверждение — ребро к Узлу с источником; только Confirmed фичи; real-вопросы от людей (не только Red-Team); CP падает при брешах; commitment — по NPV; хотя бы одна фича осознанно dropped ИЛИ обоснованное «все сильные».

**Шаблон BIG Idea Statement** (преемственность из исходника 5.1): For [target] / Who [need] / The [product] / That [benefit] / Unlike [competitor] / We [differentiator] + validate by [signal] + mNSM [numeric] + revenue [model]. Каждое поле — ребро к Узлу с источником.

## 🔭 Связь с Тремя Линзами (3PL)

PITCH работает **над 3PL**: защита Банча и Vision перед всеми тремя Линзами одновременно:
- **Product Lens** — Vision → Нексус продукта, Confirmed фичи;
- **Business Lens** — mNSM/NPV в Statement, resource commitment по NPV;
- **Strategy Lens** — `Bet`/Unlike в Statement, сверка с Картой Целей портфеля.

Red-Team генерирует вопросы из всех трёх Линз: Product (JTBD выполнена?), Business (NPV > 0?), Strategy (Bet оправдана?). Связь: approved Банч → следующий цикл Product Sprint (Step 2); dropped → переоценка через BUNCH (3.bunch).
