## 🎯 Цель фазы для этой вехи

**Скаутинг возможностей и угроз** через все 3 Линзы [S1] VI.1, Принцип 5 — поиск, анализ, исследования, чтобы поднять контекст Нексусов до идей с максимальным CP. Это **Understand-фаза** Und-Id-Ex [S1] VI.5: анализ Нексуса → opportunities/threats с initial CP.

В Step 1 SCOUT — **тяжёлая фаза**: она впервые насыщает Нексусы рынка/продукта/роста конкретными методами:

- **Strategy Lens** — **First Principles** (core job JTBD, root causes, ideal solution) + **Blue Ocean (ERRC)**: Eliminate/Reduce/Raise/Create + Competitive Weakness Scouting (отзывы G2/Reddit/App Store) → `Strategic-Opportunity` + `Bet-candidate`.
- **Business Lens** — **Business Model Canvas (BMC)** + **Lean Canvas** → `Monetization-Hypothesis` + `mNSM-numeric` + Рычаги (Lever).
- **Product Lens** — **Jobs To Be Done (JTBD)** (functional/emotional/social job) + **Design Thinking** → `JTBD-Opportunity` + фич-кандидаты.

Методы исполняются Кортексом как **аугментация** (альтернативные точки зрения) и **челендж** (опровержение) — 2-я и 3-я оптимизации роли [S2] III.4.

## 👤 Роль

- **Product Engineer** — ставит задачу Scouting Agent (фокус, источники, ограничения); валидирует стратегическое позиционирование; формирует Ставки (Bets). Валидирует, что каждый Opportunity опирается на конкретный Узел с источником.
- **Portfolio Manager** — сверяет стратегические возможности с Картой Целей; разрешает конфликты каннибализации [S1] III.3. Формирует «цифровой профиль бизнес-модели» [S2] III.6.
- **Growth Engineer** — валидирует `mNSM-numeric` и потенциал Рычагов (Business Lens); проверяет монетизируемость NSM (не vanity).
- **Зона человека:** стратегическое позиционирование и Ставки — всегда за человеком [S2] III.4. Кортекс аугментирует (предлагает альтернативы) и челенджит (оспаривает), не решает.

## 🤖 Кортекс-агент фазы

**Scouting Agent** ([reformation] §1.5) — работает в трёх режимах по Линзам:

- **Strategy-режим:** First Principles по Узлам Нексуса рынка; ERRC-канвас как челендж к интуиции инженера; Competitive Weakness Scouting (парс отзывов).
- **Business-режим:** BMC + Lean Canvas из Узлов Нексуса роста; генерация `Monetization-Hypothesis` с ребром к `mNSM-numeric`.
- **Product-режим:** JTBD-интервью (структурированный разбор) → `JTBD-Opportunity`.

**Режим работы** [S2] III.4: **аугментация** — 2–3 альтернативы First Principles, не одна; **челендж** — контр-аргумент «почему это не сработает» к каждой возможности.

- **Input:** Узлы Нексусов рынка/продукта/роста из PULSE (`Competitor`, `Trend`, `Gap`, `Force`, юнит-экономика, JTBD-заметки).
- **Action:** скаутинг через 3 Линзы; назначение initial CP; контр-аргументация.
- **Output:** `Strategic-Opportunity` (5–10), `Monetization-Hypothesis`, `JTBD-Opportunity`, `Bet-candidate` — с initial CP, рёбрами к Узлам и контр-аргументами.

## 🧠 Человек (метапознание)

- **Чекпоинт:** стратегические возможности — зона высокого workslop (красивые ERRC без рыночной проверки [S2] III.7). Инженер проверяет: опирается ли каждый Opportunity на конкретный `Competitor`/`Trend`-узел с источником?
- **Veto-право:** Кортекс может предложить «Blue Ocean» там, где рынок уже красный → сверка с Нексусом рынка.
- **Вопросы:** «Эта возможность — Ставка на новый рынок или улучшение в существующем? Какой `Bet` она обслуживает?» · «Контр-аргумент Кортекса — обоснован Узлом или общая фраза?»

## 📦 Артефакт = узлы Нексуса (+ Context Ripeness)

- **Создаются (Strategy Lens):** `Strategic-Opportunity` (5–10, тип: market gap / competitive weakness / trend alignment) с initial CP, рёбрами к `Competitor`/`Trend`/`Gap-рынка`; `Bet-candidate`.
- **Создаются (Business Lens):** `Monetization-Hypothesis` с ребром к `mNSM-numeric`; `Lever` (Рычаг) — измеримое свойство модели роста, рост NPV при инвестиции [S1].
- **Создаются (Product Lens):** `JTBD-Opportunity` (functional/emotional/social job) с рёбрами к фич-кандидатам.
- **Метаданные каждого Узла:** `источник`, `initial CP`, `контр-аргумент` (от челенджа), `mNSM-ребро`, `timestamp`, `статус увядания`.
- **Прирост CR:** +20–35% суммарно по 3 Нексусам (скаутинг — ядро созревания на null base: рынок +10–15%, продукт +5–10%, рост +5–10%).

## 🚦 Confidence Point (гейт к след. фазе)

- **Критерий перехода к BUNCH:** ≥ 5 `Strategic-Opportunity` с initial CP, каждый с контр-аргументом и привязкой к `Bet`-кандидату; ≥ 1 `Monetization-Hypothesis` с `mNSM-numeric`; ≥ 3 `JTBD-Opportunity`.
- **Стоимость риска:** «Если Ставка неверна — сколько инвестируем в неверный стратегический сценарий?» — в `Bet-candidate`.

## 🛡️ Анти-workslop

- **Workslop-риск:** **Blue Ocean как самообман** («создадим новый рынок» без evidence); ERRC без связи с конкурентами; JTBD-инсайт без интервью; BMC без связи с Рычагами.
- **Признак:** Opportunity без `источника`; контр-аргумент отсутствует; CP назначен интуитивно; TAM без ссылки; `Monetization-Hypothesis` без ребра к `mNSM-numeric`.
- **Правило верификации:** каждый `Strategic-Opportunity` — рёбра к Узлам; Cross-Lens evidence; CP = медиана источников, не интуиция; рыночные цифры — трассировка до URL/даты [reformation] §1.8; каждый JTBD-Opportunity — из интервью с источником.

## 🔭 Связь с Тремя Линзами (3PL)

SCOUT — единственная фаза Step 1, где **все три Линзы активны последовательно** [S1] VI.1:

- **Strategy Lens** (Strategic Sprint → Нексус рынка → Ставки/Bets): First Principles + Blue Ocean ERRC + Competitive Weakness. `Strategic-Opportunity` → потенциал Рычага (Business Lens) и фич-кандидат через JTBD (Product Lens).
- **Business Lens** (Business Sprint → Нексус роста → Рычаги/Lever → NPV): BMC + Lean Canvas. `Monetization-Hypothesis` + `mNSM-numeric` → NPV фич в BUNCH.
- **Product Lens** (Product Sprint → Нексус продукта → Банч): JTBD + Design Thinking. `JTBD-Opportunity` → фич-кандидаты в BUNCH.

Связь между Линзами — Cross-Lens evidence: каждый `Strategic-Opportunity` получает подтверждение из двух других Линз, иначе CP понижается.
