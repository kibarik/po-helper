# 4.2 Первичный скоринг гипотез ценности — Полная инструкция

## 🎯 Цель этой инструкции

Отобрать из сформированных гипотез ценностного предложения те, которые стоит проверять в первую очередь. Ресурсы на валидацию всегда ограничены — скоринг помогает вложить их туда, где максимальная отдача.

**Время на выполнение:** 2–3 часа (для новичка) · 1 час (для опытного)  
**Уровень:** Новичок (всё объясняется подробно)  
**На выходе (артефакт):** Ранжированный список гипотез ценностного предложения с обоснованием приоритетов. Топ-1–3 гипотезы, отобранные для валидации на шаге 4.3.

---

## 📋 Что такое первичный скоринг гипотез ценности?

### Простыми словами

**Первичный скоринг гипотез ценности** — это структурированная оценка и ранжирование гипотез ценностного предложения с целью выбрать, какие из них проверять первыми.

На предыдущем шаге (4.1) ты сформировал несколько гипотез. Теперь нужно ответить на вопрос: *какую из них проверять первой?*

Это не окончательный выбор — это расстановка приоритетов. Та гипотеза, которая получит низкий балл сейчас, может оказаться важной позже, если высокоприоритетные не подтвердятся.

### Аналогия

Представь, что у тебя есть 5 семян, и нужно выбрать, какие 2 посадить в этом сезоне, потому что горшков больше нет. Ты смотришь на:
- Какое семя с большей вероятностью прорастёт (доказательная база)
- Какое принесёт самый вкусный плод (ценность для потребителя)
- Какое ты сможешь продать дороже (ценность для компании)

Скоринг гипотез работает точно так же: смотришь на несколько критериев, выставляешь баллы, выбираешь лучших кандидатов на «посадку» — валидацию.

---

## 🎯 Зачем нужен скоринг?

### 3 главные причины

**1. Одна потребность — много ценностных предложений, ресурсы ограничены**

В PAF прямо указано: *одна и та же потребность может быть закрыта при помощи разных ценностных предложений, но ресурсы на их проверку всегда ограничены.* Скоринг — механизм рационального распределения этих ресурсов.

Проверять все гипотезы параллельно — дорого и путает результаты. Проверять одну случайную — неэффективно. Скоринг даёт систему.

**2. Избежать ловушки «самой очевидной идеи»**

Почти всегда в команде есть «любимая идея» — та, которую все хотят проверить первой. Скоринг делает выбор объективным: принудительно оцениваешь все гипотезы по одинаковым критериям и видишь реальную картину.

**3. Зафиксировать логику выбора**

Скоринг — это документация решения. Через три месяца, когда спросят «почему вы проверяли именно это», у тебя будет ответ с обоснованием, а не «казалось хорошей идеей».

### Что будет плохо без этого шага?

- Команда потратит время на валидацию гипотезы, которая изначально имела низкую потенциальную ценность
- Будет проверена «любимая идея» основателя вместо наиболее обоснованной гипотезы
- При отрицательных результатах непонятно, что проверять дальше

## 📊 Структура артефакта

Результат шага 4.2 — **Матрица скоринга гипотез ценностного предложения** и **краткое обоснование топ-3 гипотез** для валидации.

### Из чего состоит матрица скоринга

**Критерий 1: Ценность для потребителя (VP Customer Value)**
Насколько сильно это ЦП решает реальную потребность потребителя? Чем больше ресурсов (времени, денег, нервов) сэкономит потребитель — тем выше балл.

**Критерий 2: Доказательная база (Evidence)**
На каком количестве и качестве знаний из исследований строится эта гипотеза? Чем больше подтверждений из реальных интервью и данных — тем выше балл.

**Критерий 3: Ценность для компании (Company Value)**
Насколько это ЦП соответствует целям и бизнес-модели компании? Помогает ли оно достичь метрик, важных для компании?

**Критерий 4: Стоимость и скорость валидации (Validation Cost)**
Насколько быстро и дёшево можно проверить эту гипотезу? Чем проще провести эксперимент — тем выше балл.

**Критерий 5: Дифференциация (Differentiation)**
Насколько это ЦП отличается от того, что уже есть на рынке? Чем сложнее конкурентам скопировать — тем выше балл.

## 📝 Phase 1: Настройка матрицы скоринга

### Step 1.1: Создай таблицу скоринга

Открой Google Sheets или Notion и создай следующую структуру:

```
| Гипотеза | VP Customer Value (1-5) | Evidence (1-5) | Company Value (1-5) | Validation Cost (1-5) | Differentiation (1-5) | ИТОГО | Приоритет |
|----------|-------------------------|----------------|---------------------|----------------------|----------------------|-------|-----------|
| ЦП #1    |                         |                |                     |                      |                      |       |           |
| ЦП #2    |                         |                |                     |                      |                      |       |           |
| ...      |                         |                |                     |                      |                      |       |           |
```

### Step 1.2: Определи веса критериев

Не все критерии равнозначны. PAF ставит во главу угла ценность для потребителя и доказательную базу. Рекомендуемые веса:

| Критерий | Вес | Почему |
|----------|-----|--------|
| VP Customer Value | ×2 | Ценность для потребителя — приоритет #1 в PAF |
| Evidence | ×2 | Знание > интуиция; слабая доказательная база = высокий риск |
| Company Value | ×1.5 | Важно, но вторично по отношению к ценности потребителя |
| Validation Cost | ×1 | Ресурсная эффективность — тактический критерий |
| Differentiation | ×1.5 | Без дифференциации трудно захватить долю рынка |

**Формула итогового балла:**
```
ИТОГО = (Customer Value × 2) + (Evidence × 2) + (Company Value × 1.5) + (Validation Cost × 1) + (Differentiation × 1.5)
Максимум: 5×2 + 5×2 + 5×1.5 + 5×1 + 5×1.5 = 10+10+7.5+5+7.5 = 40
```

> Ты можешь адаптировать веса под контекст своего продукта — например, увеличить вес Validation Cost, если ресурсы очень ограничены.

## 📝 Phase 2: Оценка каждой гипотезы

### Step 2.1: Критерий 1 — VP Customer Value (Ценность для потребителя)

**Что оцениваем:** Насколько сильно это ценностное предложение решает реальную потребность потребителя? Насколько существенно оно сократит его усилия (время, деньги, нервы)?

**Шкала оценки:**

| Балл | Описание |
|------|----------|
| 1 | Закрывает слабую / периферийную потребность. Потребитель «мог бы» этим воспользоваться, но ни разу не упомянул это как болевую точку. |
| 2 | Решает реальную потребность, но умеренно — есть удобные альтернативы. |
| 3 | Закрывает важную потребность, потребитель упоминал это в интервью, но не как критичное. |
| 4 | Решает значимую потребность, которая часто упоминалась в интервью как болевая точка. |
| 5 | Закрывает ключевую («горящую») потребность, потребитель активно ищет решение и не находит достойного. Существенное сокращение ресурсов потребителя. |

**Вопросы для оценки:**
- Насколько часто потребители упоминали эту потребность в интервью?
- Сколько ресурсов (время/деньги/нервы) потребитель тратит сейчас из-за отсутствия этого решения?
- Как близко это ЦП к «идеальному конечному результату» потребителя?
- Если потребитель получит это — сколько ресурсов он сэкономит?

**Пример оценки (EduMatch):**

```
ЦП #1 «AI-подбор репетитора за 24 часа»
→ Customer Value: 5
Обоснование: В 14 из 14 интервью родители указывали время поиска
как главную боль. Сокращение с 5 часов до 24 часов автоподбора —
критическая ценность. Близко к «магической кнопке».

ЦП #2 «Персонализированная замена при отсутствии прогресса»
→ Customer Value: 4
Обоснование: Важная потребность (упоминали 9 из 14),
но менее срочная — проблема возникает через 4+ недели.
```

---

### Step 2.2: Критерий 2 — Evidence (Доказательная база)

**Что оцениваем:** Насколько хорошо эта гипотеза подкреплена реальными знаниями из исследований? Чем больше и качественнее доказательства — тем ниже риск при валидации.

**Шкала оценки:**

| Балл | Описание |
|------|----------|
| 1 | Чистая интуиция — нет данных из исследований, только ощущение команды. |
| 2 | Единичный источник: 1–2 интервью или косвенные данные. |
| 3 | Несколько источников: 3–5 интервью + один косвенный индикатор (аналог, данные рынка). |
| 4 | Хорошая база: 6–10 интервью + несколько источников данных + аналог. |
| 5 | Сильная база: 10+ интервью с согласованными ответами + количественные данные + аналог на смежном рынке. |

**Что считается доказательством:**
- Качественные интервью с потребителями
- Количественные данные (опросы, аналитика продукта)
- Аналоги: похожие продукты решили похожую проблему похожим способом
- Данные конкурентов: у них есть эта функция и она пользуется спросом

**Что НЕ считается доказательством:**
- «Я сам бы этим пользовался»
- «Логично, что это нужно»
- Мнение стейкхолдера без подтверждения из исследований

**Пример оценки:**

```
ЦП #1 «AI-подбор репетитора за 24 часа»
→ Evidence: 4
Обоснование: 14 интервью + анализ конкурентов (никто не даёт
гарантию скорости подбора) + аналог Profi.ru (быстрый подбор
как USP — занял долю рынка).

ЦП #3 «Геймификация уроков через приложение»
→ Evidence: 1
Обоснование: Идея возникла на брейншторме, без интервью.
Ни один родитель не упоминал геймификацию как важную характеристику.
```

### Step 2.3: Критерий 3 — Company Value (Ценность для компании)

**Что оцениваем:** Насколько это ЦП соответствует бизнес-целям и модели компании? Приближает ли оно к ключевым метрикам?

**Шкала оценки:**

| Балл | Описание |
|------|----------|
| 1 | Противоречит текущей бизнес-модели или не влияет ни на одну бизнес-метрику. |
| 2 | Слабое влияние на бизнес-метрики; сложно монетизировать. |
| 3 | Умеренное соответствие: влияет на одну-две бизнес-метрики. |
| 4 | Хорошее соответствие: напрямую влияет на выручку или стратегическую цель. |
| 5 | Отличное соответствие: это ЦП является ключевым для достижения главной бизнес-цели компании. |

**Вопросы для оценки:**
- Как это ЦП влияет на конверсию, удержание, LTV?
- Соответствует ли это гипотезам бизнес-модели?
- Является ли этот сегмент потребителей приоритетным для компании?

**Пример оценки:**

```
ЦП #1 «AI-подбор репетитора за 24 часа»
→ Company Value: 5
Обоснование: Быстрый подбор = ключевой крюк для первой оплаты.
Напрямую влияет на конверсию в первую транзакцию — главную
метрику на этапе запуска.

ЦП #4 «Корпоративные пакеты для школ»
→ Company Value: 2
Обоснование: B2B сегмент не является приоритетом на этапе
запуска, длинный цикл продаж, требует отдельных ресурсов.
```

---

### Step 2.4: Критерий 4 — Validation Cost (Стоимость и скорость валидации)

**Что оцениваем:** Насколько быстро и с какими ресурсами можно проверить эту гипотезу? Чем проще и быстрее — тем выше балл.

**Шкала оценки:**

| Балл | Описание |
|------|----------|
| 1 | Требует значительных ресурсов: разработка прототипа 2+ недели, большой бюджет на трафик, сложный доступ к ЦА. |
| 2 | Требует умеренных ресурсов: 1–2 недели на подготовку, средний бюджет. |
| 3 | Умеренная сложность: лендинг + 3–5 дней настройки, небольшой бюджет. |
| 4 | Относительно просто: лендинг или реклама за 1–2 дня, минимальный бюджет. |
| 5 | Очень просто: можно проверить за 1 день (пустышка в интерфейсе, быстрый лендинг, 5 интервью). |

**Типы методов валидации и их сложность (от простого к сложному):**

| Метод | Время | Стоимость | Сложность |
|-------|-------|-----------|-----------|
| 5–10 глубинных интервью | 2–3 дня | Бесплатно | Низкая |
| Реклама / брошюра (smoke test) | 1–3 дня | 3–15 тыс. руб. | Низкая |
| Лендинг + трафик | 3–7 дней | 10–50 тыс. руб. | Средняя |
| Пустышка в существующем продукте | 1–2 дня | Низкая (если продукт есть) | Средняя |
| Пилотный проект / образец | 1–4 недели | Средняя–высокая | Высокая |

**Пример оценки:**

```
ЦП #1 «AI-подбор репетитора за 24 часа»
→ Validation Cost: 4
Обоснование: Можно проверить лендингом + таргетом за 3–5 дней
и бюджетом ~15 тыс. руб. У нас уже есть база потенциальных
пользователей из предыдущих интервью.

ЦП #5 «Собственная AI-модель прогноза успеваемости»
→ Validation Cost: 1
Обоснование: Требует разработки ML-модели, данных за несколько
месяцев, не может быть проверена без сложного прототипа.
```

---

### Step 2.5: Критерий 5 — Differentiation (Дифференциация)

**Что оцениваем:** Насколько это ЦП отличается от существующих предложений на рынке? Насколько сложно конкурентам его скопировать?

**Шкала оценки:**

| Балл | Описание |
|------|----------|
| 1 | Полная коммодити: всё то же самое делают 3+ конкурента. Нет отличия. |
| 2 | Слабая дифференциация: есть незначительные отличия, но конкурент может скопировать за месяц. |
| 3 | Умеренная: заметное отличие, но в принципе реализуемо конкурентом за квартал. |
| 4 | Сильная: уникальное предложение, для копирования конкуренту нужны значительные ресурсы или время. |
| 5 | Очень сильная: фундаментальное преимущество (данные, технология, партнёрства), крайне сложно скопировать. |

**Вопросы для оценки:**
- Делает ли кто-то из конкурентов то же самое?
- Если да — насколько легко нам выиграть за счёт исполнения?
- Есть ли у нас уникальный актив (данные, технология, партнёрства), который конкурентам сложно повторить?

**Пример оценки:**

```
ЦП #1 «AI-подбор репетитора за 24 часа»
→ Differentiation: 4
Обоснование: Ни один из основных конкурентов не даёт гарантию
скорости подбора. AI-матчинг под стиль обучения ребёнка —
уникален для рынка. Скопировать за 1–2 квартала возможно,
но у нас будет фора и данные пользователей.

ЦП #2 «Персонализированная замена при отсутствии прогресса»
→ Differentiation: 5
Обоснование: Абсолютно новая концепция «гарантии результата»
на рынке репетиторства. Требует данных о прогрессе и алгоритма
замены — фундаментальное преимущество.
```

## 📝 Phase 3: Подсчёт и интерпретация результатов

### Step 3.1: Заполни матрицу и посчитай баллы

**Пример заполненной матрицы (EduMatch):**

| Гипотеза | Customer Value ×2 | Evidence ×2 | Company Value ×1.5 | Validation Cost ×1 | Differentiation ×1.5 | ИТОГО | Приоритет |
|----------|-------------------|-------------|--------------------|--------------------|----------------------|-------|-----------|
| ЦП #1: AI-подбор за 24ч | 5×2=10 | 4×2=8 | 5×1.5=7.5 | 4×1=4 | 4×1.5=6 | **35.5** | 🥇 1 |
| ЦП #2: Замена при нет прогресса | 4×2=8 | 4×2=8 | 4×1.5=6 | 3×1=3 | 5×1.5=7.5 | **32.5** | 🥈 2 |
| ЦП #3: Геймификация уроков | 2×2=4 | 1×2=2 | 3×1.5=4.5 | 4×1=4 | 2×1.5=3 | **17.5** | 4 |
| ЦП #4: Корпоративные пакеты | 3×2=6 | 2×2=4 | 2×1.5=3 | 2×1=2 | 3×1.5=4.5 | **19.5** | 3 |
| ЦП #5: ML прогноз успеваемости | 4×2=8 | 1×2=2 | 4×1.5=6 | 1×1=1 | 5×1.5=7.5 | **24.5** | — |

### Step 3.2: Интерпретация результатов

**После подсчёта проверь:**

**1. Топ-3 — логичны?**
Посмотри на гипотезы с наибольшим баллом. Соответствуют ли они здравому смыслу и твоему пониманию рынка? Если нет — перепроверь оценки по каждому критерию.

**2. Есть ли «тёмная лошадка»?**
Иногда гипотеза с высоким Differentiation но низким Evidence заслуживает дополнительного исследования до валидации — это может быть «голубой океан».

**3. Что делать с гипотезами низкого балла?**
- Балл < 15: не тратить ресурсы. Пометить как «low priority», не отбрасывать — вернуться при pivot.
- Балл 15–25: рассмотреть позже, если топ-гипотезы не подтвердятся.
- Балл > 25: кандидат на первичную валидацию.

---

### Step 3.3: Оформи обоснование выбора топ-3

Напиши короткий документ (0.5–1 страница) с обоснованием:

```markdown
# Обоснование топ-гипотез для валидации — Stage 4.2

**Дата:** ГГГГ-ММ-ДД
**Участники скоринга:** [список]

## Топ-1: ЦП #1 — AI-подбор репетитора за 24 часа
**Балл:** 35.5 / 40
**Ключевые сильные стороны:**
- Самая высокая ценность для потребителя (5/5): решает «горящую» боль
- Хорошая доказательная база (4/5): 14 интервью
- Быстро валидируется (4/5): лендинг + таргет за неделю
**Риски:**
- Дифференциация (4/5): конкурент может скопировать за квартал
**Вывод:** Проверяем первой.

## Топ-2: ЦП #2 — Персонализированная замена при отсутствии прогресса
**Балл:** 32.5 / 40
**Ключевые сильные стороны:**
- Наивысшая дифференциация (5/5): уникальная концепция на рынке
- Очень ценна для потребителя (4/5)
**Риски:**
- Более сложная в валидации (3/5): требует лендинга с продуманным описанием
- Валидация займёт больше времени
**Вывод:** Проверяем второй, параллельно с ЦП #1, если позволяют ресурсы.

## Следующие кандидаты
- ЦП #4 (19.5): рассмотреть, если топ-2 не подтвердятся
- ЦП #5 (24.5): интересная гипотеза, но требует данных. Перед валидацией —
  провести 3–5 доп. интервью для проверки доказательной базы.
```

## 🚨 Troubleshooting

### Problem 1: «Все гипотезы набрали примерно одинаковый балл»

**Solution:**
Так бывает, если гипотезы слишком похожи. Проверь: они действительно разные ЦП или это вариации одного? Если вариации — объедини их в одну и проверь более сфокусированную формулировку. Если разные — попробуй скорректировать оценку по Differentiation: если они похожи, дифференциация у них низкая, и это должно отразиться в баллах.

---

### Problem 2: «Команда не соглашается с результатами скоринга»

**Solution:**
Это нормально и ценно. Проведи скоринг независимо каждым участником, потом сравните и обсудите расхождения > 1 балла. Расхождения — точки скрытых предположений, которые важно сделать явными. Итоговый балл — среднее или результат обсуждения.

---

### Problem 3: «Гипотеза с низким баллом кажется интуитивно лучшей»

**Solution:**
Выясни, почему. Если интуиция основана на реальном знании, которое не попало в скоринг — добавь его как источник в Evidence. Если нет — доверяй скорингу больше, чем интуиции. Именно для этого скоринг и нужен.

---

### Problem 4: «Не знаю, как оценить Differentiation — не хватает знаний о рынке»

**Solution:**
Вернись в Stage 3 и заполни пробелы в конкурентном анализе. Если времени нет — поставь Differentiation = 2 (по умолчанию для неизвестной ситуации) и пометь как «уточнить».

---

### Problem 5: «Скоринг занял 5 часов и всё ещё не готов»

**Solution:**
Скоринг — это не исследование. Это структурированная дискуссия на базе уже собранных данных. Если затягивается — значит, чего-то не хватает в исходных данных (знания или карточки гипотез не заполнены). Установи таймер: 15 минут на гипотезу. Если за 15 минут не можешь оценить — фиксируй причину и двигайся дальше.

---

### Problem 6: «После скоринга всё равно не понятно, что делать с остальными гипотезами»

**Solution:**
Остальные гипотезы — в «банк». Они не выброшены, они ждут. После валидации топ-гипотез (шаг 4.3), если они не подтвердятся, эти гипотезы станут следующими кандидатами. Пометь их статус в карточках как «Low priority — в очереди».

## ❓ FAQ

### Q: Нужно ли проводить скоринг с командой или можно одному?

**A:** Желательно с командой (2–4 человека). Разные роли видят разные аспекты: разработчики — Validation Cost, маркетинг — Differentiation, аналитики — Evidence. Независимый скоринг каждым, затем сравнение и обсуждение расхождений — лучший формат.

---

### Q: Что если только одна гипотеза набрала высокий балл, а остальные — низкие?

**A:** Это нормально. Сконцентрируй ресурсы на этой одной гипотезе. Одновременная проверка нескольких — не цель. Цель — найти одну подтверждённую. Но убедись, что у тебя есть 2–3 гипотезы «в запасе» на случай, если первая не пройдёт валидацию.

---

### Q: Как часто нужно пересматривать скоринг?

**A:** Скоринг — это не документ на всю жизнь. Пересматривай при: получении новых данных из исследований, изменении ресурсов, появлении нового знания о конкурентах. В среднем — раз в 2–4 недели на активном этапе Discovery.

---

### Q: Можно ли проверять несколько гипотез параллельно?

**A:** Да, если ресурсы позволяют. Но убедись, что у каждой — отдельный эксперимент с отдельной метрикой. Смешивание экспериментов делает результаты нечитаемыми. Обычно параллельно тестируют 2–3 гипотезы максимум.

---

### Q: Что значит «ресурсы ограничены» — как понять, хватит ли их?

**A:** Ответь на вопрос: «Если мы потратим 2 недели и X рублей на проверку КАЖДОЙ из 5 гипотез — у нас хватит времени/денег до следующей точки принятия решения?» Если нет — скоринг нужен, чтобы выбрать. Если да — можете тестировать шире, но всё равно начинайте с топ-гипотез.
