# 4.3 Обновление Confidence Points

## 🎯 Цель

Обновить CP на основе validation results.

**Время:** 1-2 часа  
**После:** Updated Bunch v2.0

---

## 📋 CP Update Rules

### Successful Validation → Increase CP

**Smoked test (CP 1-3 → Pass):**
```
New CP = 4-5 (some data, needs more validation)
```

**Prototype test (CP 4-6 → Pass):**
```
New CP = 6-7 (validated with small sample)
```

**Customer validation (CP 7-8 → Pass):**
```
New CP = 8-9 (strong validation)
```

### Failed Validation → Action Required

**Options:**
1. **Refine hypothesis** → Re-test
2. **Pivot** → Different approach
3. **Drop** → Remove from Bunch

---

## 📝 Process

### Step 1: Review All Validations (30 min)

**For each feature:**
- Did validation pass or fail?
- How strong was the signal?
- What did you learn?

### Step 2: Update CP (30 min)

**Template:**
```markdown
## Confidence Point Updates

### [Feature 1]
- **Previous CP:** 5/10
- **Validation:** Landing page test (8% conversion)
- **Result:** ✅ Pass
- **New CP:** 7/10 (increased due to positive validation)
- **Notes:** Strong signal, proceed to development

### [Feature 2]
- **Previous CP:** 4/10
- **Validation:** Prototype test (40% completion rate)
- **Result:** ❌ Fail
- **New CP:** Dropped from Bunch
- **Notes:** Users didn't understand value prop, needs rework

### [Feature 3]
- **Previous CP:** 3/10
- **Validation:** Smoke test (12% signups)
- **Result:** ✅ Pass
- **New CP:** 5/10 (good signal, but needs more validation)
- **Notes:** Proceed to prototype validation
```

### Step 3: Create Updated Bunch (30 min)

**Template:**
```markdown
# Bunch v2.0 (Post-Validation)

## Confirmed Features (5-7)
1. **[Feature 1]** — CP: [X]/10 — Ready for development
2. **[Feature 2]** — CP: [X]/10 — Ready for development

## Needs More Validation (0-3)
1. **[Feature A]** — CP: [X]/10 — Needs additional testing

## Dropped (from original Bunch)
1. **[Feature X]** — Reason: [Why dropped]

## Summary
- **Original Bunch:** [X] features
- **Validated:** [X] features
- **Confirmed:** [X] features
- **Dropped:** [X] features
- **Average CP:** Increased from [X] to [Y]
```

**Updated Bunch v2.0** ready for stakeholder presentation
