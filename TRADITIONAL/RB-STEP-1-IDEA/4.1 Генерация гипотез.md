# 4.1 Генерация гипотез

## 🎯 Цель

Создать Hypothesis Cards для каждой Bunch feature.

**Время:** 1-2 часа  
**После:** Hypothesis Cards для всех Bunch items

---

## 📋 Что такое Hypothesis Card?

**Простыми словами:** Hypothesis Card = structured statement о том, что мы проверяем.

---

## 📝 Process

### Step 1: Create Hypothesis Cards (1-2 hours)

**Template:**
```markdown
## Hypothesis Card: [Feature Name]

### Hypothesis Statement
"We believe that [feature]
for [target segment]
will achieve [mNSM outcome]
if [success criteria]."

### We'll Know This Is True When
- **Metric 1:** [Metric] reaches [threshold] by [timeframe]
- **Metric 2:** [Metric] reaches [threshold] by [timeframe]

### Validation Method
- **Method:** [Smoke test/Prototype/Interview/etc]
- **Target:** [X] customers/users
- **Timeframe:** [X] days

### Current CP
- **Confidence Point:** [X]/10
- **Risk level:** [Low/Medium/High]

### Dependencies
- **Must exist first:** [What needs to be true?]
- **Blocked by:** [What might prevent this?]
```

**Пример:**
```markdown
## Hypothesis Card: AI-Powered Audience Insights

### Hypothesis Statement
"We believe that AI-powered audience insights
for SMB marketing teams (1-10 people)
will achieve $5k MRR
if 50% of users upgrade after 30-day trial."

### We'll Know This Is True When
- **Trial-to-paid:** 50% conversion within 60 days
- **Retention:** 40% retain after 90 days
- **NPS:** ≥40

### Validation Method
- **Method:** Landing page smoke test
- **Target:** 500 visitors, 50 signups
- **Timeframe:** 14 days

### Current CP
- **Confidence Point:** 6/10
- **Risk level:** Medium (new tech, but clear market need)

### Dependencies
- **Must exist first:** AI API access, data sources
- **Blocked by:** Regulatory changes, API limits
```
