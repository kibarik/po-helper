# 📋 Шаблоны для Customer Validation

## 🔹 Шаблон 1: Карточка гипотезы потребителя

### Базовая информация:
```
ID гипотезы: H-001
Дата создания: DD.MM.YYYY
Статус: ⬜ Новая | ✅ Подтверждена | ❌ Опровергнута | 🔄 В процессе
```

### Тип гипотезы:
⬜ Гипотеза поведения
⬜ Гипотеза проблемы
⬜ Гипотеза мотивации
⬜ Гипотеза блокеров

### Формулировка гипотезы:

#### Для гипотезы поведения:
```
Когда <роль>, 
<описание ситуации>, 
то <описание поведения>, 
потому что <мотивация>.
```

**Пример:**
```
Когда [项目经理 一线城市],
[需要协调团队任务并且当前使用的工具很慢],
[то он переключается на email и phone calls],
[потому что это быстрее и надежнее, чем ждать загрузки в current tool].
```

#### Для гипотезы проблемы:
```
Когда <роль>, 
<описание ситуации>, 
то <описание проблемы>, 
а это мешает <ожидаемый результат>.
```

**Пример:**
```
Когда [项目经理 一线城市],
[нужно быстро распределить задачи между членами команды],
[то он вынужден тратить 30+ минут на ручное распределение],
[а это мешает [他快速响应客户需求并且保持团队效率].
```

#### Для гипотезы мотивации:
```
<Роль> хочет 
<описание желаемого поведения>, 
а не <описание текущего поведения>, 
чтобы <ожидаемый результат>.
```

**Пример:**
```
[项目经理] хочет
[одним кликом распределять задачи с учетом загрузки команды],
а не [тратить 30+ минут на ручное распределение],
чтобы [мочь мгновенно реагировать на изменения требований клиентов].
```

#### Для гипотезы блокеров:
```
<Роль> 
<описание поведения/проблемы>, 
потому что <описание причины поведения/проблемы>.
```

**Пример:**
```
[项目经理] 
[тратит 30+ минут на ручное распределение задач каждый день],
потому что [не доверяет автоматическим инструментам и боится, что важная задача будет упущена].
```

### Критерий валидации:
```
Гипотеза подтверждена, если:
- [ ] Минимум 70% респондентов подтверждают поведение/проблему
- [ ] Приведены конкретные примеры из реального опыта
- [ ] Указаны причины/мотивация

Гипотеза опровергнута, если:
- [ ] Менее 30% респондентов подтверждают
- [ ] Поведение отличается от предполагаемого
- [ ] Причины/мотивация иные
```

### Оценка гипотезы (ICE/RICE):

```
Impact (Влияние): [1-10]
Confidence (Уверенность): [1-10]  
Ease (Легкость): [1-10]

ICE Score: [ ]
```

### Связанные гипотезы:
```
Данная гипотеза связана с:
- [ ] H-XXX (другая гипотеза)
- [ ] Профиль потребителя: [название]
- [ ] Ценностное предложение: [название]
```

## 🔹 Шаблон 2: Скрипт深度интервью

### Introduction (2-3 минуты):
```
"Здравствуйте! Меня зовут [Ваше Имя], я работаю над [краткое описание продукта].

Я провожу исследования, чтобы лучше понять, как люди решают [тема исследования].

Сегодняшний разговор займет около 45-60 минут. Нет правильных или неправильных ответов — мне интересно именно ваш опыт и мнение.

Все, что вы расскажете, останется конфиденциальным и будет использовано только в агрегированном виде.

Если вам комфортно, я буду делать заметки во время нашего разговора. У вас есть ко мне вопросы?"
```

### Warm-up (5 минут):
```
1. Расскажите немного о себе:
   - Кем вы работаете?
   - Сколько времени вы в этой сфере?
   - Какие задачи вы решаете ежедневно?

2. Расскажите о последнем раз, когда вы [ситуация из гипотезы]:
   - Когда это было?
   - В каком контексте это произошло?
```

### Core Questions (30-40 минут):

#### Block 1: Альтернативные решения
```
1. Как вы сейчас решаете [проблема/задача]?
   - Расскажите шаг за шагом, как вы в последний раз [решали проблему]?
   
2. Какие инструменты/способы вы используете?
   - Почему выбрали именно их?
   
3. Как часто вы сталкиваетесь с [проблема]?
   - В каких ситуациях это происходит?
   
4. Сколько времени это занимает у вас?
   - Сколько ресурсов (деньги, нервы) вы тратите?
```

#### Block 2: Проблемы и причины
```
5. С какими проблемами вы сталкиваетесь, когда [решаете задачу]?
   - Приведите 2-3 последних примера
   
6. Почему для вас это является проблемой?
   - Что именно вас беспокоит?
   - К чему это приводит?
   
7. Что для вас является самым сложным/дорогим/затратным?
   - Почему?
   
8. Пробовали ли вы решать это по-другому?
   - Что именно пробовали?
   - Почему это не сработало?
```

#### Block 3: Цели и ожидания
```
9. К чему вы стремитесь, когда [решаете задачу]?
   - Какой идеальный результат для вас?
   
10. Что для вас означает успех в этом процессе?
    - Как вы измеряете, что у вас "получилось"?
    
11. Что бы вы изменили в текущем процессе, если бы могли?
    - Почему именно это?
```

#### Block 4: Мотивация
```
12. Почему для вас важно [решать эту проблему]?
    - Что происходит, если вы не решаете её?
    
13. К каким более высоким целям это приводит?
    - Зачем вам это нужно в принципе?
    
14. Что было бы, если бы эта проблема исчезла?
    - Как бы это изменило вашу работу/жизнь?
```

#### Block 5: Willingness to Pay
```
15. Сколько вы тратите сейчас на решение [проблемы]?
    - Деньги, время, ресурсы
    
16. Сколько вы готовы заплатить за идеальное решение?
    - Почему именно столько?
    
17. Вы бы предпочли:
    - [ ] Единоразовую плату
    - [ ] Ежемесячную подписку
    - [ ] Оплату за использование
```

### Deep Dive (на основе ответов):
```
[Подготовьте дополнительные вопросы на основе ответов респондента]

Примеры:
- "Вы упомянули X. Расскажите об этом подробнее..."
- "Почему вы выбрали именно этот подход?"
- "Что произошло в последний раз, когда вы попробовали Y?"
```

### Wrap-up (5 минут):
```
18. Есть ли что-то еще, что вы бы хотели добавить о [тема]?
    
19. Если бы вы могли изменить одну вещь в [текущий процесс], что бы это было?
    
20. Знакомы ли вам другие люди, которые сталкиваются с [проблема]?
    - Можем ли мы связаться с ними?

Спасибо огромное за ваше время! Ваш опыт очень полезен для нас. Если у вас появятся дополнительные мысли, я буду рад(а) услышать их."
```

```
|----|----------|----------------|-------------------|-------------|-----------|----------|

ICE Score = (Impact + Confidence + Ease) / 3

Выбрать топ-10 гипотез для проверки.
```

### RICE Scoring:

```
| ID | Гипотеза | Reach | Impact | Confidence | Effort | RICE Score | Priority |
|----|----------|-------|--------|------------|--------|------------|----------|
| H-001 | [Текст] | [ ] | [ ] | [ ] | [ ] | [ ] | [ ] |
| H-002 | [Текст] | [ ] | [ ] | [ ] | [ ] | [ ] | [ ] |
| H-003 | [Текст] | [ ] | [ ] | [ ] | [ ] | [ ] | [ ] |

RICE Score = (Reach × Impact × Confidence) / Effort

Reach: Количество людей за квартал
Impact: 3 = огромное, 2 = большое, 1 = среднее, 0.5 = небольшое, 0.25 = минимальное
Confidence: 100% = 1, 80% = 0.8, 50% = 0.5
Effort: Человеко-месяцы
```

## 📝 Инструкция по использованию шаблонов:

### 1. Гипотезы:
- Скопируйте "Шаблон 1" для каждой гипотезы
- Заполните все поля
- Сохраните в отдельную базу (Notion/Google Sheets)

### 2. Интервью:
- Используйте "Шаблон 2" как основу для скрипта
- Адаптируйте под ваш контекст
- Подготовьте дополнительную questions

### 3. Результаты интервью:
- Заполняйте "Шаблон 3" сразу после каждого интервью
- Не откладывайте обработку на потом
- Фиксируйте прямые цитаты

### 4. Профили потребителей:
- Создавайте профиль по "Шаблону 4" после 6+ интервью
- Обновляйте по мере получения новых данных
- Добавляйте цитаты и примеры

### 5. Скоринг:
- Используйте "Шаблон 5" для приоритизации гипотез
- Используйте "Шаблон 6" для выбора фокус-сегмента
