# 2.3 instruction — Linza Product через JTBD-агент

> Этап **RB-STEP-1 — BIG IDEA** (прелюдия) · Линза: **Product** (Product Sprint → Нексус продукта → Банч) [S1] VI.1 · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-1-IDEA/2.3 Linza Product.md` — JTBD-интервью (5–10, гайд), сегментация по hiring criteria, Empathy Map, Problem Statement (HMW), Ideation (Brainwriting/SCAMPER/Reverse). Эта инструкция: анализ транскриптов **JTBD-агентом** (RAG по корпусу интервью), **авто-сегментация** (с workslop-фильтром «персона ≥ N интервью»), авто-генерация Empathy Map — преемственность методов The Mom Test + JTBD сохранена.

## 🎯 Цель в PAF-оптике

Через Product Lens провести скаутинг возможностей продукта на основе **JTBD**
(люди «нанимают» продукт для работ: functional/emotional/social jobs — концепт из исходника 2.3).

В PAF-оптике цель — не «провести 5 интервью и описать персоны»,
а **насытить Нексус продукта** Узлами customer context до CR,
при котором Кортекс генерирует фич-кандидаты Банча с высоким CP
(Identify-фаза Und-Id-Ex [S1] VI.5).
Результат — `JTBD-Opportunity`-узлы и `Segment`-узлы (поведенческие, не демографические).

## 👤 Роль и ответственность

- **Product Engineer** — формулирует гайд интервью
  (The Mom Test: фокус на истории, не мнениях — преемственность из исходника 2.3);
  валидирует сегментацию;
  veto на AI-сгенерированные персоны без evidence.
- **Product Ops** — поддерживает **JTBD RAG** (индекс по корпусу транскриптов);
  настройка ASR (автоматическое распознавание речи) → кодирование.
- **Discovery/Launcher PM** (если есть) — вайбкодит smoke-прототип для проверки top JTBD [S2] III.6.
- **Зона человека:** **проведение интервью** (на ранней зрелости Кортекса) —
  эмпатия и контекст интервью не делегируются;
  ИИ — кодировщик транскриптов, не интервьюер [reformation] §2.9.

## 🤖 Что делает Кортекс-агент

**JTBD-агент** [reformation] §1.5 — анализ транскриптов интервью:

- **Кодирование:** ASR-транскрипт → извлечение jobs (functional/emotional/social),
  triggers, hiring criteria, quotes.
- **Авто-сегментация:** кластеризация по hiring criteria (не демографии)
  → `Segment`-узлы (поведенческие).
- **Workslop-фильтр:** `Segment`-узел создаётся только при ≥N интервью в кластере
  (правило «персона ≥4 интервью» [reformation] §2.9) —
  защита от AI-сгенерированных персон.
- **Авто-генерация Empathy Map** (Says/Thinks/Does/Feels/Pains/Gains — из исходника 2.3) из цитат.
- **Авто-формулировка Problem Statement (HMW).**

- **Вход:** JTBD RAG (корпус транскриптов) + гайд интервью.
- **Выход:** `JTBD-node` (functional/emotional/social jobs),
  `Segment`-узел, `Empathy-Map`-узел, `Problem-Statement`-узел,
  `JTBD-Opportunity`-узел.

**3 типа Jobs** (преемственность из исходника 2.3):
- Functional — что делают;
- Emotional — что чувствуют;
- Social — как выглядят в глазах других.

**Промпт-идея:**
> «Закодируй транскрипты: jobs (3 типа), triggers, hiring criteria, quotes.
> Кластеризуй в сегменты (поведенческие).
> Не создавай Segment-узел, если в кластере <4 интервью.
> Сгенерируй Empathy Map из цитат.
> Для каждой JTBD-Opportunity — initial CP и цитата-доказательство.»

## 🧠 Что делает человек (метапознание)

- **Главный workslop-риск стадии:** AI-сгенерированные «персоны» без evidence [reformation] §2.9.
  Инженер проверяет: у каждого `Segment`-узла ≥N цитат-доказательств?
- **Veto-право:** JTBD-Opportunity без цитаты-доказательства → CP=0.
  Цитата обязана быть дословной с указанием интервью.
- **Вопрос:**
  - «Этот сегмент — поведенческий (по hiring criteria) или демографический?
    Какая цитата подтверждает job?»
  - «Empathy Map в Says — это дословные цитаты или парафраз Кортекса?»

## 📦 Артефакт = узлы Нексуса

- **Создаются/обновляются:** `JTBD-node` (functional/emotional/social),
  `Segment` (поведенческий, с ≥N интервью),
  `Empathy-Map`, `Problem-Statement` (HMW),
  `JTBD-Opportunity` (10–20, из исходника 2.3), `Quote` (доказательство).
- **Метаданные:** `источник` (интервью[]), `CP`, `timestamp`,
  `цитата-доказательство`, `статус увядания`.
- **Прирост Context Ripeness:** +15–20% к Нексусу продукта (customer context — ядро).

## 🚦 Confidence Point-гейт

- **Критерий перехода к 3.1:** ≥10 `JTBD-Opportunity`-узлов с цитатами-доказательствами,
  ≥2 `Segment`-узла (каждый ≥4 интервью), ≥3 `Problem-Statement`.
- **Стоимость риска:** «Суррогатная валидация по форумам/отзывам вместо живых интервью»
  [reformation] §2.9 — помечать source типа.

## 🔭 Связь с Тремя Линзами (3PL)

- **Активная:** **Product Lens** [S1] VI.1.
- **Связь со Strategy Lens:** `JTBD-Opportunity` → `Strategic-Opportunity` (2.1)
  через `Gap-рынка`.
- **Связь с Business Lens:** JTBD → mNSM (какая job монетизируется, 2.2).

## 🛡️ Анти-workslop чекпоинт

- **Workslop-риск:** **AI-персоны без интервью** (главный риск Customer-стадий);
  суррогатная валидация; генерация JTBD не из транскриптов, а из «общеизвестного».
- **Признак:** Segment без цитат; JTBD без интервью-источника;
  Empathy Map без дословных Says.
- **Правило верификации:** `Segment` ≥4 интервью [reformation] §2.9;
  каждая `JTBD-Opportunity` — цитата; CP=0 без источника;
  ИИ только кодировщик (на ранней зрелости), не интервьюер.
