# 3.1 instruction — Консолидация = Understand-фаза Und-Id-Ex

> Этап **RB-STEP-1 — BIG IDEA** (прелюдия) · Линза: **над 3PL** (сборочная, агрегирует все три линзы) · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-1-IDEA/3.1 Консолидация.md` — агрегация opportunities из 3 линз в master list (18–35), дедупликация/кластеризация, preliminary ranking по impact/feasibility/confidence. Эта инструкция: переопределяет консолидацию как **Understand-фазу методики Und-Id-Ex** [S1] VI.5 (Understand → Identify → Execute) и заменяет «ranking» на **авто-кластеризацию opportunities по узлам Нексуса** ([reformation] §1.7).

## 🎯 Цель в PAF-оптике

Консолидация в PAF — это **Understand-фаза Und-Id-Ex** [S1] VI.5:
анализ Нексуса (рынка, продукта, роста) и сбор всех opportunities в единый граф,
сгруппированный по Узлам-источникам.

Цель — не «master list в Notion», а **подготовка к Identify-фазе**:
Кортекс на следующем шаге (3.2 CP, 3.3 Банч) будет генерировать фичи из этого графа.

На null base это discovery с нуля (гипотезы потребностей → ЦП → реализации) [S1] VI.5;
на data base — извлечение знаний о поведении.

## 👤 Роль и ответственность

- **Product Engineer** — задаёт логику кластеризации
  (по Узлам Нексуса, по линзам, по типам opportunity);
  валидирует дедупликацию.
- **Product Ops** — поддерживает граф opportunities как рёбра к Узлам Нексуса
  (не плоский список).
- **Зона человека:** решение, какой кластер opportunity становится кандидатом в Банч
  (позже в 3.3) — стратегический выбор, не автоматика.

## 🤖 Что делает Кортекс-агент

**Consolidation-агент** (режим Scouting Agent [reformation] §1.5, Understand-фаза):

- Агрегирует `Strategic-Opportunity` (2.1) + `Monetization-Hypothesis` (2.2)
  + `JTBD-Opportunity` (2.3) в единый граф.
- **Авто-кластеризация по Узлам Нексуса:**
  opportunities группируются не по произвольным темам (как в исходнике 3.1),
  а по Узлам-источникам
  (к какому `Competitor`/`Trend`/`Gap`/`JTBD`/`Growth-Lever` привязаны).
- **Дедупликация** по семантической близости (эмбеддинги)
  с пометкой «объединено из [линзы]».
- **Флаги перекрёстных opportunities:**
  встречаются во всех 3 линзах → высокий приоритет для Identify.

- **Вход:** графы opportunities из 2.1/2.2/2.3 + Узлы Нексусов.
- **Выход:** `Opportunity-Cluster`-узлы с рёбрами к Узлам Нексуса,
  `Cross-Lens-Opportunity`-флаги.

**Промпт-идея:**
> «Агрегируй opportunities из 3 линз.
> Кластеризуй по Узлам Нексуса-источника (не по темам).
> Дедуплицируй. Пометь Cross-Lens те, что встречаются во всех 3 линзах.
> Для каждого кластера — initial CP как медиана CP источников.»

## 🧠 Что делает человек (метапознание)

- **Чекпоинт:** дедупликация Кортексом может **объединить неравное**
  (JTBD-opportunity и strategic-opportunity, которые звучат похоже, но разные).
  Инженер проверяет рёбра: какой Узел-источник у каждого opportunity?
- **Veto-право:** Cross-Lens-Opportunity без явного ребра к Узлу каждой линзы
  → понижение CP.
- **Вопрос:**
  - «Этот кластер — реальная сходимость трёх линз или артефакт эмбеддингов?»
  - «Кортекс объединил 2 opportunities — у них один Узел-источник или разные?»

## 📦 Артефакт = узлы Нексуса

- **Создаются/обновляются:** `Opportunity-Cluster` (с рёбрами к Узлам Нексуса),
  `Cross-Lens-Opportunity` (флаг),
  мета-узел `Understand-Snapshot` (снимок Understand-фазы).
- **Метаданные:** `источники` (линзы + Узлы), `CP` (медиана), `timestamp`,
  `cross-lens-flag`, `статус увядания`.
- **Прирост Context Ripeness:** +5–10%
  (консолидация углубляет связи между Узлами).

## 🚦 Confidence Point-гейт

- **Критерий перехода к 3.2:** существует `Understand-Snapshot`
  с ≥3 `Opportunity-Cluster`,
  каждый кластер имеет рёбра к Узлам Нексуса;
  минимум 1 `Cross-Lens-Opportunity`.
- **Стоимость риска:** «Если консолидация поверхностная —
  Identify-фаза сгенерирует фичи из шума» — фиксируется в снимке.

## 🔭 Связь с Тремя Линзами (3PL)

- Над 3PL: консолидация собирает все три линзы. [S1] VI.1
- **Связь:** `Cross-Lens-Opportunity` — это opportunities,
  видимые из всех трёх линз одновременно
  (самые сильные кандидаты в Банч).

## 🛡️ Анти-workslop чекпоинт

- **Workslop-риск:** **master list ради master list**
  (18–35 opportunities без связи с Узлами);
  кластеризация по темам, а не по evidence.
- **Признак:** кластер без рёбер к Узлам Нексуса;
  Cross-Lens-флаг без доказательства из 3 линз.
- **Правило верификации:** каждый `Opportunity-Cluster` — рёбра к Узлам;
  Cross-Lens — evidence из каждой линзы;
  CP=медиана источников, не интуиция.
