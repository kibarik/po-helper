# 1.1 instruction — Нексус Рынка как цифровой двойник

> Этап **RB-STEP-1 — BIG IDEA** (прелюдия) · Линза: **Strategy** (Strategic Sprint → Нексус рынка → Ставки/Bets) [S1] VI.1 · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-1-IDEA/1.1 Нексус Рынка.md` — 5 секций (Market Sizing TAM/SAM/SOM, Competitive Landscape 4 типов, Trends, Opportunities, Threats) с ручным заполнением и CR как галочным чеклистом (Step 5.2). Эта инструкция: (а) превращает секции в **граф Узлов** с метаданными, (б) делает **Context Ripeness вычисляемой live-метрикой** с фазой «увядания» ([reformation] §1.3 разрыв 1, 2), (в) вводит **Nexus-Builder Agent** ([reformation] §1.5).

## 🎯 Цель в PAF-оптике

Насытить **Нексус рынка** до Context Ripeness, достаточного для скаутинга возможностей/угроз
(Принцип 5 [S1]) и поднятия CP рыночных гипотез до CP-gate стратегической линзы.
Рынок — объект Strategy Lens: Strategic Sprint оперирует Нексусом рынка и Ставками (Bets) [S1] VI.1.

TAM/SAM/SOM, тренды, конкуренты — не «отчёт», а **аргументы насыщения Банча**
на фазе Understand методики Und-Id-Ex [S1] VI.5.
В методике это null base (новый рынок → discovery с нуля) или data base
(есть рынок → извлекаем знания о поведении).

## 👤 Роль и ответственность

- **Product Engineer** — формулирует задачи Nexus-Builder Agent по типам Узлов;
  валидирует цифры; поднимает CP рыночных гипотез.
- **Product Ops** — поддерживает Nexus RAG (индекс по Нексусу рынка),
  выставляет пороги свежести по типам Узлов (TAM устаревает медленнее, чем отзывы конкурентов).
- **Portfolio Manager** (если есть) — потребляет Нексус рынка для Ставок (Bets) стратегической линзы [S1] VI.1.
- **Зона человека:** признание **Ставки** (на какой рыночный сценарий делаем ставку) —
  стратегическое решение, не передаётся ИИ [S2] III.4.

## 🤖 Что делает Кортекс-агент

**Nexus-Builder Agent** [reformation] §1.5 — заполняет Узлы Нексуса рынка из внешних источников
(отчёты, G2/Capterra/Reddit, App Store, SimilarWeb), считает Context Ripeness, ставит флаг «увядания».

- **Вход:** RAG по рынку + целевые URL/источники, заданные инженером.
- **Действие:** парсит источники → создаёт/обновляет Узлы с метаданными →
  вычисляет CR (полнота по покрытию типов Узлов × актуальность по `timestamp`) → помечает увядание.
- **Выход:** Узлы типов `Force`, `Trend`, `Constant`, `TAM-SAM-SOM`,
  `Competitor` (4 типа: прямые/косвенные/новые/заменители — из исходника 1.1),
  `Gap-рынка`, `Bet` (Ставка, Strategy Lens).

**Структура Узлов Нексуса рынка** (преемственность 5 секций исходника 1.1):
- Market Sizing → `TAM-SAM-SOM` (два метода: top-down + bottom-up, с автосверкой);
- Competitive Landscape → `Competitor` (4 типа — прямые/косвенные/новые/заменители);
- Trends → `Trend` (technology/regulatory/consumer behavior);
- Opportunities → `Gap-рынка` + `Strategic-Opportunity`-кандидаты;
- Threats → `Force` (рыночные силы) + `Bet`-риски.

**Промпт-идея:**
> «Создай/обнови Узлы Нексуса рынка.
> Для TAM-SAM-SOM обязательно два метода (top-down + bottom-up) с автосверкой.
> Для каждого Competitor — 4 типа (прямые/косвенные/новые/заменители).
> Каждый числовой Узел обязан хранить URL+дату.
> CP=0 для любого Узла без верифицированного источника.
> Маркируй каждый Узел: fact/hypothesis/benchmark.»

## 🧠 Что делает человек (метапознание)

- **Критический чекпоинт:** рыночные цифры — топ-зона ИИ-галлюцинаций во всей библиотеке
  [reformation] §1.8.
  Каждый числовой Узел (TAM/SAM/SOM, темпы роста, доли) инженер проверяет:
  «Откуда эта цифра? Какой URL? Какая дата?»
- **Veto-право:** Узел без `источника` → CP=0 и пометка `fact/hypothesis/benchmark` [reformation] §1.8.
- **Вопрос:**
  - «TAM = $15 млрд — это top-down из отчёта или bottom-up из (клиенты × цена)?
    Расходятся ли два метода больше чем на порядок?»
  - «Этот конкурент — прямой или косвенный? Какие отзывы подтверждают слабость?»

## 📦 Артефакт = узлы Нексуса

- **Создаются/обновляются:** `Force`, `Trend`, `Constant`,
  `TAM-SAM-SOM` (с двумя методами), `Competitor` (×4 типа), `Gap-рынка`, `Bet`.
- **Метаданные каждого Узла:**
  - `источник` (URL/интервью/analytics);
  - `CP` (0–10);
  - `timestamp`;
  - `статус увядания` (fresh/aging/stale);
  - `тип` (fact/hypothesis/benchmark).
- **Прирост Context Ripeness:** целевой скачок 0% → ≥60% по Нексусу рынка [S1] III.1
  (порог для стратегических решений).

## 🚦 Confidence Point-гейт

- **Критерий перехода к 1.2:** CR Нексуса рынка ≥60%
  **и** минимум один `Bet`-узел (Ставка стратегической линзы) с CP≥4.
- **Стоимость риска** (Принцип 4 [S1]): «Если TAM завышен вдвое —
  сколько ресурсов мы сожжём на SOM, которого нет?» — фиксируется в `Bet`-узле.

## 🔭 Связь с Тремя Линзами (3PL)

- **Активная:** **Strategy Lens** [S1] VI.1.
- **Связь с Business Lens:** SOM → потенциал mNSM (см. 1.3 Нексус Роста).
- **Связь с Product Lens:** `Gap-рынка` → фич-кандидаты Банча (см. 2.3).

## 🛡️ Анти-workslop чекпоинт

- **Workslop-риск:** **убедительные, но неверные рыночные цифры** [S2] III.7;
  TAM как самоцель (vanity).
- **Признак:** TAM посчитан одним методом без сверкой;
  конкуренты без отзывов; тренды без `timeline`.
- **Правило верификации:** TAM/SAM/SOM — два метода + автосверка [reformation] §3.5;
  каждый числовой Узел — URL+дата; CP=0 без источника.
