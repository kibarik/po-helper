# 5.2 instruction — Проектирование монетизации из mNSM с агент-A/B-симуляцией цен и NPV-оценкой схем

> Этап **RB-STEP-5 — BUSINESS MODEL VALIDATION** (Stage 4) · Линза: Business · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-5-BUSINESS-MODEL-VALIDATION/5.2 Схема монетизации.md`.

## 🎯 Цель в PAF-оптике
Спроектировать узел `monetization` в Нексусе системы роста, **выводя схему монетизации из mNSM** (монетизируемой NSM), а не из абстрактного «как зарабатывать». Схема определяется по 5 измерениям (тип платежа / предмет оплаты / плательщик / ценообразование / момент оплаты) [из 5.2], а ценовой коридор и NPV каждой схемы оцениваются через **Кортекс** — агент-A/B-симулятор цен с синтетическими агентами-клиентами (4-я роль «секретарь/адъютант» [S2 III.8]). Главная метрика — NPV схемы монетизации, не «выбранный тариф».

## 👤 Роль и ответственность
- **Growth Engineer** — владелец узла `monetization`; максимизация mNSM; определение «юнита оплаты» (1 урок / 1 месяц / 1 сделка) и его связи с mNSM.
- **Инженер по развитию бизнеса / Bizdev Architect** — NPV-оценка схем монетизации; выбор схемы с максимальным NPV при приемлемом риске.
- **Product Ops** — настройка A/B-симулятора цен (синтетические агенты-клиенты), RAG по WTP из интервью Stage 2–3 и ценам конкурентов.
- **Зона человека:** выбор ценового коридора и момента оплаты — стратегическое решение о позиционировании (дорого/дёшево = сигнал качества), не передаётся ИИ как актору [S2 III.4]. Кортекс симулирует, человек решает.

## 🤖 Что делает Кортекс-агент
- **Проектировщик монетизации** — из узла mNSM (из 4.5/4.6) и узла `bm-hypothesis` (из 5.1) выводит 5 измерений схемы: тип платежа (разовый/регулярный/гибрид), предмет оплаты (сделка/ресурсы/доступ/результат/без монетизации от пользователя), плательщик (потребитель/поставщик/обе стороны/третья сторона), ценообразование (фикс/%/tiered/freemium), момент оплаты (до/после/в момент) [из 5.2 Phase 1–3].
- **Ценовой коридор из 3 методов** — рассчитывает коридор по трём методам ценообразования [из 5.2]: (1) cost-plus (от COGS из 5.4 вверх), (2) value-based (от дельты ценности для клиента из 4.x), (3) competitor-based (от цен конкурентов из Нексуса рынка). Пересечение трёх методов = целевой коридор.
- **A/B-симулятор цен** — генерирует синтетических агентов-клиентов (4-я роль «секретарь/адъютант» [S2 III.8]) с параметрами из портретов Stage 2–3 и WTP из интервью; симулирует конверсию при разных ценовых точках; выдаёт кривую эластичности спроса по цене.
- **NPV-оценщик схем** — для каждой схемы монетизации считает NPV (через ARPU × Lifetime − CAC − COGS, дисконтированный); ранжирует схемы по NPV.
- **Sanity-чек** — WTP должна быть из интервью (RAG по транскриптам Stage 2–3); синтетическая конверсия маркируется как симуляция, не a posteriori.

## 🧠 Что делает человек (метапознание)
- **Отличить симуляцию от данных:** агент-A/B-симулятор даёт «убедительные, но потенциально неверные» цифры конверсии [Harvard × BCG, S2 III.4]. Человек обязан маркировать: симулированная конверсия ≠ валидированная WTP; реальная WTP подтверждается только в Stage 6–8 (Soft Launch, A/B).
- **Маркировка факт/гипотеза/бенчмарк:** цены конкурентов — факт (с URL+датой); WTP из интервью — гипотеза с цитатой; симулированная конверсия — бенчмарк/модель.
- **Veto на схему:** выбор схемы монетизации с максимальным NPV, но с приемлемым стратегическим риском (например, отказ от слишком низкой цены, разрушающей восприятие ценности) — за Bizdev Architect.
- **Проверка mNSM-связи:** каждая схема монетизации должна быть связана ребром с mNSM; схема, не монетизирующая mNSM, — отбраковывается.

## 📦 Артефакт = узлы Нексуса системы роста
- `monetization` (агрегирующий узел) с CP.
- 5 дочерних узлов по измерениям схемы (тип платежа, предмет оплаты, плательщик, ценообразование, момент оплаты), каждый с `источник[]`, `CP`, `факт/гипотеза/бенчмарк`.
- `price-corridor` — коридор из 3 методов (cost-plus / value-based / competitor-based) + целевая ценовая точка.
- `price-elasticity-curve` — кривая эластичности из A/B-симулятора (маркирована как симуляция).
- `monetization-npv` — NPV каждой рассмотренной схемы + ранжирование.
- **Context Ripeness** узла `monetization` — целевой ≥50%.

## 🚦 Confidence Point-гейт
**CP-gate монетизации (5.2):** {тип платежа + предмет оплаты + плательщик определены и обоснованы} ∧ {ценовой коридор из ≥2 методов} ∧ {NPV хотя бы одной схемы > 0 на реалистичном сценарии} ∧ {схема связана с mNSM} ∧ {CR ≥ 50%}. Это гейт, прописанный в reformation.md §5.7 как первый из трёх CP-gate стадии 5.

## 🔭 Связь с Тремя Линзами (3PL)
**Business Lens.** Монетизация — прямой **Рычаг (Lever)**: цена (ARPU) и тип платежа — измеримые свойства БМ, рост NPV при инвестиции. Связь с Product Lens — через mNSM (схема монетизирует именно mNSM); со Strategy Lens — через позиционирование цены (дорого/дёшево относительно конкурентов = ставка портфеля).

## 🛡️ Анти-workslop чекпоинт
- **Workslop в ценовых схемах:** «убедительный» ценовой коридор без обоснования методом. Гейт: каждый коридор — через ≥2 из 3 методов, с источником.
- **Галлюцинации WTP:** симулированная WTP не подставляется как факт; реальная WTP — только из интервью с цитатой.
- **Ложное ощущение готовности:** A/B-симуляция ≠ реальный A/B (Stage 6–8); симулированная конверсия не поднимает CP без реальных данных.
- **NPV как чёрный ящик:** формула NPV схемы раскрывается (ARPU × Lifetime − CAC − COGS, ставка дисконтирования); входные параметры трассируются до узлов 5.1/5.4.
