# 2.6 instruction — AI Segmenter with Workslop Filter

> Этап **RB-STEP-2 — CUSTOMER VALIDATION** (Stage 1) · Линза: Product (контекст: Business/Strategy) · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-2-CUSTOMER-VALIDATION/2.6 Формирование профилей сегментов.md` (кластеризация по поведению, разрезы сегментации, правило «демография — в конце», профили сегментов).

## 🎯 Цель в PAF-оптике
Заменить ручную кластеризацию и ручные CJM/Empathy Map на **Сегментатор с workslop-фильтром**: Кортекс кластеризует узлы Нексуса в сегменты по поведенческим разрезам (не по демографии), с явным **правилом «персона ≥4 интервью»** и **метапознание-чеком** против AI-сгенерированных персон. Сегмент = подграф Нексуса с уникальным триггером/путём/проблемой, а не «карточка в Notion» [reformation 2.6, 2.9].

## 👤 Роль и ответственность
- **Product Engineer** — задаёт разрез сегментации (ситуация-триггер / уровень боли / поведение / цель / мотивация / ролевой контекст / опыт — из исходного `2.6`); валидирует сегменты.
- **Product Ops** — поддерживает workslop-фильтр (правило ≥4 интервью на персону) и метапознание-гейт.
- **Growth Engineer** — добавляет к каждому сегменту mNSM-профиль (платёжеспособность, готовность платить).
- **Зона человека**: выбор фокус-сегмента — стратегический вызов, переходит в 2.7.

## 🤖 Что делает Кортекс-агент
**Сегментатор (AI Segmenter Agent) с workslop-фильтром:**
- **Вход**: спелый подграф Нексуса потребителя из 2.5 (единицы знания о поведении).
- **Действие**: кластеризует узлы по поведенческим разрезам; формирует профили сегментов (название/описание/триггеры/начальное→целевое/мотивация/поведение/ключевые проблемы/ценности/уровень боли — структура из исходного `2.6`); применяет **workslop-фильтр**: персона = узел Сегмент допускается только при ≥4 подтверждающих интервью (иначе отбрасывается как AI-выдумка); метапознание-чек.
- **Выход**: 2–5 поведенческих сегментов как подграфов Нексуса, каждый с mNSM-профилем и CP.

### Разрезы сегментации (по исходному `2.6`)
PAF не фиксирует разрез — он определяется контекстом. Наиболее частые:
| Разрез | Вопрос | Пример |
|---|---|---|
| Ситуация-триггер | Что запускает потребность? | Экстренный vs плановый поиск |
| Уровень боли | Насколько остро? | Критично vs просто неудобно |
| Поведение | Как решает? | Через знакомых vs через интернет |
| Цель | Чего хочет? | Быстро найти vs найти надолго |
| Мотивация | Почему важно? | Снять тревогу vs оптимизировать |
| Ролевой контекст | Кто решает? | Родитель vs взрослый ученик |
| Опыт | Первый раз / повторно? | Новичок vs опытный |

Совет: начинать с разреза «ситуация-триггер × уровень боли» — они наиболее значимы для продуктовых решений.

### Структура профиля сегмента (по исходному `2.6`)
```
Профиль сегмента
├── Название и описание (поведенческое, не «женщины 25–40»)
├── Ситуации-триггеры
├── Начальное → Целевое состояние
├── Мотивация
├── Текущее поведение (как решает сейчас)
├── Ключевые проблемы (где болит и почему)
├── Ценности (что считают «хорошим»)
├── Уровень боли
└── Вспомогательные характеристики (демография — В КОНЦЕ, не в начале)
```

### Работslop-фильтр: правило «персона ≥4 интервью»
Главный workslop-риск Stage 1 — AI-сгенерированные «персоны» [reformation 2.9]. Фильтр: узел Сегмент принимается только если подтверждён ≥4 интервью с цитатами-доказательствами. Сегмент из 1–3 интервью → кандидат на доработку (не персона). Сегмент из «интуиции Кортекса» без интервью → отбрасывается (CP=0). Это превращает классическую Empathy Map/персону в **верифицированный подграф**.

### Пример кластеризации (по исходному `2.6`, AI-репетитор)
«Экстренный поиск» (плохая оценка/экзамен, нужно быстро, готовы платить за скорость) · «Плановый поиск» (начало года, важна долгосрочная совместимость, есть время) · «Повторный поиск» (текущий не подошёл, есть опыт, проблема передачи контекста). Каждый — отдельный подграф с уникальным триггером/путём/проблемой. Проверка: каждый реальный респондент относится к одному кластеру; «не вписывающиеся» — возможный новый сегмент или выброс.

## 🧠 Что делает человек (метапознание)
- **Чекпоинт**: Product Engineer проверяет каждый сегмент на правило «демография — в конце, не в начале» (поведение первично).
- **Veto-право**: отбраковать сегмент-«средний пользователь» (мешанина паттернов) или AI-персону без 4 интервью.
- **Метапознание**: главный workslop-риск стадии — AI-сгенерированные «персоны»; фильтр обязателен [reformation 2.9].

### Проверка через карточки интервью (по исходному `2.6`)
После авто-кластеризации Product Engineer возвращается к карточкам интервью и проверяет: соответствует ли каждый реальный человек сегменту? Нет ли «не вписывающихся»? Если человек не попал ни в один кластер — это возможный новый сегмент (провести ещё интервью) или выброс (исключить с обоснованием). Эта триангуляция (человек ↔ авто-кластер) — ключевая защита от workslop.

### Почему продукт не может быть «для всех» (по исходному `2.6`)
Разные сегменты: разные проблемы, разная готовность платить, разные каналы. Смешать их в «портрет среднего» → образ, не описывающий никого → продукт для абстрактного среднего не попадает ни в кого. Сегментация = фокус = управляемая структура вместо хаоса данных. 2–5 чётких профилей готовят почву для выбора ОДНОГО фокус-сегмента в 2.7.

## 📦 Артефакт = узлы Нексуса потребителя
- Узлы Сегмент (2–5), каждый — подграф: триггер + путь + проблема + мотивация + ценности + уровень боли.
- Метаданные: `разрез сегментации`, `список подтверждающих интервью (≥4)`, `уровень боли`, `готовность платить (mNSM)`, `CP`, `демография (вспомогательная)`.
- Context Ripeness: сегментация повышает полноту (структурирует хаос).

## 🚦 Confidence Point-гейт
- **Критерий**: каждый сегмент ≥4 интервью; разрезы сегментации осмыслены; нет AI-персон без доказательной базы.
- **Стоимость риска**: «Сколько стоит фокус на выдуманном сегменте (продукт не для кого)?»

## 🔭 Связь с Тремя Линзами (3PL)
**Product Lens** — сегменты = объекты Product Sprint. **Business Lens** — mNSM-профиль сегмента (Growth Engineer) определяет монетизируемость. Strategy Lens — узлы рынка проверяют ёмкость сегмента.

## 🛡️ Анти-workslop чекпоинт
- **Правило «персона ≥4 интервью»** — критический фильтр стадии [reformation 2.9].
- Трассировка сегмента до конкретных интервью (список в метаданных).
- Запрет на авто-генерацию персоны из одного интервью или из «интуиции» Кортекса.

## 🔄 Преемственность: классика → PAF
| Было (исходный `2.6`) | Стало (PAF) |
|---|---|
| Ручная кластеризация единиц знания | **Сегментатор-агент** (кластеризация узлов) |
| Empathy Map / CJM / Buyer Profile | **Подграф Нексуса** с поведенческими узлами |
| Персона = карточка (риск workslop) | Персона = узел **только при ≥4 интервью** |
| Разрезы сегментации вручную | Те же разрезы, но с **workslop-фильтром** и метапознание-чеком |
| Демография в начале профиля | Демография **в конце** (поведение первично) |
| Профиль = артефакт-документ | Сегмент = **спелый подграф** с mNSM-профилем |

- Skill Map **сектор 1** (Discovery) × сегментация [S3].
- Профстандарт **B/01.5** «Продуктовые исследования» [S4].
- Уровень: **средний круг** (анализ, кластеризация, проектирование).

## 📖 Пример верификации сегмента (AI-репетитор)
**Авто-кластер Сегментатора**: «Экстренный поиск» (триггер=плохая оценка/экзамен, боль=острая, поведение=сарафанное радио+площадки, мотивация=сдать вовремя).
**Workslop-фильтр**: подтверждён ли ≥4 интервью? — Да, интервью #1, #3, #5, #7, #9 с цитатами («дочь провалила контрольную, надо срочно»). → узел Сегмент принимается, CP высокий.
**Анти-workslop-сценарий**: Кортекс «предлагает» сегмент «Занятые родители, ценящие время» на основе 2 интервью и интуиции → фильтр отбрасывает (≤4 интервью, нет поведенческой уникальности) → это workslop, не персона.
**Проверка человеком**: возвращаемся к карточкам — каждый из 12 респондентов отнесён к одному из 3 кластеров; #5 «не вписывается» (ищет репетитора для себя, не для ребёнка) → отдельный кандидат-сегмент или исключение.

## ❓ Контрольные вопросы (метапознание)
1. Каждый сегмент подтверждён ≥4 интервью с цитатами? (правило workslop-фильтра)
2. Разрез сегментации — поведенческий (триггер/боль/поведение), а не демографический?
3. Нет ли сегмента-«среднего пользователя» (мешанины паттернов)?
4. Демография — в конце профиля, не в начале?
5. Каждый ли реальный респондент отнесён к сегменту? («не вписывающиеся» — новый сегмент/выброс)
6. Есть ли mNSM-профиль у каждого сегмента (Growth Engineer)?
7. 2–5 чётких профилей (а не 10 размытых)?

## ⚠️ Типичные ошибки 2.6
- **AI-персона из 1–3 интервью** → главный workslop стадии; фильтр ≥4 обязателен.
- **Демографический разрез** («женщины 25–40») → поведенческий (триггер/боль/опыт).
- **«Средний пользователь»** → мешанина; разделить на 2–3 сегмента или исключить выбросы.
- **10+ размытых сегментов** → консолидировать до 2–5 чётких.
- **«Не вписывающиеся» проигнорированы** → возможный новый сегмент, провести доп. интервью.
- **mNSM-профиля нет** → Growth Engineer обязан навесить на каждый сегмент.
