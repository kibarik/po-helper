# 5.1 instruction — BIG Idea Statement с anti-workslop

> Этап **RB-STEP-1 — BIG IDEA** (прелюдия) · Линза: **над 3PL** (синтез Видения из всех трёх линз) · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-1-IDEA/5.1 BIG Idea Statement.md` — шаблон Statement (For/Who/The/That/Unlike/We + validate by + mNSM + revenue) и Supporting Details (Problem/Solution/Market/Segment/Business Model/Advantage/Validation/Timeline/Team/Financial Ask). Эта инструкция: синтез **Vision (Видения)** [S1] базовая логика Кортексом из Узлов всех трёх Нексусов с **челенджем** (3-я оптимизация роли [S2] III.4) и **anti-workslop гейтом** на формулировки.

## 🎯 Цель в PAF-оптике

BIG Idea Statement = артикуляция **Видения (Vision)** [S1] базовая логика:
образ продукта, нужный рынку и компании, из фич (Features).

В PAF Vision — не слоган,
а **узел Нексуса продукта**, синтезируемый из доказанной части Банча (4.3)
+ `Bet` (2.1) + mNSM (1.3).

Цель — не «написать красивый Statement»,
а **сформулировать Vision, который проходит челендж**:
Кортекс аугментирует (предлагает альтернативы)
и оспаривает (где допущения без evidence?).

Это **Fit-точка** стадии ([reformation] §1.6):
BIG Idea + Bunch v2.0 + Go-Decision → Нексусы v1.0.

## 👤 Роль и ответственность

- **Product Engineer** — формулирует финальный Vision (за человеком [S2] III.4);
  валидирует, что Statement опирается на Confirmed фичи (4.3),
  а не на Needs-More.
- **Product Ops** — поддерживает Vision как Узел Нексуса продукта с версионированием.
- **Portfolio Manager** — сверяет Vision с Картой Целей портфеля
  (не отклонился ли от Ставок).
- **Зона человека:** **формулировка Vision** —
  стратегический образ, не делегируется.
  Кортекс аугментирует + челенджит [S2] III.4.

## 🤖 Что делает Кортекс-агент

**Vision-Synth Agent** (режим Scouting Agent [reformation] §1.5, синтез + челендж):

- Синтез Vision из Узлов:
  `Bunch-v2` Confirmed фич (4.3) + `Bet` (2.1) + `mNSM-numeric` (1.3)
  + `JTBD-Opportunity` (2.3) + `Monetization-Hypothesis` (2.2).
- Генерация по шаблону исходника 5.1
  (For/Who/The/That/Unlike/We + validate by + mNSM + revenue).
- **Челендж-режим:**
  для каждой части Statement Кортекс помечает допущения без evidence
  (например, «Unlike [конкурент]» — есть ли `Competitor`-узел?
   «achieve [mNSM]» — есть ли `mNSM-numeric` с NPV-ребром?).
- **Anti-workslop гейт** на формулировки:
  отсев слоганов без измеримой основы;
  отсев vanity-mNSM; отсев «Unlike» без конкурента.

- **Вход:** Узлы всех трёх Нексусов + `Bunch-v2` (4.3).
- **Выход:** `Vision-v1`-узел (Statement + Supporting Details),
  `Assumption-Audit`-узел (допущения без evidence).

**Промпт-идея:**
> «Синтезируй BIG Idea Statement из Confirmed фич Банча, Bet, mNSM, JTBD.
> Примени челендж: пометь каждое утверждение без evidence-узла.
> Anti-workslop: отсей vanity-mNSM, "Unlike" без конкурента,
> количественные обещания без NPV-ребра.
> Верни Statement + Assumption-Audit.»

## 🧠 Что делает человек (метапознание)

- **Чекпоинт:** Statement — топ-зона workslop
  («красивый слоган без измеримой основы» [S2] III.7).
  Инженер проходит Assumption-Audit: какие допущения без evidence?
- **Veto-право:** «achieve $500k ARR» без `mNSM-numeric` и NPV-ребра →
  отсев обещания или понижение CP Vision.
- **Вопрос:**
  - «Каждое утверждение Statement опирается на Confirmed фичу / Узел с источником?
    Какое допущение — самое рискованное?»
  - «Unlike [конкурент] — у нас есть Competitor-узел с источником?»

## 📦 Артефакт = узлы Нексуса

- **Создаются/обновляются:** `Vision-v1`-узел
  (Statement + Supporting Details из исходника 5.1)
  с рёбрами к `Bunch-v2`, `Bet`, `mNSM-numeric`, `JTBD-Opportunity`, `Competitor`;
  `Assumption-Audit`-узел.
- **Метаданные:** `источники` (Узлы), `CP-Vision`,
  `assumptions` (без evidence), `timestamp`, `версия`, `статус увядания`.
- **Прирост Context Ripeness:** +5%
  (Vision замыкает Видение в Нексусе продукта).

## 🚦 Confidence Point-гейт

- **Критерий перехода к 5.2:** `Vision-v1` существует;
  Assumption-Audit пройден
  (критичные допущения помечены или устранены);
  Vision опирается только на Confirmed фичи.
- **Стоимость риска:** «Какое допущение Vision —
  самое дорогое при ошибке?» — в Assumption-Audit.

## 🔭 Связь с Тремя Линзами (3PL)

- Над 3PL: Vision синтезирует все три линзы.
- **Связь:** Product Lens (Vision → Нексус продукта),
  Business Lens (mNSM/NPV в Statement),
  Strategy Lens (`Bet`/Unlike в Statement).

## 🛡️ Анти-workslop чекпоинт

- **Workslop-риск:** **Statement как слоган без evidence**;
  количественные обещания без NPV-ребра;
  «Unlike» без конкурента; vanity-mNSM.
- **Признак:** утверждение без ребра к Узлу;
  Assumption-Audit отсутствует;
  Vision опирается на Needs-More фичи.
- **Правило верификации:** Assumption-Audit обязателен;
  каждое утверждение — ребро к Узлу с источником;
  только Confirmed фичи.
