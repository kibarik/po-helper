# 3.1 instruction — Кортекс-агенты рыночных сил: Porter/PESTEL → Force-узлы

> Этап **RB-STEP-3 — MARKET RESEARCH** (Stage 2) · Линза: Strategy (контекст: Product/Business) · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-3-MARKET-RESEARCH/3.1 Анализ рыночной среды.md` (Porter's Five Forces + PESTEL + порог входа).

## 🎯 Цель в PAF-оптике
Не «заполнить матрицу Портера вручную», а насытить Нексус рынка **Force-узлами** (по 5 силам Портера и 6 PESTEL-категориям) через Кортекс-агент рыночных сил, с event-based refresh и пересчётом CP. Силы рынка — это аргументы насыщения Банча (Understand-фаза Und-Id-Ex) и вход для Bet-узлов стратегической линзы [S1] VI.1.

## 👤 Роль и ответственность
- **Product Engineer** — формулирует фокус-вопрос («на каком рынке/нише работаю?»), ставит задачу агенту рыночных сил, валидирует Force-узлы, решает, какие 1–2 силы наиболее значимы для его ситуации (исходный `3.1`: «сосредоточься на 1–2 самых значимых силах»).
- **Product Ops** — настраивает RAG по отраслевым отчётам (McKinsey, BCG, Gartner, Statista, Росстат), триггеры event-based refresh, авто-расчёт CP по Force-узлам.
- **Portfolio Manager** — потребитель Force-узлов при оценке привлекательности рынка для портфеля (входит в Ставки).
- **Зона человека**: интерпретация противоречивых сигналов сил (исходный `3.1` Troubleshooting: «сосредоточься на напряжениях — они станут основой для стратегических решений»).

## 🤖 Что делает Кортекс-агент
**Агент рыночных сил (Market Forces Agent)** заменяет ручной Porter + PESTEL:
- **Скаутинг источников**: отраслевые отчёты, годовые отчёты публичных конкурентов (раздел «Risks»), статистические базы, новостные агрегаторы — по перечню из исходного `3.1` Phase 0.
- **Генерация Force-узлов** — по 5 силам Портера:
  1. Угроза новых конкурентов (барьеры: лицензии, капзатраты, сетевые эффекты, switching costs, патенты);
  2. Рыночная власть покупателей (альтернативы, switching cost, ценовая чувствительность, концентрация);
  3. Рыночная власть поставщиков (концентрация, switching cost, риск вертикальной интеграции — критично для AI-продуктов: зависимость от LLM-API);
  4. Угроза заменителей (альтернативные способы удовлетворения потребности);
  5. Интенсивность конкуренции (количество игроков, темп роста, дифференциация, ценовые войны).
- **Генерация Force-узлов** — по 6 PESTEL-категориям (Political, Economic, Social, Technological, Environmental, Legal) с net-эффектом и приоритетом.
- **Оценка порога входа** (Barrier to Entry) по 7 барьерам (регуляторные, капитальные, технологические, каналы сбыта, сетевые эффекты, бренд, ресурсы) — шкала 1–5.
- **event-based refresh**: мониторинг источников; при появлении новых данных — обновление Force-узла, пересчёт CP, аларм при смене уровня угрозы.
- **CP-пересчёт**: каждый Force-узел получает CP; рост CP при появлении верифицированных данных, падение при устаревании (увядание).

## 🧠 Что делает человек (метапознание)
- **Veto на галлюцинированную силу**: если агент выдал «Высокая угроза новых конкурентов» без URL+даты обоснования → Узел отклоняется, CP=0.
- Различает **факт** (конкретная цифра из отчёта с источником) vs **гипотеза** (оценочное суждение) vs **бенчмарк** (аналогия по другому рынку) — маркировка обязательна.
- Для AI-продуктов: критически оценивает **власть поставщиков** (LLM-API) — стресс-тест «провайдер поднял цены ×3 или закрыл доступ» (исходный `3.1` Step 1.3).
- Решает, какие силы наиболее значимы — агент даёт все, человек фокусирует стратегию на 1–2.

## 📦 Артефакт = Force-узлы Нексуса рынка
Каждый Force-узел:
```
type: Force (porter_{1..5} | pestel_{P,E,S,T,E,L})
title: <название силы/фактора>
level: Высокая | Средняя | Низкая
justification: <обоснование из источника>
source: {URL, дата, издание}   ← ОБЯЗАТЕЛЬНО
cp: <Confidence Point>
timestamp: <дата создания/обновления>
decay_status: созревание | увядание
impact_on_product: <вывод для решений>
```
Итог: сводный Force-подграф с общим выводом о привлекательности рынка + оценка порога входа (Низкий/Средний/Высокий).

### Шаблон Force-узла (Porter) с примером
```
type: Force (porter_1)
title: Угроза новых конкурентов
level: Высокая
justification:
  - Регуляторные барьеры: минимальные (лицензия только при выдаче документов об образовании);
  - Капиталовложения: низкие (MVP за 2–3 мес, команда из 3 чел.);
  - Сетевые эффекты лидеров: умеренные;
  - Switching costs клиентов: низкие;
  - Патенты: отсутствуют.
source: [{url: <отчёт>, date: 2024, publisher: <издание>}, ...]
cp: 6
impact_on_product: Низкий порог входа → дифференциация через качество подбора и скорость.
```

### PESTEL — 6 категорий макросреды (исходный `3.1` Phase 2)
| Буква | Категория | Примеры факторов |
|---|---|---|
| **P** | Political | Господдержка отрасли, санкции, налогообложение |
| **E** | Economic | ВВП, инфляция, доступность инвестиций, доходы ЦА |
| **S** | Social | Демография, поведение, ценности, цифровая грамотность |
| **T** | Technological | AI-революция, снижение себестоимости, устаревание |
| **E** | Environmental | ESG-требования, климатические риски (для digital — минимально) |
| **L** | Legal | ФЗ-152/GDPR, лицензирование, ограничения AI |

Каждый PESTEL-фактор → Force-узел с `net-эффект (+/−/нейтр)` и `приоритетом`. Топ-3 возможности и топ-3 угрозы из PESTEL (исходный `3.1` Step 2.7).

### Оценка порога входа (Barrier to Entry) — 7 барьеров
Шкала 1–5 по каждому: регуляторные, капитальные затраты, технологическая сложность, доступ к каналам сбыта, сетевые эффекты лидеров, узнаваемость бренда лидеров, доступ к ресурсам (кадры/данные). Средний балл → итог: Низкий/Средний/Высокий порог входа + стратегия преодоления по каждому барьеру.

## 🚦 Confidence Point-гейт
Force-подграф спел для скаутинга, когда:
- CP каждого Force-узла ≥ порога (узлы без источника → CP=0, не принимаются);
- покрыты все 5 сил Портера + хотя бы 2–3 фактора на каждую PESTEL-категорию (исходный `3.1` Success Criteria);
- портфель Force-узлов даёт явный net-вывод о привлекательности рынка.

CP-gate Strategy Lens (см. `3.6`): силы учитываются в стоимости риска Ставки.

## 🔭 Связь с Тремя Линзами (3PL)
Force-узлы — объект **Strategy Lens** (привлекательность рынка для портфеля). Связь:
- **Product Lens**: угроза заменителей и власть покупателей → аргументы Банча фич (дифференциация).
- **Business Lens**: власть поставщиков (LLM-API) → Рычаг COGS токенов; порог входа → оценка ресурсов для старта.

## 🛡️ Анти-workslop чекпоинт
- **Главная ловушка**: ИИ генерирует «правдоподобные» уровни сил без источника. Каждый Force-узел — URL+дата.
- PESTEL без привязки к продукту = workslop. Правило исходного `3.1` Troubleshooting: «не пиши "экономика нестабильна" — пиши "инфляция 8% снижает покупательную способность ЦА на ~15%"».
- Силы, противоречащие друг другу, — фиксируются как «напряжения» (явные, не замалчиваются).
- Porter's Five Forces + PESTEL — **дополняют** друг друга (Porter = структура отрасли, PESTEL = макросреда); запрет на дублирование.

### Стресс-тест «власть поставщиков» для AI-продуктов (исходный `3.1` Step 1.3)
Критический Force-узел для AI-продукта — власть поставщика LLM-API. Обязательный стресс-тест: «если провайдер (OpenAI/Anthropic) поднимет цены ×3 или закроет доступ — что произойдёт с продуктом?». Этот Force-узел связан с Рычагом **COGS токенов** (RB-STEP-5 `5.4`) и напрямую влияет на Bet-узлы Strategy Lens. Без этого стресс-теста NPV-потенциал SOM (`3.3`) может оказаться иллюзорным (COGS-эксплозия при масштабировании).

### Риски трансформации 3.1 (reformation §3.9)
- **Галлюцинации уровней сил**: ИИ выдаёт «Высокая угроза» без источника. Снятие: URL+дата на каждый Force-узел; CP=0 без источника.
- **PESTEL как workslop**: общие фразы без привязки к продукту. Снятие: каждый фактор → вывод «поэтому мы должны...».
- **Устаревание данных** (CR-увядание Force-узлов): без event-based refresh матрица Портера/PESTEL устаревает. Снятие: периодический refresh + аларм при падении CR.

### Success Criteria Force-подграфа (исходный `3.1`)
- Заполнены все 5 сил Porter's Five Forces с обоснованием и уровнем угрозы;
- PESTEL-матрица: ≥2–3 фактора на каждую из 6 категорий;
- Сводная PESTEL с топ-3 возможностями и топ-3 угрозами;
- Порог входа оценён по 7 барьерам; CR после шага ~25–35% (хорошая база для трендов и объёма).
