# 3.3 instruction — Агент-оценщик TAM/SAM/SOM + mNSM + верификатор цифр

> Этап **RB-STEP-3 — MARKET RESEARCH** (Stage 2) · Линза: Strategy (контекст: Product/Business) · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-3-MARKET-RESEARCH/3.3 Оценка объёма рынка (TAM SAM SOM).md` (top-down/bottom-up, правило ФРИИ 10%, драйвер роста).
> ⚠️ **Критический анти-workslop файл всей стадии**: TAM-галлюцинации — главный риск библиотеки (reformation §3.9).

## 🎯 Цель в PAF-оптике
Насытить Нексус рынка **TAM-SAM-SOM-узлом** через агента-оценщика с мульти-метод расчётом, автосверкой методов и переводом SOM в **потенциал mNSM/NPV** (Принцип 6). Драйвер роста → **Рычаг (Lever)** модели роста. go/no-go по объёму заменяется на **CP-gate** с условием `mNSM-потенциал SOM > 0`. Это Understand-фаза Und-Id-Ex: рыночные цифры — аргумент насыщения Банча.

## 👤 Роль и ответственность
- **Product Engineer** — фиксирует предпосылки (аксиомы расчёта: география, период, целевой потребитель, бизнес-модель, средний чек, допущения), ставит задачу агенту-оценщику, валидирует цифры, решает go/no-go через CP-gate.
- **Product Ops** — настраивает RAG по статистическим базам (Росстат, Statista, Gartner, годовые отчёты конкурентов, Crunchbase), связывает TAM-SAM-SOM-узел с mNSM/NPV-калькулятором Нексуса роста.
- **Portfolio Manager** — потребитель оценки для Ставок (NPV портфеля); оценивает, достаточен ли SAM для венчурного/корпоративного сценария.
- **Зона человека**: проверка допущений и явная маркировка «данные/допущение»; veto на галлюцинированную цифру. NPV-потенциал — стратегическая зона, не делегируется [S2] III.4.

## 🤖 Что делает Кортекс-агент
**Агент-оценщик TAM/SAM/SOM (Market Sizing Agent)** + **Агент-верификатор рыночных цифр (Number Verifier)**:

**Агент-оценщик** ведёт мульти-метод расчёт по исходному `3.3`:
- **TAM** двумя методами с автосверкой:
  - **Top-Down** — от общего рынка → фильтр по географии → по сегменту → по области потребностей;
  - **Bottom-Up** — `TAM = потенциальные клиенты × частота × средний чек`, с цепочкой фильтров сегмента.
  - Сравнение методов; расхождение >2–3× → флаг, проверка допущения.
- **SAM** — фильтрация TAM по типам конкуренции (прямые/косвенные/«не решают») + доля захвата (правило **ФРИИ: 10%** от конкурентов; 1–5% от косвенных).
- **SOM** — от SAM с учётом ресурсных ограничений; **драйвер роста** (по Аркадию Морейнису: измеримая величина, влияющая на измеримый результат) → это **Рычаг (Lever)** модели роста.
- **5 принципов PAF** (исходный `3.3`): деньги=функция от покупателей; оценка зависит от бизнес-модели; оценка на конкретный период; явные допущения; равномерное распределение — не всегда верно.
- **mNSM/NPV-потенциал SOM** (Принцип 6): SOM → потенциал монетизируемой ценности → mNSM-гипотеза → NPV-оценка. Для AI-продуктов — учёт **COGS токенов** в себестоимости.
- **Автосверка**: сравнение TAM top-down vs bottom-up; сравнение SAM «сверху-вниз» vs «от конкурентов» (см. `3.5`); расхождение >2× → аларм.

### TAM — два метода детально (исходный `3.3` Phase 2)
**Top-Down** (сужение «пирога»): `общий рынок индустрии × % географии × % сегмента × % области потребностей = TAM`. Каждый шаг — источник или явное допущение.
**Bottom-Up** (от конкретных людей): `потенциальные клиенты × частота покупки × средний чек = TAM`. Цепочка фильтров сегмента (демография → гео → возраст → поведение).
Правило PAF: «советуем использовать оба метода, сравнить результаты». Разница до 2–3× допустима; берётся консервативная оценка, строятся 3 сценария (pessimistic/base/optimistic).

### SAM — правило ФРИИ 10% (исходный `3.3` Phase 3)
«Эксперты акселератора ФРИИ советуют ориентироваться на 10%, как потенциально возможную долю рынка, которую можно отжать у конкурентов». Алгоритм: TAM → распределить по типам конкуренции → доля захвата (прямые 10%, косвенные 1–5%, новый рынок 3–10%) → + новые клиенты → SAM.

### SOM — драйвер роста = Рычаг (исходный `3.3` Phase 4)
«Вы всегда ограничены возможностями по обеспечению сделок». Драйвер роста (Морейнис) = измеримая величина, влияющая на измеримый результат. Для маркетплейса — сторона поставщиков; для SaaS — скорость привлечения; для AI-продукта — COGS токенов. В PAF драйвер → **Рычаг (Lever)**, влияющий на NPV модели роста (связь с RB-STEP-5).

### Стресс-тест и 3 сценария (исходный `3.3` Phase 5, Troubleshooting)
- **Стресс-тест**: пересчёт SOM ×0,5 — что получится? Правило: если SOM > 10% от SAM — скорее всего переоценён.
- **3 сценария**: pessimistic / base / optimistic. На этом этапе достаточно base; 3 сценария — при высокой неопределённости в ключевом допущении, меняющем вывод go/no-go.
- **Точность ±50% — норма**: цель не точный ответ, а проверка порядка величины ($10 млн vs $100 млн vs $1 млрд — разные стратегии). Ошибка на 50% внутри порядка нормальна; ошибка на 10× — проблема (исходный `3.3` FAQ).

### Привязка к области потребностей (исходный `3.3` Phase 6)
«Оценка должна проводиться не в целом по профилю, а в контексте конкретной области потребностей». Профиль потребителя (Stage 2) задаёт сегмент; оценка — только для потребности, которую решает продукт (напр. «найти репетитора», а не «рынок образования в целом»). Без этой привязки TAM-SAM-SOM-узел = абстракция без операционной пользы.

**Агент-верификатор рыночных цифр (критический анти-workslop)**:
- Каждая рыночная цифра обязана иметь **URL+дату**. Цифра без источника → CP=0, флаг «галюцинация», в расчёт не принимается.
- Маркировка **факт** (источник верифицирован) / **гипотеза** (допущение, явно) / **бенчмарк** (аналогия по другому рынку).
- Стресс-тест: пересчёт SOM ×0,5 (пессимистичный сценарий); правило «если SOM > 10% от SAM — скорее всего переоценён» (исходный `3.3` Troubleshooting).
- Детектор «иллюзии точности»: «SAM = 487 356 000 ₽» без источника = workslop; честная запись — «SAM ≈ 500 млн ₽ ± 50%, уверенность низкая» (исходный `3.6` FAQ).

## 🧠 Что делает человек (метапознание)
- **Veto на галлюцинированную цифру** — главный навык этого подшага. PAF/исходный `3.6`: «запись "SAM = 487 356 000 ₽" без источника — это иллюзия точности».
- Проверяет каждое допущение: «реалистичный вариант или оптимистичный?» (исходный `3.3` Troubleshooting).
- Различает **факт/гипотеза/бенчмарк** — особенно для «данных нет, берём аналог с другого рынка».
- Оценивает, достаточен ли SOM для точки безубыточности и NPV>0 — стратегическое решение.
- Для AI-продуктов: критически оценивает COGS токенов в себестоимости SOM (риск COGS-эксплозии).

## 🚦 Confidence Point-гейт
TAM-SAM-SOM-узел проходит CP-gate, когда:
- `{CP рыночной гипотезы ≥ порога} ∧ {стоимость риска приемлема} ∧ {mNSM-потенциал SOM > 0}`;
- каждая рыночная цифра имеет URL+дату (иначе CP=0);
- расхождение методов TAM < 2× (или обосновано);
- SOM не превышает 10% от SAM (иначе флаг переоценки).
Отказ (NO-GO) — осознанный gate, не провал (исходный `3.6`: «сэкономил месяцы разработки»).

## 🔭 Связь с Тремя Линзами (3PL)
- TAM-SAM-SOM-узел — объект **Strategy Lens** (привлекательность рынка для Ставок).
- **Business Lens**: SAM/SOM → Рычаги модели роста → NPV; для AI — COGS токенов как отдельный рычаг.
- **Product Lens**: SOM → потенциал mNSM → аргумент Банча фич.

## 🛡️ Анти-workslop чекпоинт
- **TAM-галюцинации — критический риск всей библиотеки** (reformation §3.9). Каждая рыночная цифра: URL+дата. Без источника → CP=0.
- Запрет на «рынок в целом»: оценка только в контексте области потребностей (исходный `3.3` Phase 6, исходный `3.6`).
- Явная маркировка данных/допущений; стресс-тест SOM ×0,5.
- Иллюзия точности = workslop; честная погрешность ±50% — норма на этом этапе (исходный `3.3` FAQ).
- Для AI-продуктов: COGS токенов не прятать в «серверы» — отдельный рычаг (связь с RB-STEP-5 `5.4`).

### Риски трансформации 3.3 (reformation §3.9)
- **Галлюцинации рыночных цифр** — главный риск: RAG только по верифицированным источникам; CP=0 без источника; маркировка факт/гипотеза/бенчмарк.
- **Псевдо-NPV без данных**: NPV-потенциал SOM на основе допущений без валидации → иллюзия. Снятие: явные допущения для верификации в Customer Validation.
- **Метапознание оператора**: доверие AI-расчёту TAM/SAM/SOM без проверки = ложная оценка. Снятие: veto на цифру без источника.
- **COGS-эксплозия AI-фич**: спрятанные токен-затраты в «серверах» → ложный SOM/breakeven. Снятие: AI-COGS как отдельный рычаг, стресс-тест usage ×2/×5.

### Success Criteria TAM-SAM-SOM-узла (исходный `3.3`)
- Зафиксированы предпосылки расчёта (аксиомы: география, период, потребитель, бизнес-модель);
- TAM рассчитан ≥ одним методом (top-down или bottom-up), желательно обоими со сверкой;
- SAM с учётом конкурентов и доли захвата (правило ФРИИ 10%); SOM с учётом ресурсных ограничений и драйвера роста;
- Все расчёты содержат источники или явные допущения; грубая оценка рентабельности; оценка в контексте области потребностей; CR ~55–65%.
