# 3.5 instruction — Авто-скоринг как скаутинг (Принцип 5)

> Этап **RB-STEP-3 — MARKET RESEARCH** (Stage 2) · Линза: Strategy (контекст: Product/Business) · Инструкция для **Инженера Продукта нового поколения** (PAF, [S1],[S2]).
> Преемственность метода: исходный раннбук `RB-STEP-3-MARKET-RESEARCH/3.5 Скоринг конкурентов.md` (взвешенная матрица 1–5, 7 критериев, правило ФРИИ 10%, SAM «от конкурентов»).

## 🎯 Цель в PAF-оптике
Заменить ручную взвешенную матрицу 1–5 на **авто-скоринг Competitor-узлов** как реализацию **Принципа 5 (скаутинг вместо приоритизации)** [S1]. Веса критериев = **Рычаги (Lever)** модели; «ближайшие» конкуренты (топ-5–7) → аргументы Identify-фазы Und-Id-Ex («почему клиент выберет нас»). Дополнительный выход: уточнение **SAM «от конкурентов»** (третий метод оценки рынка) с автосверкой против SAM из `3.3`.

## 👤 Роль и ответственность
- **Product Engineer** — определяет **контекст конкуренции** («за какого клиента, в каком канале, в какой момент решения конкурирую?»), настраивает веса критериев (= Рычаги), валидирует топ «ближайших», формулирует «почему выберут нас» для каждого из топ-5.
- **Product Ops** — настраивает авто-скоринг по Competitor-узлам (веса → взвешенный балл → ранжирование), event-based пересчёт при обновлении Competitor-узлов, авто-расчёт SAM «от конкурентов» и автосверку с SAM `3.3`.
- **Portfolio Manager** — потребитель топа «ближайших» для оценки каннибализации и Ставок портфеля.
- **Зона человека**: настройка весов (= стратегический выбор Рычагов); позиционирование vs. топ-5 (стратегическая зона [S2] III.4).

## 🤖 Что делает Кортекс-агент
**Агент-скаутинг (Scouting Agent)** + **Агент-скоринг (Scoring Agent)**:

**Авто-скоринг** по 7 критериям (веса из исходного `3.5`):
| Критерий | Вес | Что оценивает |
|---|---|---|
| Совпадение сегмента | 25% | Целевой клиент совпадает? |
| Совпадение задачи | 20% | Ту же задачу решают? |
| Совпадение ценового сегмента | 15% | Один диапазон? |
| Совпадение канала привлечения | 15% | За того же клиента в том же канале? |
| Размер аудитории | 10% | Угроза давления ресурсами |
| Темп роста | 10% | Угроза вытеснения |
| Качество продукта | 5% | Сила по отзывам |

Шкала 1–5; итоговый балл = взвешенная сумма. **Веса = Рычаги**: критерии «сегмент» и «задача» (45%) — главные Рычаги, потому что «близость» определяется борьбой за одного клиента.

**Выход скаутинга**:
- Топ «ближайших» (балл ≥ 3,5) — обычно 3–7 компаний; граница «ближайших/остальных» по разрыву в баллах.
- **SAM «от конкурентов»**: `Σ аудиторий ближайших × доля захвата` (правило **ФРИИ: 10%** от аудитории; 5% консервативно, 15% оптимистично) + прирост от новых клиентов.
- **Автосверка**: SAM «от конкурентов» vs SAM «сверху-вниз» из `3.3`; расхождение >2× → аларм, проверка допущений (исходный `3.5` FAQ).

**Перевод в аргументы Identify (Und-Id-Ex)**: для каждого из топ-5 — карточка позиционирования:
- их главная сила / главная слабость (из teardown `3.4`);
- «почему клиент выберет нас»;
- риск ответного хода + наш ответ.
→ топ-3 конкурентные угрозы (вероятность / горизонт / ответ).

## 🧠 Что делает человек (метапознание)
- **Настройка весов = стратегический выбор** Рычагов (не делегируется). Если канал критичен (конкурируем за один SEO-ключевик) — вес «канала» поднимается до 20–25%.
- Проверка: результирующий топ соответствует интуиции? Нет → пересмотр весов.
- **Контекст конкуренции** — за какого клиента боремся (исходный `3.5` Phase 0: «конкурент, занимающий половину корпоративного рынка, может не быть твоим, если ты работаешь с SMB»).
- Формулировка «почему выберут нас» — стратегическое позиционирование; если не получается — возможно, нужно вернуться к сегменту (исходный `3.5` Troubleshooting).

## 📦 Артефакт = обновлённые Competitor-узлы + ранжированный список
Каждый Competitor-узел дополняется:
```
scoring: {segment, task, price, channel, size, growth, quality, weighted_total}
nearest: true | false   (по порогу ≥ 3,5)
positioning: {their_strength, their_weakness, why_us, counter_risk}
```
Плюс агрегаты:
- ранжированный топ «ближайших» с обоснованием;
- SAM «от конкурентов» (метод, базовый/консервативный/оптимистичный сценарий, сравнение с SAM `3.3`, расхождение %);
- топ-3 конкурентные угрозы с ответными мерами.

### SAM «от конкурентов» — третий метод оценки (исходный `3.5` Phase 3)
Формула: `SAM = Σ аудиторий ближайших конкурентов × доля захвата`, где доля = 5% (консервативно) / 10% (базовый, правило ФРИИ) / 15% (оптимистично) + прирост от новых клиентов. Данные об аудитории: пресс-релизы, SimilarWeb (трафик × конверсия 5–15%), Crunchbase/LinkedIn, App Store (загрузки). Нет данных → честная маркировка «допущение».
**Автосверка**: SAM «от конкурентов» vs SAM «сверху-вниз» из `3.3`. Расхождение >2× → аларм, проверка допущений (исходный `3.5` FAQ: либо TAM ошибочен, либо конкуренты плохо охватывают рынок).

### Веса критериев = Рычаги (PAF-трансформация исходного `3.5`)
В исходном раннбуке веса (25/20/15/15/10/10/5) — фиксированный набор. В PAF веса — это **Рычаги (Lever)**: выбор веса = стратегический выбор того, что определяет близость конкурента. Если канал критичен (конкурируем за один SEO-ключевик) — вес «канала» поднимается. Инженер настраивает веса как Рычаги; Product Ops пересчитывает баллы при изменении Competitor-узлов.

### Позиционирование vs. топ-5 (исходный `3.5` Phase 4) — вход в RB-STEP-4
Для каждого из топ-5 ближайших: их сила / слабость / «почему клиент выберет нас» / риск ответного хода / наш ответ. Это **Identify-фаза** Und-Id-Ex: аргументы для ценностного предложения (переход в RB-STEP-4 `4.1`). Топ-3 конкурентные угрозы с вероятностью, горизонтом и ответными мерами.

### Принцип 5 — почему скоринг = скаутинг, а не приоритизация [S1]
В классическом `3.5` скоринг — выбор «с кем бороться в первую очередь» (приоритизация списка конкурентов). В PAF это **скаутинг** (Принцип 5): поиск возможностей и угроз на карте целей, чтобы поднять контекст Нексуса до идеи с максимальным CP. Скаутинг не выбирает «элемент из списка» — он насыщает Банч аргументами позиционирования. Запрещённый синоним (`naming_conventions`): «приоритизация задач» → «скаутинг возможностей/угроз».

### Шкала оценки 1–5 (исходный `3.5` Step 1.2)
| Балл | Значение |
|---|---|
| 5 | Полное совпадение / максимальная угроза |
| 4 | Высокое совпадение / значительная угроза |
| 3 | Частичное совпадение / умеренная угроза |
| 2 | Слабое совпадение / низкая угроза |
| 1 | Почти нет совпадения / минимальная угроза |

Граница «ближайших» — по разрыву в баллах (обычно ≥ 3,5); топ «ближайших» = 3–7 компаний. Если список >10 — поднять порог до 4,0 или добавить фильтр «мог бы клиент выбрать их вместо нас напрямую?» (исходный `3.5` Troubleshooting).

## 🚦 Confidence Point-гейт
Скаутинг-подграф спел, когда:
- CP Competitor-узлов с баллами ≥ порога (данные с источником);
- топ «ближайших» (3–7) выделен и обоснован;
- SAM «от конкурентов» рассчитан и сопоставлен с SAM `3.3` (расхождение <2× или обосновано);
- для каждого из топ-5 сформулировано «почему выберут нас»;
- CR конкурентной части ≥ 70% (исходный `3.5` Success Criteria).

## 🔭 Связь с Тремя Линзами (3PL)
- Скаутинг «ближайших» — объект **Strategy Lens** (конкурентная позиция для Ставок).
- **Product Lens**: «почему выберут нас» → аргумент Identify Банча фич (позиционирование → ценностное предложение, переход в RB-STEP-4).
- **Business Lens**: SAM «от конкурентов» → Рычаг модели роста; Channel-совпадение → CAC.

## 🛡️ Анти-workslop чекпоинт
- **Скаутинг ≠ приоритизация списка задач** (Принцип 5, запрещённый синоним `naming_conventions`). Это поиск возможностей/угроз на карте целей, не выбор элемента из бэклога.
- Веса без обоснования = workslop; каждый вес связан с Рычагом.
- Все конкуренты получили одинаковые баллы → добавить критерий (модель монетизации) или уточнить подкритерии (исходный `3.5` Troubleshooting).
- SAM «от конкурентов» без данных об аудитории → честная маркировка «допущение», консервативная оценка (исходный `3.5` FAQ).

### Риски трансформации 3.5 (reformation §3.9)
- **Скаутинг подменяется приоритизацией** (запрещённый синоним): возврат к «выбору элемента из списка». Снятие: явный Принцип 5; веса = Рычаги, не ранжирование задач.
- **Все конкуренты с одинаковыми баллами**: не выделить «ближайших». Снятие: уточнение критериев/подкритериев, добавление модели монетизации.
- **SAM «от конкурентов» без данных**: галлюцинированная аудитория. Снятие: честная маркировка «допущение», консервативная оценка; автосверка с SAM `3.3`.
