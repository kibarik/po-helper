**Синтез проблемы и механизма решения:**
Необходимо создать унифицированный процесс обработки БФТ, где результаты быстрой генерации и глубокой аналитики сосуществуют в одном контексте. В качестве решения используется автоматическое создание GitHub Issue, которое служит контейнером. Оркестратор `gh-issue-agents` через GitHub App координирует работу агентов: результат синхронного вызова `/bft-fast` (быстрый черновик) и вывод фонового процесса `bft-writer` (глубокая проработка) записываются в комментарии к этому Issue. Это позволяет получить единый артефакт, содержащий все стадии эволюции мысли без ручного объединения.

**Требования (агрегированы):**

*   Обеспечить оркестрацию агентов и агрегацию их вывода в едином GitHub Issue, где БФТ «живёт» как Issue, который автоматически открывается при отправке мыслей на генерацию; к Issue должен прикрепляться результат `/bft-fast`, а в фоне должен стартовать тяжёлый многошаговый `bft-writer`, при этом `gh-issue-agents` выступает оркестратором фоновых субагентов, привязанных к Issue, реализуя модель «один Issue = fast-draft (сразу) + deep-БФТ (позже)» без ручного связывания. — from #125 (Оркестрация агентов и агрегация их вывода в едином GitHub Issue).
