### Синтез проблемы и решения

Цель инкремента — развернуть и исправить инструменты для интерактивного и визуального планирования (Excalidraw, Sprint Planner), а также интегрировать их с процессом ревью. Основные проблемы заключаются в том, что установка проекта не обеспечивает обнаружение команд планирования агентом, отсутствует реализация навыка визуальной доски для ИИ-декомпозиции, а процесс ревью не гарантирует полноту разбора обратной связи и приёмки. Решение включает исправление скриптов установки для видимостиslash-команд, реализацию навыка `excalidraw-board` с MCP-сервером и шаблонами рендера, внедрение новой стадии визуального планирования (от декомпозиции до консенсуса) перед созданием задач в JIRA и добавление обязательного шага BFT-ревью с проверкой всех комментариев и подтверждением приёмки PO.

### Требования (агрегированы)

*   **Обеспечить обнаружение команд планирования в целевом проекте** — Гарантировать, что BUILD-команды (`/sprint-build`, `/sprint-activate`) докатываются установкой через `install.sh` (`.claude/commands/`) и обнаруживаются Claude Code агентом, чтобы PO мог их вызвать после `/sprint-deliver`. — from #47
*   **Реализовать навык excalidraw-board для визуализации** — Добавить блок `excalidraw` в `.mcp.`, настроить оркестрацию в `.claude/skills/excalidraw-board/SKILL.md` (команды `open`, `fill`, `sync`, `save`, `list`, `present`) и добавить рецепты рендера (`sprint-table`, `okr-tree`) для мгновенного получения предзаполненной доски. — from #51
*   **Внедрить стадию визуального планирования с гейтом консенсуса** — Создать новый этап между `sprint-load` и `sprint-build`: ИИ-декомпозиция «как я вижу» → ревью командой → правки → материализация в JIRA только после явного согласия команды. — from #83
*   **Обеспечить полноценность BFT-ревью** — Реализовать обязательный разбор всех Inline-комментариев и комментариев под постом в Confluence, а также верификацию факта приёмки и проверки каждого артефакта со стороны PO. — from #89
