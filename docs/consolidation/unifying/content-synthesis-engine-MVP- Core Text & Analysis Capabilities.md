### Синтез проблемы и механизма решения

Для эффективной работы необходимы фундаментальные AI-навыки, позволяющие трансформировать сложные технические данные и разрозненную информацию в структурированные, понятные бизнес-решения. Существующие проблемы включают: сложность перевода технических деталей в бизнес-эффекты, отсутствие контекста при старте работы с Issue, неструктурированный процесс принятия стратегических решений и избыточную сложность генерируемых ИИ текстов.

В рамках инкремента `content-synthesis-engine:MVP` требуется создать набор независимых строительных блоков (building blocks). Механизм решения объединяет четыре направления:
1.  **Автоматизация контекста кода**: Интеграция Qodo для первичного анализа и планирования решений в репозитории.
2.  **Семантический перевод**: Преобразование «сырых» технических сигналов в структурированные бизнес-карточки с понятными последствиями.
3.  **Поддержка принятия решений**: Использование теории принятия решений и инструментов (SWOT, деревья вариантов) для структурирования стратегических развилок.
4.  **Текстовая постобработка**: Сквозной слой редактирования для упрощения и удаления шума из любого вывода ИИ.

*   Разработать навык-переводчик (например, `/tech-brief`), который принимает сырое техобращение, распознает источник и намерение, собирает детальный контекст (`po-research`), определяет ответственных через People Graph и выдает простую бизнес-карточку с явным бизнес-эффектом и ключевыми действиями.
