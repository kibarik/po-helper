# 🎓 Customer Validation — Практические примеры

## Пример 1: Полный цикл для B2B SaaS продукта

### Контекст:
Представим, что мы создаем продукт для автоматизации распределения задач между членами команды проект-менеджеров в ритейле.

---

## 🔹 Часть 1: Формирование гипотез

### Big Idea:
"Проект-менеджерам в ритейле сложно быстро распределять задачи между командой, потому что текущие инструменты медленные и не учитывают загрузку каждого сотрудника. Мы создадим продукт, который автоматически распределяет задачи с учетом загрузки, навыков и приоритетов."

### Гипотезы (20 штук):

#### Гипотезы поведения (5 штук):

**H-001 (Поведение):**
```
Когда [проект-менеджер в ритейле],
[появляется новый проект или изменение приоритетов],
то [он вручную перераспределяет задачи между командой в Excel или текущей PM-системе],
потому что [автоматические инструменты "не понимают" контекст реальной работы и часто делают ошибки].
```

**H-002 (Поведение):**
```
Когда [проект-менеджер в ритейле],
[нужно срочно перераспределить задачи из-за болезни сотрудника],
то [он звонит или пишет каждому члену команды лично],
потому что [это быстрее, чем перестраивать задачи в системе и ждать, пока сотрудники увидят изменения].
```

**H-003 (Поведение):**
```
Когда [проект-менеджер в ритейле],
[конец месяца и нужно закрыть все отчетные задачи],
то [он работает до 22:00, manually проверяя каждый статус],
потому что [не доверяет автоматическим статусам и боится, что важная задача будет пропущена].
```

**H-004 (Поведение):**
```
Когда [проект-менеджер в ритейле],
[проводит планирование на следующую неделю],
то [он использует белую доску или sticky notes],
потому что [это позволяет видеть всю картину целиком и легко менять приоритеты].
```

**H-005 (Поведение):**
```
Когда [проект-менеджер в ритейле],
[нужно объяснить загрузку команды руководству],
то [он создает вручную Excel-отрет с графиками],
потому что [текущие инструменты не имеют подходящих визуализаций для non-technical stakeholders].
```

#### Гипотезы проблем (5 штук):

**H-006 (Проблема):**
```
Когда [проект-менеджер в ритейле],
[распределяет задачи вручную],
то [он тратит 1-2 часа ежедневно на перераспределение],
а это [мешает ему заниматься стратегическими задачами и коммуникацией с stakeholders].
```

**H-007 (Проблема):**
```

[использует автоматические инструменты для распределения],
то [система распределяет задачи без учета реальной загрузки и навыков],

```

**H-008 (Проблема):**
```

**H-009 (Проблема):**
```

**H-010 (Проблема):**

## 🔹 Часть 3: Скрипт интервью (адаптированный)

### Introduction:
```
"Здравствуйте! Меня зовут Алекс, я работаю над инструментом для управления проектами. 

Мы проводим исследование, чтобы лучше понять, как проект-менеджеры распределяют задачи между командами и с какими проблемами сталкиваются.

Сегодняшний разговор займет около 45-60 минут. Нет правильных или неправильных ответов — мне интересен именно ваш опыт и мнение.

Если вам комфортно, я буду делать заметки. Все рассказанное останется конфиденциальным.

У вас есть вопросы?"
```

### Warm-up:
```
1. Расскажите немного о себе:
   - Как долго вы работаете проект-менеджером?
   - В какой индустрии/компании работаете?
   - Сколько человек в вашей команде?
   
2. Расскажите о последнем раз, когда вам нужно было распределить задачи между командой:
   - Когда это было?
   - Какой была ситуация?
   - Как вы решали эту задачу?
```

### Core Questions:

#### Block 1: Текущее поведение
```
1. Как вы сейчас распределяете задачи между командой?
   - Расскажите шаг за шагом, как вы в последний раз распределяли задачи.
   
2. Какие инструменты вы используете?
   - Jira? Asana? Monday.com? Excel?
   - Почему выбрали именно эти инструменты?
   
3. Как часто вам приходится перераспределять задачи?
   - В каких ситуациях это происходит?
   
4. Сколько времени у вас уходит на распределение/перераспределение?
   - Бывают ли дни, когда вы тратите на это больше времени?
```

#### Block 2: Проблемы
```
5. С какими проблемами вы сталкиваетесь при распределении задач?
   - Приведите 2-3 последних примера
   
6. Почему для вас это является проблемой?
   - К чему это приводит?
   - Как это влияет на вашу работу/команду?
   
7. Что для вас является самым сложным в этом процессе?
   - Почему?
   
8. Использовали ли вы автоматические инструменты для распределения?
   - Если да, то почему перестали/продолжаете использовать?
   - Если нет, то почему не пробовали?
```

#### Block 3: Цели
```
9. К чему вы стремитесь при распределении задач?
   - Какой идеальный результат для вас?
   
10. Что для вас означает "идеально распределенные задачи"?
    - Как вы понимаете, что распределение было успешным?
    
11. Что бы вы изменили в текущем процессе, если бы могли?
    - Почему именно это?
```

#### Block 4: Мотивация
```
12. Почему для вас важно эффективно распределять задачи?
    - Что происходит, когда задачи распределены плохо?
    
13. К каким более высоким целям это приводит?
    - Зачем вам вообще нужно управление проектами?
    
14. Что было бы, если бы распределение задач занимало 5 минут вместо 2 часов?
    - Как бы это изменило вашу работу?
```

#### Block 5: Payment
```
15. Сколько времени вы тратите сейчас на распределение задач?
    - В неделю? В месяц?
    
16. Если бы был инструмент, который сокращал это время на 80%, сколько вы были бы готовы платить?
    - В месяц?
    
17. Вы бы предпочли:
    - [ ] Единоразовую покупку
    - [ ] Ежемесячную подписку
    - [ ] Оплату за количество пользователей
```

### Wrap-up:
```
18. Есть ли что-то еще, что вы бы хотели добавить об управлении проектами?
    
19. Если бы вы могли изменить одну вещь в ваших текущих инструментах, что бы это было?
    
20. Знакомы ли вам другие проект-менеджеры, которые могли бы поделиться своим опытом?

Спасибо огромное за ваше время!
```

## 🔹 Часть 5: Формирование профилей

### Profile 1: "Ритейл PM — Ручной контролер" (P-001)

#### Демография:
```
Возраст: 28-35 лет
Пол: 50/50 M/F
Локация: Москва, Санкт-Петербург
Доход: $80,000-$120,000/year
```

#### Контекст:
```
Ситуации: Ежедневное распределение задач, изменение приоритетов
Частота: Каждый день, несколько раз в день
Команда: 5-15 человек
Время: В течение всего дня
```

#### JTBD:
```
Когда [появляются новые задачи или изменяются приоритеты],
я хочу [быстро перераспределить задачи с учетом загрузки и навыков],
чтобы [команда могла продолжать работу без задержек и я мог фокусироваться на стратегии].
```

#### Текущие решения:
```
1. Excel для планирования (каждое утро)
2. Jira для execution
3. Личные звонки/сообщения для срочных изменений
4. Whiteboards для team planning sessions
```

#### Проблемы:
```
Критические:
- 1-2 часа daily на ручное перераспределение
- Не может быстро реагировать на изменения (30+ минут)
- Автоматические инструменты не учитывают контекст

Важные:
- Интеграция между инструментами отсутствует
- Руководство не видит реальную загрузку команды
- Постоянная необходимость перепроверять статусы
```

#### Цели:
```
Функциональные:
- Распределять задачи за 5 минут вместо 2 часов
- Учитывать загрузку и навыки каждого сотрудника
- Быстро реагировать на изменения (15 минут)

Эмоциональные:
- Чувствовать контроль над ситуацией
- Не испытывать стресса от перегрузки команды
- Доверять инструменту

Социальные:
- Выглядеть эффективным перед руководством
- Быть хорошим лидером для команды
```

#### Мотивация:
```
Почему важно:
- "Это основа моей работы. Если задачи распределены плохо, все страдает."

К высшим целям:
- Эффективная команда -> Удовлетворенные клиенты -> Growth компании

Что будет, если проблема исчезнет:
- "Я смогу фокусироваться на стратегии, а не на tactical work."
- "Я смогу уходить домой в 18:00, а не в 20:00."
```

#### Willingness to Pay:
```
Текущие затраты: $50/month (Jira) + $20/month (Excel 365)
Готовность платить: $200-400/month за идеальное решение
Предпочтение: Monthly subscription per user
```

#### Цитаты:
```
"I spend 10+ hours per week just reshuffling tasks. That's crazy."

"I tried automatic tools before. They don't understand our context. I'd rather do it myself."

"When a client calls with urgent changes, I need 15 minutes, not 30. Client trust is at stake."

"I want to trust the system, but I've been burned too many times."
```

#### Оценка:
```
Объем: 3 (medium segment)
Платежеспособность: 3 (high)
Уровень боли: 3 (critical)
Доступ: 2 (medium - доступен через LinkedIn/industry events)

Total Score: 3 × 3 × 3 × 2 = 54
```
