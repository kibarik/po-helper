# 3.3 AI-формирование Bunch

## 🎯 Цель

Использовать ИИ (Кортекс) для анализа и создания Bunch.

**Время:** 2-4 часа  
**После:** Bunch из 5-7 features с refined CP

---

## 📋 Что такое Кортекс?

**Простыми словами:** Кортекс = "AI system" которая помогает анализировать Нексусы и принимать решения.

**В этой инструкции:** Используем ChatGPT/Claude как Кортекс.

---

## 📝 Process

### Step 1: Prepare Prompt (30 min)

**Template:**
```markdown
**Prompt for Кортекс:**

"Analyze these [N] opportunities and identify top 5-7 for Bunch.

**Context from Нексус Рынка:**
[paste key insights: TAM/SAM/SOM, competitors, trends, opportunities]

**Context from Нексус Продукта:**
[paste: Current state, Vision, Gap]

**Context from Нексус Роста:**
[paste: Business model, unit economics, growth levers]

**Opportunities to analyze:**
[paste full master list with initial CPs]

**Selection criteria:**
1. Maximize mNSM potential (monetizable value)
2. Minimize time-to-market (0-6 month window)
3. Fit within team capabilities
4. Synergy (complementary features amplify impact)

**For each selected opportunity:**
- Refine Confidence Point (1-10 scale)
- Estimate mNSM impact (low/medium/high)
- Flag dependencies (must-do before/after)
- Estimate Bunch Window (time to business result)

**Return:**
1. Ranked Bunch (top 5-7)
2. Rationale for each selection
3. Refined Confidence Points
4. mNSM estimates
5. Dependencies flagged"
```

### Step 2: Execute AI Analysis (30 min)

**Send prompt to:**
- ChatGPT (GPT-4 recommended)
- Claude (Opus or Sonnet)
- Or your preferred AI tool

### Step 3: Review & Refine (1-2 hours)

**Questions для AI response:**
1. Сделаны ли разумные предположения?
2. Учитывала ли AI реальную команду capabilities?
3. Реалистичны ли mNSM estimates?

**Refine if needed:**
- Adjust CP (if AI over/underestimated)
- Re-rank based on team knowledge
- Add context AI missed

---

## ✅ Output Template

```markdown
# Bunch v1.0

## Selected Features (5-7)

### 1. [Feature Name]
- **Description:** [What it does]
- **Source:** [Which opportunity?]
- **Confidence Point:** [X]/10
- **mNSM Impact:** [Low/Medium/High]
- **Dependencies:** [What must exist first?]
- **Bunch Window:** [Time to result]

### 2. [Feature Name]
...

## Bunch Summary
- **Total features:** [X]
- **Average CP:** [X]
- **Bunch Size:** [X] features
- **Bunch Window:** [X] months
- **Expected mNSM:** [Description]

## Not Included (Why?)
1. **[Opportunity]:** Dropped because [reason]
2. **[Opportunity]:** Deferred because [reason]
```

---

## 🎯 Success Criteria

✅ 5-7 features selected  
✅ All have CP estimates  
✅ Dependencies flagged  
✅ mNSM estimated per feature  
✅ Bunch Window defined
