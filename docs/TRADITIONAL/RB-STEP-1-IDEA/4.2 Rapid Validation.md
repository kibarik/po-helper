# 4.2 Rapid Validation

## 🎯 Цель

Проверить гипотезы через rapid experiments.

**Время:** 1-3 дня (на feature)  
**После:** Validation results, updated CP

---

## 📋 Validation Methods по Confidence Level

### CP 1-3: Smoke Tests (1-3 days)

**Methods:**
1. **Landing page test**
   - Create simple landing page
   - Measure click/signup rate
   - Target: 5-10% conversion

2. **Concierge MVP**
   - Manual service behind "automated" UI
   - Test willingness to pay
   - Target: 3-5 paying customers

3. **Wizard of Oz**
   - Looks automated, actually manual
   - Test core value proposition
   - Target: 10 positive responses

### CP 4-6: Prototypes (2-5 days)

**Methods:**
1. **Figma/Maze prototype**
   - Interactive mockup
   - Test with 5-10 users
   - Target: 70%+ complete tasks

2. **Pre-sale**
   - Take orders before building
   - Test willingness to pay
   - Target: 5-10 pre-orders

### CP 7-8: Customer Validation (3-7 days)

**Methods:**
1. **Deep interviews**
   - 5-10 customers on specific value prop
   - Target: 80% express strong interest

2. **Data analysis**
   - Analyze existing customer behavior
   - Look for proxy metrics
   - Target: Clear signal in data

---

## 📝 Process

### For Each Bunch Item:

**Step 1: Choose Validation Method (30 min)**
- Based on current CP
- Based on time available
- Based on resources

**Step 2: Execute Validation (1-5 days)**
- Run experiment
- Collect data
- Document results

**Step 3: Analyze Results (30 min)**
- Did hypothesis validate?
- What surprised you?
- What did you learn?
