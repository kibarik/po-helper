# 6.deliver — DELIVER (Отгрузка)

> Стадия 6 · Роль: **Deliverer** · Артефакт: рендер `<service>.puml` + публикация · **STOP** (сухой прогон → ок человека) перед публикацией.
> Только после 🟢/🟡 на [[5.validate]]. Непроверенное за готовое не выдаётся.

## 🎯 Цель

Превратить связанную wire-диаграмму в презентабельный вид (как `Blueprint 24h` [SB4]) и опубликовать/передать. Раньше этой стадии картинку не полируем (принцип 11).

## 👤 Роль

**Deliverer** — доводит форму и публикует. Отвечает за то, что финальный вид **не потерял** вертикали и моменты истины ради эстетики.

## 📥 Вход

- `<service>.puml` + `validation.md` (🟢/🟡).

## 🛠️ Метод (по шагам)

1. **Финальная раскладка.** Из wire-режима (`salt`) → в презентабельную сетку: колонка-легенда слева (дорожки + подписанные линии), столбцы-шаги, цвета сценариев, ⭐ моменты истины, красные fail-points ([[../notation]]).
2. **Проверь сохранность смысла.** После полировки вертикали и стрелки-опоры по-прежнему читаются; ничего не «съедено» ради красоты.
3. **Рендер.** Собрать PlantUML → SVG/PNG. Инлайн-просмотр в чате — через [[diagram-view]].
4. **Сухой прогон.** Показать финальный вид человеку: что публикуем, куда, в каком формате. **STOP** — ждать «ок».
5. **Публикация.** Куда решил человек: файл в репозитории, Confluence, доска. Приложить `validation.md` (🟡 → явный список открытых вопросов).
6. **Манифест.** Короткий `delivery.md`: что отгружено, где лежит, версия, режим (as-is/to-be), нерешённые вопросы.

## 📤 Выход

- Отрендеренная диаграмма (SVG/PNG) + исходник `.puml`.
- Публикация в целевом месте.
- `delivery.md` (манифест).

## 🚫 Анти-паттерны

- Публиковать без 🟢/🟡 валидации.
- Публиковать без сухого прогона и «ок» человека.
- Ради красоты убрать подписи линий или стрелки-опоры.
- Отдать 🟡 без явного списка `[УТОЧНИТЬ]`.

## ✅ Done

Диаграмма отгружена, вертикали связны, моменты истины и fail-points видны, открытые вопросы честно приложены. Follow-up (обновление at-is → to-be, детализация под-Blueprint) — новый прогон конвейера с [[0.frame]].
