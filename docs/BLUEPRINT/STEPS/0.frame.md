# 0.frame — FRAME (Рамка сервиса)

> Стадия 0 конвейера · Роль: **Service Framer** · Артефакт: `artefacts/frame.md` · **STOP** после.
> Ничего не рисуем, пока не зафиксирована рамка. Без актора и границ диаграмма расплывётся.

## 🎯 Цель

Зафиксировать **что именно** схематизируем: сервис, актора, границы, режим (as-is/to-be) и ось времени (шаги). Это фундамент, на который встанут все дорожки.

## 👤 Роль

**Service Framer** — не рисует диаграмму, а очерчивает поле. Отвечает за то, чтобы актор был один, границы — явные, а конфликтующие взгляды стейкхолдеров вынесены, а не усреднены (принцип 9, [[../principles]]).

## 📥 Вход

- Запрос: «нужен Blueprint для \<сервиса\>».
- Любой имеющийся контекст: CJM, интервью, регламенты, скриншоты систем, слова стейкхолдеров.

## 🛠️ Метод (по шагам)

1. **Назови сервис и его цель.** Одна строка: «Парковочный сервис 24 ч: клиент паркуется и оплачивает» (ср. `Blueprint 24h` [SB4]).
2. **Зафиксируй актора (одного).** Кто главный клиент этого Blueprint? Разные сегменты → отдельные ветки/цвета или отдельные Blueprint (принцип 4).
3. **Отметь entry / exit points.** Где путь начинается и заканчивается? (`Drive in` … `Drive home`).
4. **Набросай ось времени — крупные шаги.** 5–12 шагов верхнего уровня, глаголами клиента. Это заголовки столбцов; детализируются на стадии JOURNEY.
5. **Выбери режим:** As-Is (чиним существующее) или To-Be (проектируем). Если чиним — **всегда сначала As-Is** (принцип 2).
6. **Собери источники и пробелы.** Что известно и откуда; что неизвестно → в «Открытые вопросы» с `[УТОЧНИТЬ у {кого}]`.
7. **Вынеси конфликты взглядов.** Где клиент, бэк-офис, регулятор расходятся — записать, не схлопывать.

## 📤 Выход — `frame.md`

```
# Frame — <сервис>
- Сервис и цель: …
- Актор: … (сегмент)
- Режим: As-Is | To-Be
- Entry / Exit: … → …
- Ось времени (черновые шаги): [шаг1, шаг2, …]
- Источники: [интервью/CJM/регламент/…]
- Открытые вопросы: [УТОЧНИТЬ …]
- Конфликты взглядов: …
```

## 🚫 Анти-паттерны

- Два актора в одной рамке → диаграмма-каша.
- Начать рисовать дорожки, не зафиксировав шаги.
- Проектировать To-Be без As-Is.
- «Потом уточним» без записи в Открытые вопросы.

## ⏸️ STOP

Вывести `frame.md`, дождаться, что человек подтвердил актора, границы и режим. Только потом → [[1.journey]].
