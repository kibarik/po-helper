Куда прикрепится доска Backlog · предложение v2

Пайплайн не меняется. Каждая стадия на своём STOP синхронизирует состояние на доску: статус задачи = контроль процесса, чек-лист AC = стадии. Так PO не теряет ни один БФТ из операционного контекста.

Жизненный цикл контроля · состояния и переходы

Пять статусов задачи Backlog. Ключевая точка — Wait for Review: черновик БФТ у внешнего PO (заказчик, который принимает результат) на приёмке требований. Обе ветки (одобрено / отказано) требуют обязательной аудит-записи в notes.

Зелёная ветка — одобрение: append-notes (кто/когда/замечания); Accepted только когда все правки внесены и учтены в БФТ. Красная ветка — отказ: append-notes (кто/какие правки), возврат в In Progress с повышением приоритета.
