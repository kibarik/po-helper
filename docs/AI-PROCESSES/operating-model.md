# ⚙️ Operating Model — правила игры AI-Native продуктовой команды (PAF)

> Единая операционная модель, на которой стоят **все 9 шагов** AI-PROCESSES. Прочитай ОБЯЗАТЕЛЬНО перед шагами.
> Источник: PAF [S1]–[S4]. Терминология — строго по [[sa_documentation/naming_conventions|naming_conventions]].

---

## 🏛️ 5 столпов опер.модели

### 1. Нексус (Nexus) — живой контекст [S1] III.1
«Живая» информационная модель объекта управления = **цифровой двойник** = динамический PRD эпохи ИИ. Существует весь ЖЦ. Состоит из **Узлов (Nodes)**. Типы: рынка, потребителя, продукта, системы роста, операционной модели, компании.
- **Метрика качества — Context Ripeness (Спелость контекста)** = **полнота × актуальность**. Два этапа: «созревание» (рост инфо) и «увядание» (потеря актуальности).
- **Обязанность инженера** — повышать Context Ripeness.
- Узел несёт метаданные: `источник[]`, `Confidence Point`, `timestamp`, `статус увядания`.

### 2. Кортекс (Cortex) — ИИ как операционный интерфейс [S1] III.1
Стандартизированная система автоматизации всех процессов: **агенты / RAG / интеграции / промпты**.
- **Ключевое правило:** инженер работает **только через проводника — Кортекс**, не вручную. Ставит задачу → Кортекс манипулирует Нексусами.
- 3 оптимизации роли [S2] III.4: **автоматизация** (скорость) · **аугментация** (качество/челендж) · **агентизация** (делегирование).
- **Сначала Нексус, потом Кортекс** [S1] IX — без спелого контекста Кортекс усиливает мусор.

### 3. Банч (Feature Bunch) — вместо Беклога [S1] VI.6, Принцип 1
Связка фич/гипотез/механиков **в моменте**, формируемая **только текущим состоянием** Нексуса под Карту Целей. **Беклог упразднён.**
- Лимиты: **Bunch Size** (скорость восприятия рынка) · **Bunch Window** (период ожидания результата).
- Три критерия банча: ① макс. mNSM; ② мин. рисков (CP); ③ эффект в окне → **комплексный NPV**.

### 4. Confidence Point (CP) — управление рисками = управление знаниями [S1] VI.7, Принцип 4
Уровень контекста и разница между текущими и нужными знаниями. **Ядро управления рисками (энтропией), а не потерь.**
- **Растёт** по stage-gate и при появлении информации, снижающей неопределённость (колесо Итамара Гилада, ICE/RICE).
- **Падает** при срабатывании рисков/отклонениях (Кортекс/инженер).
- Включает **оценку стоимости риска**: «сколько ещё потратим ресурсов при таком уровне рисков?».
- CP = **критерий прохождения гейтов** ([[fit-points]]) и **осознанного отказа**.

### 5. Цикл Product Sprint — повторяющийся движок [S1] VI.4–VI.5
PAF-спринт — не единый поток, а асинхронные «сезоны». **6 фаз**, которые крутятся в каждом шаге:

| Фаза | Действие | Артефакт |
|---|---|---|
| **PULSE** | Progress Pulse: фиксация состояния Нексуса; гэп → intent | snapshot + цель цикла |
| **SCOUT** | Скаутинг возможностей/угроз через 3 Линзы (Understand, Und-Id-Ex) | новые узлы Нексуса, opportunities/threats |
| **BUNCH** | Формирование Банча из текущего состояния (Identify) | Банч фич/гипотез (Size/Window) |
| **PITCH** | Pitching of Trust: защита Банча → CP | final CP, гейт-решение |
| **EXECUTE** | Discovery: валидация гипотез; Build (6–8): Build+Bale+Germination | результаты/инкремент |
| **HARVEST** | Harvesting: фиксация NPV/mNSM, инкремент контекста | обновлённый Нексус → след. шаг |

**Und-Id-Ex** [S1] VI.5: Understand (анализ Нексуса) → Identify (генерация идей, рост CP) → Execute (масштабирование). Null base (нет данных → discovery с нуля) / Data base (есть продукт → извлекаем знания).

## 🔭 Три Линзы (3PL) [S1] VI.1

| Линза | Объект | Спринт |
|---|---|---|
| **Продукта** | продукт | Product Sprint (Нексус продукта → Банч) |
| **Бизнеса** | модель роста | Business Sprint (Рычаги/Lever → NPV) |
| **Стратегии** | портфель бизнес-юнитов | Strategic Sprint (долгосрочные цели + Нексус рынка → Ставки/Bets) |

Каждый шаг AI-PROCESSES знает, в какой линзе работает, и связан с двумя другими через **Карту Целей (Goal Map)**.

## 🧭 6 принципов PAF [S1] Часть II

1. **Прозрачность до очевидности** — Нексусы, отсутствие Беклога.
2. **Совместная унификация контекста** — единый Нексус, а не разрозненные документы. *«Контекст — новый способ конкуренции»*.
3. **Гибкая стабильность (Stagility)** — стабильность цели + гибкость пути.
4. **Управление рисками = управление знаниями** — не потери, а риски (энтропия); инструмент — Confidence Point.
5. **Скаутинг вместо приоритизации** — нет списка задач, есть карта целей; поиск возможностей/угроз.
6. **Прибыль через ценность** — сначала востребованный продукт и PMF, затем масштабируемая бизнес-модель; метрика mNSM.

## 👥 AI-Native роли [S1] III.3, [S2] III.6

| Роль | Объект управления | Функция |
|---|---|---|
| **Product Engineer** (замена PM) | продукт | Максимизация монетизируемой ценности через свойства; работает через Кортекс. НЕ вайбкодер. |
| **Product Ops / Product Architect** | product operations | Настройка/эффективность Нексусов и Кортекса; **рост ИИ-зрелости команды** (аналог scrum-master AI-эпохи). |
| **Growth Engineer / Growth PM** | модель роста | Минимизация time-to-activation, максимизация LTV/mNSM; контроль **COGS токенов**. |
| **Portfolio Manager / Bizdev Architect** | портфель | Максимизация NPV портфеля, баланс маржинальности/каннибализации. |
| **Discovery / Launcher PM** | запуск продукта | Минимизация time-to-market/time-to-PMF; человек-команда, **вайбкодинг**. |
| **AI UX Designer** | UX & UI | Метафоры ИИ-взаимодействия, tone-of-voice. |
| **Technical / AI Product Manager** | продукт с ИИ | Качество моделей (точность/прецизионность/полнота); KPI моделей. |

**Зона человека (не передавать ИИ) [S2] III.4:** стратегическое планирование (распределение ресурсов) и синхронизация взаимодействия людей. Агенты — инструменты, не акторы; ответственность не передать.

**Профиль компетенций:** **Comb-shaped** (гребёнка за счёт Кортекса), не T-shaped. Команда = **Pod** (может быть из 1 человека — весь цикл).

## 🛡️ Гвардраилы качества (критично для каждой фазы) [S2] III.4, III.7

### Метапознание — критический фильтр прироста качества от ИИ
> Junior без метапознания + ИИ = **нулевой прирост**. Компетентный, но забивший на верификацию = **«убедительные, но неверные»** результаты (Harvard × BCG).

**→ В каждой фазе Sprint:** явный чекпоинт метапознания — человек верифицирует ИИ-результат, имеет veto, трассирует каждое утверждение до источника.

### Workslop — «рабочий мусор» (~15,4 %; MIT: 95 % без отдачи от ИИ)
Генерация материалов, перекладывающих когнитивную нагрузку на потребителя. **Снижает, а не повышает эффективность.**

**→ В каждой генерирующей фазе (SCOUT/BUNCH/EXECUTE):** workslop-фильтр. Топ-зоны: рыночные цифры (TAM), AI-персоны, офферы/скрипты, вайбкодинг, AI-креативы.

### Анти-patterns
- «Кортекс = один большой промпт» → Кортекс = агенты + RAG + интеграции.
- «Документация как цель» → цифровой профиль продукта (Нексус).
- «Метрика ради метрики» → mNSM/NPV/CP.
- **COGS-эксплозия AI-фич** → AI-COGS как отдельный рычаг (cost-per-token × tokens-per-user).
