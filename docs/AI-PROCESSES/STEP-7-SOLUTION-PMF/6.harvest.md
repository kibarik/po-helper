# 6.harvest — HARVEST / PMF-точка (Step 7: Solution & PMF) 🔴

> Фаза ТЯЖЁЛАЯ · Линза: Product · **PMF-точка (🟫): Harvesting NPV/mNSM + PMF-детектор (конвергенция ≥3 сигналов).** Гейт [[fit-points#PMF]].
> Источник: [[AI-TRANSFORMATION/RB-STEP-7-SOLUTION-VALIDATION/7.6 instruction]] · [S1] VI.4, Принцип 6.

---

## 🎯 Цель фазы для Step 7

**Harvesting цикла Product Sprint** [S1] VI.4: фиксация **NPV/mNSM** как инкремента контекста в Нексусе продукта. Внедрение количественной (AARRR, иерархия метрик, когортный retention, воронка продаж) и качественной (Feedback Matrix, 5 Whys, JTBD-опросы) аналитики + определение критериев **Product/Market Fit**. NSM превращается в **mNSM** (монетизируемая NSM); **PMF-детектор** непрерывно мониторит сигналы.

Свойства PMF [раннбук 7.6]:
- PMF — **градиент**, не бинарное (продукты двигаются по градиенту ценности);
- PMF существует **только в контексте конкуренции** (в монополии нет PMF — есть отсутствие выбора);
- ценность = степень удовлетворения потребности **по сравнению с альтернативами**.

Цель — достичь **конвергенции ≥3 сигналов** PMF, пройти гейт 🟫, зафиксировать инкремент контекста → триггер нового Progress Pulse → переход к Step 8 (Acquisition / PCF).

> Цикл Product Sprint замыкается. Логика «кругов на воде»: ты не видишь данных конкурентов, но если пользователи возвращаются месяц за месяцем — они находят продукт ценным по сравнению с альтернативами.

## 👤 Роль

**Product Engineer** — отвечает за конвергенцию сигналов PMF (≥3); стратегическое решение «достигнута ли масштабируемая бизнес-модель» (по Бланку/Дорфу) — переход MVP → Product Growth. **Discovery/Launcher PM** — минимизация time-to-PMF; итерации инкремента по сигналам аналитики; вайбкодинг A/B-тестов; сокращение time-to-Aha через онбординг. **AI UX Designer** — онбординг вокруг Aha Moment; tone-of-voice опросов. **Technical/AI Product Manager** — контроль **COGS токенов** как рычага; стресс-тест «COGS-эксплозия» при масштабировании (usage ×2/×5). **Product Ops** — PMF-детектор, дашборды, увядание метрик.

> **Зона человека** [S2] III.4: **решение об объявлении PMF и масштабировании — стратегический выбор, не делегируется агенту.** Ответственность не передать; агенты = инструменты, не акторы.

## 🤖 Кортекс-агент (из 7.6)

- **Аналитика-агент** — трекинг событий (имена `[Объект]_[Действие]`); **иерархия метрик**: NSM (North Star — ключевая ценность, не выручка) → Traction → Key → Pulse; **AARRR** (Dave McClure: Acquisition/Activation/Retention/Revenue/Referral); **когортный retention** (Mixpanel/PostHog); **воронка продаж** (количество, конверсия, длительность); frequency/adoption (ключевые/специальные/служебные/ненужные фичи).
- **Aha Moment-агент** — выявляет событие первого осознания ценности тремя методами: (1) анализ retention («оставшиеся» vs «ушедшие»); (2) качественные интервью; (3) гипотезы из core user flow. Отслеживает % достигших Aha за X дней.
- **PMF-детектор-агент** — непрерывно мониторит по алгоритму выбора метрики PMF:
  - **высокочастотная потребность** → retention-плато (кривая выравнивается → PMF есть; падает к нулю → нет); событие retention = **реальное получение ценности** (проигрывание трека / завершение урока), а не вход в приложение;
  - **низкочастотная** → referral/post-deal, конверсия «порция → следующая порция»;
  - **качественно**: опрос Шона Эллиса после ключевого события (**≥40% «очень расстроюсь»**);
  - рекомендация без стимулов (referral rate, NPS > 40);
  - **NPV > 0 воспроизводимо**.
- **Feedback Matrix-агент** — 4 квадрата (Комплимент/Совет/Предложение/Критика × эмоция); **5 Whys** для корневых причин (каждый ответ — факт, не гипотеза; дерево, а не цепочка).
- **mNSM-агент** — проверяет монетизируемость NSM; связывает NSM → mNSM → ребро NPV; для AI — COGS токенов в полный CAC.
- **Harvesting-агент** — фиксирует NPV/mNSM как инкремент контекста; триггерит новый Progress Pulse (цикл заново) [S1] VI.4.
- **Dashboard-агент** — 5 блоков: North Star (ежедневно), Воронка (еженедельно), Retention (еженедельно), Revenue (ежедневно), Quality (средний рейтинг, NPS, % «Очень расстроюсь»).

## 🧠 Человек (метапознание — конвергенция сигналов PMF; валидация ИИ-расчётов)

- **Валидация ИИ-генерации PMF-сигналов**: PMF-детектор может дать ложноположительный сигнал (retention-плато на маленькой нерепрезентативной когорте; «40%» на <40 ответах). Человек требует: **≥3 конвергирующих сигнала**; минимальную выборку (**50–100 в когорте** для retention, **≥40 ответов** для опроса). Один опрос — не статистика.
- **Veto на premature PMF**: объявление PMF на одном сигнале («опрос показал 42%») без retention-плато и NPV > 0 — отбраковка. **PMF — конвергенция ≥3 сигналов** [S1] reformation 7.9.
- **Конвергенция**: сводит количественное (retention/referral) и качественное (опрос/интервью); «один сигнал — недостаточно, все три вместе — сильный аргумент». Различение поведенческого PMF (используют больше) и ценностного (получают больше ценности) — могут расходиться.
- **Стратегическое решение о масштабировании**: «перестаём сомневаться в базовой модели и масштабируем то, что работает» — переход MVP → Product Growth — за человеком. Сегментация: часто PMF есть для одного сегмента, но нет для другого — фокус на «очень расстроятся».
- **Метапознание-фильтр доверия к AI-расчётам**: проверка расчётов COGS, retention-норм по бенчмаркам (суждение по аналогии с похожим классом продуктов); маркировка факт/гипотеза/бенчмарк.

## 🚦 CP-гейт фазы — HARVEST = гейт [[fit-points#PMF]]

**Gate 7.6 → Harvesting (PMF-точка): конвергенция ≥3 сигналов** [7.6]:
1. **Количественно:** retention-плато (или referral/post-deal для низкочастотных) сопоставимо с бенчмарком по аналогии — привязано к **mNSM**;
2. **Качественно:** опрос Шона Эллиса **≥40% «очень расстроюсь»** на достаточной выборке (**≥40 ответов**);
3. **Бизнес-объектив:** **NPV > 0 воспроизводимо**;
- дополнительно: рекомендации без стимулов (NPS > 40), Aha Moment зафиксирован, Change Log инкремента собран.
- **Confidence Point PMF-гипотезы ≥ высокого порога** (PMF — дорогой гейт, отсюда масштабирование).
- До конвергенции — итерации инкремента, **не масштабирование**.

> **Не пройден → главная точка осознанного отказа** в Product Discovery [S1] reformation 7.9. **Premature PMF = масштабирование непродукта.** После PMF → Step 8 (Acquisition) ищет способ привлечения, который окупается и масштабируется.

## 🛡️ Анти-workslop

- **Workslop в метриках / premature PMF** (главный риск 7.6 [S1] reformation 7.9) — объявление PMF на vanity-метрике, на маленькой выборке, на одном сигнале, без NPV.
- **Метапознание-фильтр**: PMF — конвергенция ≥3 сигналов; каждый сигнал проверяется на достаточность выборки и репрезентативность; **retention-событие = реальное получение ценности** (не вход в приложение).
- **Трассировка до Нексуса**: каждый `pmf-signal` связан с `mnsm-node` и `npv-node`; сигнал без связи с монетизируемой ценностью — vanity.
- **NSM ↔ mNSM-путаница**: NSM («завершённые уроки», «минуты прослушивания») ценностна, но может быть не монетизируема; mNSM-агент проверяет монетизируемость; без неё NPV не считается.
- **Bottleneck смещается на дистрибуцию** [S2] III.5 (Голдратт): после PMF bottleneck переходит на Germination/последнюю милю; COGS токенов дистрибуции — отдельный рычаг (мост в Step 8).
- **Дофаминовая петля ИИ** [S2] III.7: метрика ритуала = изменение CP/NPV, а не «приятно поработать с аналитиком».
- **Аналитика до запуска**: планировать и настраивать трекинг — до запуска MVP; собирать данные — нет (пользователей ещё нет). Ретроспективно восстановить не трекавшееся крайне сложно.
- **PMF как gate в Product Growth**: после конвергенции ≥3 сигналов и NPV > 0 команда переходит от фазы «поиск модели» (MVP) к Product Growth. Это стратегическое решение **за человеком, не за PMF-детектором**.
