## 🎯 Цель фазы для Step 6

Сформировать **Bale** = инкремент + **оффер** + план дистрибуции [S1] VI.4 — ядро цикла Product Sprint, связывающее продукт с дистрибуцией. На этой фазе Bale собирается из двух половин:

### Половина 1 — Оффер (6.2): AI-генерация УТП + tone-of-voice + гейт метапознания
- **Генератор офферов (4 формулы из источника)** — ≥5 вариантов УТП по RAG из Нексуса рынка/продукта (не свободная генерация):
  - ① **Максвелл** — результат + время/условие + гарантия/снятие риска;
  - ② **4U** — Usefulness / Uniqueness / Ultra-specificity / Urgency;
  - ③ **«Привычное без привычного»** — категория без главного неудобства;
  - ④ **AIDA(s)** — Attention / Interest / Desire / Action / satisfaction.
- **Оффер ≠ УТП**: оффер = УТП + цена + условия + CTA + снятие риска.
- **tone-of-voice** от AI UX Designer; **messaging-иерархия** (Главный тезис → 3–4 аргумента → доказательства).
- **Elevator Pitch** (3 варианта: инвестор/клиент/партёр, ≤30 сек).
- **A/B-симулятор офферов** — синтетические агенты-клиенты (4-я роль «секретарь/адъютант» [S2] III.8): тест «понятность с нуля» и тест «замены конкурентом».

### Половина 2 — Бюджет как инвестиция в NPV (6.4): Monte-Carlo + критерии Банча
- **Симуляция воронки/бюджета** Monte-Carlo по вероятностной воронке Visits→Leads→Prospects→Clients→Activated (CR1–CR4 как распределения, не точки). `CAC_max = LTV/3` — ограничение, не цель.
- **5 блоков бюджета**: Acquisition (реклама) / Производство материалов / Эксперименты (15–20% от Acquisition) / Активация.
- **AI-COGS как отдельный рычаг** — `cost-per-token × tokens-per-user` для AI-фич привлечения; стресс-тест «COGS-эксплозия» (usage ×2/×5); **COGS токенов в полный CAC**. Запрещено прятать в «серверы/OpenAI API» [reformation 5.3].
- **Бизнес-Банч каналов** (2–3 канала в моменте) под Карту Целей с **комплексным NPV** — 3 критерия Банча [S1] VI.6: ① макс. mNSM; ② мин. рисков (CP); ③ эффект в окне (Bunch Window).

**Метрики-вершины** — mNSM/NPV/COGS, не CAC/LTV как самоцель.

## 👤 Роль

- **AI UX Designer** — **владелец половины офферов**: tone-of-voice офферов/месседжинга; проектирование метафор ИИ-взаимодействия (намерение→черновик→уточнение).
- **Growth Engineer** — владелец бюджета-NPV; оффер и канал как рычаги mNSM; контроль COGS токенов; A/B-симуляция; минимизация time-to-activation.
- **Product Engineer** — выбор Топ-3 офферов для Bale; veto на публикацию; связка оффер ↔ проверенные ЦП Stage 4; Product/Channel Fit; veto на масштабирование (Gate-B).
- **Product Ops** — настройка генератора офферов, гейта метапознания, агента симуляции воронки; версионирование Offer/Budget-узлов; event-пересчёт при данных из Soft Launch.

**Зона человека:** выбор главного оффера (стратегический выбор); NPV-моделирование и распределение ресурсов (агенты = инструменты, не акторы [S2]); юридическая проверка обещаний/сравнительной рекламы.

## 🤖 Кортекс-агент

- **Генератор офферов (4 формулы)** + **AI UX Designer-агент tone-of-voice** + **агент-оффер-билдер** (сборка блоков Заголовок/Подзаголовок/Что входит/Цена/Гарантия/CTA) + **агент Elevator Pitch** + **агент-A/B-симулятор офферов**.
- **Агент-симуляция воронки/бюджета** + **агент-Monte-Carlo CPL/CAC** + **агент-COGS-калькулятор** + **агент-оптимизатор бизнес-Банча каналов** + **агент-санити/анти-workslop** (детект «убийц экономики»: COGS-эксплозия, ложный breakeven, каскадная ошибка в sensitivity).

## 🧠 Человек (метапознание) — ОБЯЗАТЕЛЬНЫЙ ГЕЙТ

- **Гейт метапознания (анти-workslop) перед каждым публикуемым оффером**: проверяемость каждой цифры (87%, 10 минут, 23 параметра — где источник?), каждой гарантии (юридически исполнима?), каждого обещания результата. **Офферы — топ-зона workslop всей стадии** [reformation 6.3].
- **Тест «замены конкурентом»**: подставь имя конкурента в УТП — если не сломалось, оффер недостаточно уникален → CP не поднимается.
- **Проверяемость каждой цифры бюджета**: CPL/CAC/LTV из RAG с URL+датой; маркировка факт/гипотеза/бенчмарк. Workslop в финмоделях — «убедительные, но неверные» числа [reformation 5.9].
- **Вопросы себе**:
  - A/B-симулятор подтвердил, или доверяем «красивому тексту»? (Harvard × BCG [S2]).
  - Симуляция прошла (синтетика) или a posteriori подтверждено (Soft Launch)? — CP не поднимается без реальных данных.
  - CAC_max = LTV/3 — а LTV верифицирован, или прогноз? AI-COGS учтён в полном CAC, или спрятан?

## 📦 Артефакт = узлы Нексуса (роста + рынка)

- **Offer-узлы** (формула, УТП-заголовок, подзаголовок, цена, гарантия, CTA, сегмент, CP, `источник-выгоды[]` — ссылки на узлы Нексуса рынка/продукта, timestamp, статус увядания).
- **Messaging-Framework-узел** (иерархия Headline → Аргументы → Доказательства).
- **Elevator-Pitch-узлы** (3 варианта, хронометраж 25–35 сек).
- **Budget-узлы** по периодам (Acquisition/Production/Experiments/Activation) с CP, `источник+дата`.
- **Funnel-узел** (CR1–CR4 как распределения Monte-Carlo); **CAC-max-узел** (LTV/3); **AI-COGS-узел** (отдельный рычаг, стресс-тест ×2/×5).
- **Lever-узлы (Рычаги NPV)** — ранжирование по эластичности NPV (связка с 5.5).
- **Бизнес-Банч каналов** (2–3 канала в моменте) с комплексным NPV.
- Контекст: **Bale** = эти узлы + план дистрибуции (6.3/6.4) + скрипт (6.6).

## 🚦 CP-гейт

- **Gate-A (публикация оффера)**: гейт метапознания пройден ∧ каждая цифра/гарантия верифицирована ∧ тест «замены конкурентом» провален конкурентом ∧ Elevator Pitch ≤30 сек. CP оффера ≥ gate.
- **Gate-A (запуск канала, связка с 6.3)**: CP канала ≥ порога ∧ CAC-прогноз ≤ CAC_max ∧ COGS-стресс пройден ∧ mNSM-потенциал > 0.
- **Gate-B (масштабирование — откладывается до Soft Launch в EXECUTE)**: CP вырос после валидации ∧ фактический CAC ≤ CAC_max ∧ mNSM подтверждена ∧ **AI-COGS-стресс пройден** ∧ NPV > 0.

---

## 🛡️ Анти-workslop — ГЛАВНАЯ ЗОНА (офферы 6.2)

- Офферы — топ-зона workslop всей стадии [reformation 6.3]. Каждый публикуемый оффер обязан пройти гейт метапознания: трассировка каждой цифры до узла Нексуса с URL+датой; каждая гарантия — юридически исполнима; каждое обещание — измеримо.
- **Запрет**: «высококачественный / профессиональный / надёжный / лучший» без измеримой конкретики → workslop.
- **COGS-эксплозия AI-фич** — спрятанный COGS даёт ложный breakeven → AI-COGS = отдельный рычаг, стресс ×2/×5, COGS в полный CAC.
- Ложное ощущение готовности (симуляция ≠ a posteriori) → Soft Launch обязателен в EXECUTE.
