# Глоссарий Карты процесса-опыта

Термины метода (автор — Андрей Шапиро, Byndyusoft). Пересказ своими словами; человеческая версия с бытовыми примерами — `docs/XPM/osnovy/slovar.md`.

| Термин | Значение |
|---|---|
| **Актор (деятель)** | Самостоятельный участник (обычно потребитель услуги). Свободен, может «сойти» с линии опыта в любой момент. Не подчинён нам. |
| **Агент** | Участник, обслуживающий процесс по регламенту/алгоритму. Человек или машинный сервис. |
| **Артефакт** | Искусственное инструментальное средство, которое мы создаём для работоспособности сервиса. |
| **Барьер** | Нежелательный эффект в потоке: разрыв, бутылочное горлышко, застойная зона, потери. |
| **Канал** | Часть тракта, несущая лишь часть потока (телефон / сайт / офис / чат-бот). |
| **Ключевая точка ●** | Основной элемент карты; место-контейнер, где происходит контакт, изменение или важное решение. Группирует операции; не равна действию. |
| **Линия тока** | Горизонтальная линия, собирающая точки в цепочку; задаёт течение логического времени. ⎯ надёжная / ┈ ненадёжная. |
| **Линия взаимодействия ⦙** | Вертикальная пунктирная линия между точками разных участников в общей ситуации. |
| **Событие ◇** | Мгновенное изменение состояния; веха для ориентации/запуска линий тока. |
| **Триггер ◇** | Условие-«запускатель», организует ветвление процесса. |
| **Поток** | Непрерывное движение чего-либо (внимание, информация, деньги, вещи) и система, его организующая. |
| **Тракт** | «Труба»/«тоннель» — конструкция, по которой движутся частицы потока. |
| **Точка контакта** | Место, где потребитель явно встречается с услугой (стойка, касса, форма на сайте). Частный случай ключевой точки. |
| **Процесс-механизм** | Заранее спроектированная последовательность запускаемых функций; держится на причинно-следственных связях. |
| **Естественное / искусственное** | Различение: что самовоспроизводится «само» vs что мы специально организуем. Тезис: само по себе надёжно не работает. |
| **Работоспособность** | Способность системы выполнять свою функцию: (1) работает ли вообще; (2) с каким качеством/долговечностью. |
| **Функциональное место** | Абстракция «роли»: связывает функцию с тем, кто её исполнит, оставляя исполнителя варьируемым (человек или программа). |
| **Услуга** | Деятельность с двумя ролями — благополучатель и исполнитель; из их взаимодействия рождается благо. |
| **Целевая система** | То, что проектируем. Рекомендуется брать не только ИС, а ближайшую «родительскую» деятельность вокруг неё. |
| **Шаг развития** | Переход «как есть» → «как будет»: осознанный проект качественного изменения целевой системы. |
