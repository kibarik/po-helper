# Паттерны картирования (типовые конфигурации)

Готовые приёмы для частых ситуаций. Одну ситуацию можно выразить разными комбинациями элементов — выбирай по контексту. Автор метода — Андрей Шапиро, Byndyusoft. Человеческие версии с примерами — `docs/XPM/kachestvo/patterny.md`.

---

## Старт и течение
- **Линейный путь с опциональными точками (◠)** — точку можно пропустить (обход поверх ●).
- **Безусловное ветвление и схождение** — поток раздваивается и снова сходится без условия.
- **Многовариантный старт от событий** — процесс начинается из разных ◇-событий.
- **Многовариантный старт через ключевые точки** — несколько входных точек.
- **Повторные вхождения** — через **предваряющие события** (◇, дающие повторный вход в фрагмент).

## Ветвления
- **Событием** — ветка запускается ◇-событием.
- **Триггером ◇** — условие-переключатель направляет поток по веткам.
- **Таблицей решений (decision table)** — много условий → таблица вариантов (инструмент `combinator.xlsx` у автора).
- **Цикл** — возврат назад по линии тока (повтор шага).

## Взаимодействия
- **Петля «пинг-понг»** — посыл → отработка → получение результата (запрос-ответ; напр. заказ в кафе).
- **Синхронизация точек на разных дорожках** — общая ситуация двух участников, вертикальная ⦙ (разговор через прилавок, общий звонок).
- **Совместные точки на дорожке одного участника** — маркер начала совместных точек до следующего указания участников.

## Навигация по карте
- **Группирующие этапы** — крупные подписи-этапы наверху (на дорожке ведущего актора) для ориентации.
- **События-ориентиры (◇)** — сопутствующие события как маркеры-«вехи» в пространстве карты.
