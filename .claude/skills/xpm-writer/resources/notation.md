# Нотация Карты процесса-опыта (справка)

Весь «алфавит» метода — **4 элемента + 5 модификаторов**. Низкий порог входа — ключевое преимущество (против сотен элементов BPMN). Автор — Андрей Шапиро, Byndyusoft.

## Значки

| Знак | Значение | Excalidraw (см. `render_excalidraw.md`) |
|---|---|---|
| ● | ключевая точка | `ellipse` с заливкой |
| ○ | ключевая точка **вне нашего контроля** | `ellipse` без заливки (только контур) |
| ◠ | **опциональная** точка (обход поверх ●) | `ellipse` + пунктирный контур |
| ◇ | **триггер** или **событие** | `diamond` |
| ⎯ | **линия тока** — уверенный переход | сплошная `arrow` |
| ┈ | линия тока с **задержкой/обрывом** — необеспеченный переход | пунктирная `arrow` |
| ⦙ | **линия взаимодействия** — синхронизация участников (вертикальная) | вертикальная пунктирная `arrow` без стрелок |

## Ключевая точка ●

Атомарное функциональное место, где происходит **одно из трёх**: **решение** (осознание) · **контакт** (пересечение линии сервиса) · **изменение** (вход → выход). Точка — **не действие**; группирует операции. Имя — **существительное**. Подробно — `concepts.md`.

## Участники

- **Актор 👩** — самостоятелен (клиент), может «сойти» с линии; дорожка сверху.
- **Агент 👱/🤖** — обслуживает по регламенту (сотрудник/система); дорожки ниже, каждая обслуживает вышестоящую.

## Триггеры и события ◇

- **Событие** — мгновенная веха для ориентации/запуска линий тока (самое короткое время жизни).
- **Триггер** — условие-«запускатель», организует ветвление (пускает поток по одной/нескольким веткам).

## Линии

- **Линия тока** (горизонтальная): ⎯ надёжная / ┈ ненадёжная. Задаёт течение логического времени слева направо.
- **Линия взаимодействия** (вертикальная пунктирная ⦙): связывает точки разных участников в общей ситуации.

## Аннотация точки

Под именем точки: `вход` · `выход` · `операции` · `барьеры` · `артефакты/системы` · `каналы`. Именно тут живут разрабатываемые ИС (не отдельной дорожкой!).
