# SMART-задача — REQ-2026-014

**Заголовок:** Автоматизировать сверку выгрузки с биллингом для закрытия месяца
**Запросил:** финансист команды учёта · **Дата:** 29.06.2026 · **Команда:** учёт/биллинг

## Контекст
Финансисты вручную сверяют выгрузку с биллингом при закрытии месяца. Объём вырос, сверка занимает полдня и стала узким местом. Корень — не «неудобная выгрузка», а недостоверность закрытия без сверки.

## Суть изменения
Расхождения между выгрузкой и данными биллинга вычисляются и показываются автоматически в одном отчёте; ручная построчная сверка при закрытии месяца не требуется.

## Критерии приёмки
- [ ] Отчёт показывает все расхождения выгрузка↔биллинг за период одним списком — R1: «видеть расхождения в одном отчёте»
- [ ] Финансист закрывает месяц, не сверяя строки руками — R1: «не сводить руками»
- [ ] Значимые расхождения (выше порога) выделены — R1 + [УТОЧНИТЬ: порог]

## Ценность / метрика
Время на сверку при закрытии: с ~4 часов до < 30 минут (R1: «полдня»). Достоверность закрытия — расхождения не теряются.

## Границы
- Входит: автоматический расчёт и показ расхождений за период.
- НЕ входит: изменение самого биллинга; автоматическое исправление расхождений (только показ).

## Контекст домена
Источник выгрузки — биллинг (`NEXUS/product/billing.md`, R2). Похожий закрытый запрос — PROJ-118 (касался ручного экспорта, не сверки), R2.
