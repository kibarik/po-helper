# Правила якорения (Anchor Rules) — intake

> Главный рычаг против галлюцинаций: каждое утверждение ← якорь. Для intake источник ценности — **собственные слова запрашивающего**. Нет валидного якоря → `[УТОЧНИТЬ у запрашивающего]`, не выдумывать. Адаптация anchor-модели дома (bft-writer) под первичную распаковку запроса.

## Ранги достоверности якоря

| Ранг | Тип якоря | Когда валиден | Пример |
|---|---|---|---|
| **R1** 🥇 | **Прямая цитата запрашивающего** (из `interview.md`) | ценность, боль, образ результата и приёмки, контекст использования | `«я трачу полдня на сверку выгрузок руками» (интервью, 29.06)` |
| **R2** 🥈 | Домен знаний команды: NEXUS-узел, CORTEX (C1/C3/BR/ADR), задача трекера, страница wiki | As-Is, смежные системы, похожие прошлые запросы, ограничения | `NEXUS/product/billing.md`, `[JIRA PROJ-204](…)`, `BR-DATA-002` |
| **R3** 🥉 | Вывод агента: явная гипотеза, требующая проверки | классификация запроса, реконструкция мотива, предположение о scope | `[вывод — проверить с запрашивающим]` |
| **R4** ❌ | «Обычно так», «лучшая практика», общие знания, додуманная за стейкхолдера ценность | **НЕ валиден** | — |

## Ключевое отличие от bft-writer

В БФТ ценность якорится на PO-решение/ТЗ (R3 там = PO). Здесь ценность ещё **не подтверждена PO** — её источник это **сам запрашивающий**, поэтому цитата = высший ранг R1. Задача intake — собрать эти цитаты честно, не подменяя их догадками агента.

## Hallucination Gate (🔴)

- Утверждение о ценности/боли/приёмке без цитаты R1 и без `[УТОЧНИТЬ]` → 🔴. Нельзя приписывать стейкхолдеру то, чего он не говорил.
- Факт о домене (As-Is, смежная система) без R2 → 🔴.
- Оценочное утверждение («это всем нужно», «обычно так») без ранга → 🔴.

## Формат в артефактах

Ценностные утверждения сопровождаются цитатой-якорем прямо по тексту или сводятся в раздел «Якоря»:

| Утверждение | Якорь | Ранг | Тип |
|---|---|---|---|
| Боль — ручная сверка выгрузок | `«трачу полдня… руками» (интервью)` | R1 | Цитата |
| Источник выгрузок — биллинг | `NEXUS/product/billing.md` | R2 | Домен |
| Похоже на закрытый запрос PROJ-118 | `[JIRA PROJ-118](…)` | R2 | Трекер |
| Вероятно, это про регулярный отчёт | `[вывод — проверить]` | R3 | Вывод |

## Правила
1. Каждое ценностное утверждение ← цитата запрашивающего (R1) или `[УТОЧНИТЬ]`.
2. Не приписывать стейкхолдеру ценность/приёмку, которую он не проговорил.
3. Факты о домене ← R2 (NEXUS/CORTEX/трекер).
4. Вывод агента всегда помечается `[вывод — проверить]`, не выдаётся за факт.
5. Несколько якорей лучше одного.
