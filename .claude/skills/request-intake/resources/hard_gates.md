# Hard Gates intake + Routing-матрица + Светофор

Стадия `/req-score` совмещает две роли: **Validator** (созрел ли запрос) и **Router** (что с ним делать). Сначала прогон гейтов, затем routing-решение, затем Светофор.

## Часть A — Hard Gates (бинарные 🔴)

Любой сработавший гейт = запрос не готов к оформлению в SMART/БФТ. Адверсариальная проверка (red-team против «придуманной ценности») встроена сюда — отдельной стадии дебатов в v1 нет.

| # | Гейт | Условие 🔴 |
|---|------|-----------|
| 1 | **Hallucination** | Ценностное утверждение без цитаты-якоря R1 и без `[УТОЧНИТЬ]` (см. `anchor_rules.md`) |
| 2 | **Imagined-Value** | Ценность приписана стейкхолдеру, но не подтверждена его словами (R4) — придуманная боль |
| 3 | **No-Acceptance** | Нет образа приёмки: непонятно, как запрашивающий поймёт, что готово |
| 4 | **Solution-as-Problem** | Предложенное стейкхолдером решение выдано за проблему/цель (не отделена суть от пути) |
| 5 | **Altitude** | Артефакт опустился на системный уровень (API/БД/User Stories/декомпозиция) — это зона bft-writer |
| 6 | **Open-Questions** | Отсутствует раздел «Открытые вопросы» при наличии неизвестного (тихий пропуск) |
| 7 | **Scope-Owner** | Не определено, наша ли это команда (achievable/relevant под вопросом, а редирект не рассмотрен) |
| 8 | **Smart-Incomplete** | В smart-task.md не закрыта буква SMART без явной пометки `[УТОЧНИТЬ]` |

## Часть B — Routing-матрица (решение)

После гейтов агент классифицирует запрос. Решение — ровно одно. Только «→ БФТ» ведёт на `/req-handoff`; остальные — early exit с обоснованием в `scoring.md`.

| Решение | Когда | Действие агента | Артефакт |
|---|---|---|---|
| **Консультация** | Запрос — это вопрос, не изменение; ответ есть в домене (NEXUS/CORTEX) | Ответить grounded, со ссылкой на узлы домена, и закрыть. SMART не нужен. | ответ + `scoring.md` |
| **Отклонить** | Вне scope продукта / дубль существующего / нет подтверждённой ценности (гейт 2) | Краткое обоснование «почему нет» для стейкхолдера | `scoring.md` |
| **Редирект** | Запрос валиден, но это **не наша команда** (гейт 7) | Указать целевую команду и почему; передать контекст | `scoring.md` |
| **→ БФТ** | Валидный change-request: ценность (R1) и приёмка ясны, scope наш, все гейты 🟢/🟡 | Запустить `/req-handoff` | финальный `smart-task.md` |

Сигнал ранней классификации приходит из `interview.md`, но **окончательное** решение — здесь, после распаковки и гейтов.

## Часть C — Светофор (итог)

| Слой | 🔴/🟡/🟢 | Доказательство | Что исправить |
|------|---------|----------------|---------------|
| Якорение ценности (R1) | | каждое ценностное утверждение ← цитата | |
| Подтверждённость ценности (не придумана) | | боль из слов запрашивающего, не R4 | |
| Образ приёмки | | проверяемые критерии «готово» | |
| Проблема ≠ решение | | суть отделена от предложенного пути | |
| Высота (altitude) | | нет API/БД/User Stories/декомпозиции | |
| Scope/владелец | | команда определена (или редирект) | |
| Полнота SMART | | S/M/A/R/T закрыты или `[УТОЧНИТЬ]` | |
| Открытые вопросы | | раздел есть, `[УТОЧНИТЬ]` собраны | |

- 🟢 — все гейты пройдены, ценность заякорена, приёмка ясна → routing-решение исполняется.
- 🟡 — мелкие неточности, не блокирующие (напр. `[срок: УТОЧНИТЬ]`) → можно в хэндофф с пометкой.
- 🔴 — нарушен Hard Gate → возврат на соответствующую стадию (`/req-interview` если не вытащили ценность/приёмку; `/req-gap` если не отделена проблема; `/req-smart` если неполнота SMART).

## Возвраты (циклы)

- 🔴 гейт 1/2/3 (ценность/приёмка) → `/req-interview` (доспросить).
- 🔴 гейт 4 (проблема=решение) → `/req-gap`.
- 🔴 гейт 5/8 (altitude/SMART) → `/req-smart`.
- Решение «Редирект/Отклонить/Консультация» → early exit, пайплайн завершён.
