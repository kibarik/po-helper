# ImpactMap — шаблон и правила

> ImpactMap превращает сырое интервью в структурированную карту ценности. Шесть полей отвечают на: зачем, для кого, что, почему сейчас, какие пути, почему этот. Каждое поле ← якорь (цитата запрашивающего R1 / домен R2 / вывод R3). Пустое поле без якоря → `[УТОЧНИТЬ]`.

## Шесть полей

| # | Поле | Вопрос | Источник |
|---|---|---|---|
| 1 | **Цель** | Зачем это нужно? Какой бизнес/пользовательский результат? | боль+работа из интервью (R1) |
| 2 | **Для кого** | Кто получает ценность? Кто пользователь/бенефициар? | Блок 6 интервью (R1) + NEXUS customer (R2) |
| 3 | **Что делаем** | Какое наблюдаемое изменение в продукте/процессе? (на языке результата, не реализации) | работа из интервью (R1) |
| 4 | **Почему сейчас** | Что изменилось/какой триггер/цена бездействия? | Блок 1-2 интервью (R1) |
| 5 | **Какие пути видит запрашивающий** | Варианты достижения цели, включая предложенное им решение | Блок 4 интервью (R1) |
| 6 | **Почему именно этот путь** | Чем выбранный путь лучше альтернатив (или `[УТОЧНИТЬ]`, если не обоснован) | интервью (R1) / вывод (R3) |

## Правила

1. **Цель ≠ решение.** Поле «Цель» — про результат и ценность («достоверное закрытие месяца»), не про фичу («кнопка выгрузки»). Фича живёт в поле 5 (путь).
2. **Отделяй проблему от пути.** Предложенное стейкхолдером решение — это поле 5 (один из путей), а не поле 1 и не поле 3.
3. **«Что делаем» — на высоте результата.** Наблюдаемое изменение для пользователя, без системных деталей (altitude, принцип 8).
4. **Каждое поле заякорено.** Предпочтительно цитатой (R1). Вывод агента — `[вывод — проверить]` (R3). Нет основания → `[УТОЧНИТЬ у запрашивающего]`.
5. **Множественность путей.** Поле 5 держит ≥2 пути, если они были. Не схлопывай в один (даже если стейкхолдер настаивает на своём) — обоснование выбора идёт в поле 6.

## Шаблон

```markdown
# ImpactMap — <request_id>

| Поле | Содержание | Якорь |
|---|---|---|
| Цель | <бизнес/пользовательский результат> | R1: «цитата» |
| Для кого | <бенефициар/сегмент> | R1 / R2 |
| Что делаем | <наблюдаемое изменение, на языке результата> | R1 |
| Почему сейчас | <триггер / цена бездействия> | R1 |
| Пути (видит запрашивающий) | 1. <путь, в т.ч. предложенный им>; 2. <альтернатива> | R1 |
| Почему этот путь | <обоснование выбора> | R1 / R3 / [УТОЧНИТЬ] |

## Открытые вопросы
- [УТОЧНИТЬ у запрашивающего]: <…>
```
