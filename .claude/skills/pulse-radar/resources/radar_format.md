# Формат радар-заметки — спецификация

Единый источник правил оформления `PULSE/radar`-заметок. Используется скиллом `/pulse-radar`.

---

## Имя файла

```
{pulse_radar_dir}/{ГГГГ-ММ-ДД}-pulse-radar.md
```

где дата = дата прогона скилла.

---

## Frontmatter

Порядок ключей обязателен:

```yaml
---
nexus: pulse
node_type: radar
source: <источник из адаптера, напр. "MTS Link">
window: <ГГГГ-ММ-ДД>..<ГГГГ-ММ-ДД>
generated: ГГГГ-ММ-ДД
chats_scanned: <n>
signals_count: <n>
tags: [progress-pulse, radar, chats]
---
```

`signals_count` считает только строки-сигналы секций 🔴 / 🚧 / ✅ / ❗. Строки секции
📈 (влияние — перекрёстные ссылки) и агрегат 🔕 не учитываются.

---

## Тело — фиксированный порядок секций

```
## 🔴 Инциденты
## 🚧 Блокеры
## ✅ Решения / изменения
## 📈 Влияние на цели спринта
## ❗ Требует твоего ответа
## 🔕 Прочее / инфошум
```

Все шесть секций присутствуют всегда. Пустая секция содержит одну строку `_нет сигналов_`.

---

## Формат строки сигнала

```
- <severity> <суть одной фразой> ← <Чат> · <Автор> · <ЧЧ:ММ>[ (ГГГГ-ММ-ДД)] · <KR-код|[KR?]> · <[NEW]|[идёт]>
```

где:
- **severity** ∈ `🔴` (критично) | `🟠` (важно) | `⚪` (инфо) — **всегда** ведёт строку, включая секцию ✅ (эмодзи секции ≠ severity)
- **Чат** — имя чата/встречи (источник, без выдумки); личный диалог → `ЛС <Имя>`
- **Автор** — полное имя из `NEXUS/team`; неизвестный → не додумывать, написать как есть в дампе
- **ЧЧ:ММ** — время реплики из дампа; не определено → `--:--`. Если дата реплики ≠ дате прогона — добавить ` (ГГГГ-ММ-ДД)` сразу после времени
- **KR-код** — `KR X.Y` (из `node_id kr-X-Y`) или `KR X.Y.Z` (суб-KR); нет уверенного матча → `[KR?]`
- **[NEW]** — сигнала не было в предыдущей радар-заметке; **[идёт]** — был и продолжается

**Тип `risk`** (потенциальная угроза, ещё не инцидент/блокер) кладётся в 🚧 Блокеры с
severity 🟠; при материализации → 🔴 Инциденты.

**Пример строки:**
```
- 🟠 payments: остатки только в кэше, миграция в БД не сделана ← Чат «Платежи» · Иванов Иван · 09:15 · KR 1.2 · [NEW]
```

---

## Секция «Влияние на цели спринта»

Строки добавляются только для сигналов с явной привязкой к KR. Формат:

```
- <суть сигнала> (ускоряет|тормозит|риск) KR X.Y
```

Один KR — одна строка (несколько сигналов к одному KR → объединить фразой).

---

## Секция 🔕 Прочее / инфошум

Свернуть в 1–2 строки перечислением чатов-вещаний без разбора по сигналам:

```
- ⚪ Каналы-вещание без значимых сигналов: <Название1>, <Название2>, … ← скипнуто (только анонсы/новости)
```

---

## Пустое окно

Если за окно нет значимых сигналов, тело заметки = одна строка:

```
_Значимых сигналов за окно нет._
```

frontmatter при этом всё равно заполняется (с `signals_count: 0`).

---

## Метки [NEW] vs [идёт]

- **[NEW]** — сигнал не найден в последней заметке папки `PULSE/radar/`. Первый прогон (предыдущей заметки нет) → все сигналы `[NEW]`.
- **[идёт]** — тот же сигнал (по теме + KR + чату) присутствовал в последней заметке. Диф по последней заметке, без отдельного хранилища состояния.
