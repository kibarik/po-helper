# Правила уверенности — route_confidence vs confidence узла

Два независимых показателя, не путать друг с другом (дизайн-спека §6).

## 1. `route_confidence` — уверенность классификации узла в нексус

Вычисляется Router'ом на стадии `/cindex-route` для каждого узла-кандидата.
Определяет `action` в `routing.jsonl`:

| `route_confidence` | `action` | Куда |
|---|---|---|
| ≥ **0.7** | `auto` | записывается в целевой нексус на стадии `/cindex-deliver` |
| < 0.7 | `queue` | `_intake/unrouted.md` — очередь ревью PO |

## 2. `confidence` узла — доверие к содержимому

Не путать с `route_confidence`. Это **Confidence Point узла** из Node schema
(`sa_documentation/nexus_schema.md` §2.2): контекст, ингестированный из
Confluence, — это допущение онбординга, не валидированный факт.

**`confidence` = `0.3` всегда**, для любого узла, созданного Confluence-
индексатором, независимо от `route_confidence` и от нексуса-назначения.
Оцифровка ≠ валидация — поднятие CP выше 0.3 делают уже другие процессы
(интервью, эксперименты, Steps 1–8), не индексатор.

## 3. `ttl_days` — срок актуальности узла по нексусу

По правилу empirical-узлов из `nexus_schema.md` §2.2:

| Нексус | `ttl_days` |
|---|---|
| `market` | 90 |
| `customer` | 90 |
| `growth` | 60 |
| прочие (все остальные нексусы реестра) | 180 |

## 4. Триггеры очереди `_intake/unrouted.md`

Узел уходит в очередь (`action: queue`), если срабатывает любой из триггеров:

1. **Низкий `route_confidence`** — ниже порога 0.7.
2. **Конфликт двух нексусов** — Router не может однозначно выбрать между
   двумя кандидатами из `routing_table.md` (напр. `precedents` vs
   `system-landscape` для пограничного решения).
3. **Orphan** — узел без связей (`[[wiki-links]]`) с остальным графом; риск
   создать несвязанный, ненаходимый контекст.
4. **Отсутствие подходящего нексуса** — сигнал не подпадает ни под одну
   строку `routing_table.md` (последняя строка таблицы).

Очередь не исполняется молча: PO ревьюит `_intake/unrouted.md` и принимает
решение вручную (маршрут, нексус, отклонение).
