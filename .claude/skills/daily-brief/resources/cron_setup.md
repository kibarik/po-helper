# Автозапуск сводки — раз в день / утром перед дейликом

`/daily-brief` спроектирована событийной/фоновой: **read-only и идемпотентна**, ничего необратимого не пишет (только снапшот `brief.md` и вывод). Поэтому её безопасно гонять по расписанию сколько угодно — как `/release-sync`. Ниже — способы повесить сводку на «раз в день».

## Когда запускать

За 15–30 минут **до дейлика**, чтобы к стендапу сводка уже лежала в чате. Время дейлика — из `cadence` профиля (`.claude/domain-profile.md`). Дни — только рабочие (Пн–Пт или по каденции команды).

## Вариант A — OS cron (локально / на сервере PO)

Крон дёргает Claude Code в headless-режиме. Пример: будни в 09:30, сводка в файл и (опц.) в чат.

```cron
# min hour dom mon dow   command
30 9 * * 1-5   cd /path/to/project && claude -p "/daily-brief" >> GROUND/PULSE/briefs/_cron.log 2>&1
```

- `-p` — headless (без интерактива). Навык соберёт сводку, запишет снапшот в `brief_notes`, выведет дайджест в stdout/лог.
- Доставку в рабочий чат (Telegram/Slack) навесить отдельным шагом на файл снапшота или на stdout — это вне навыка (навык отдаёт copy-ready текст, канал доставки выбирает PO).

## Вариант B — Claude Code scheduled trigger (web/remote)

Если проект живёт в Claude Code on the web / remote-окружении — завести scheduled trigger (Routine), который раз в день шлёт в сессию prompt `/daily-brief`. Расписание — тот же будний слот перед дейликом. Триггер фиксируется в окружении проекта; PO управляет им через настройки Routines.

## Вариант C — вручную

Просто набрать `/daily-brief` перед дейликом. Автоматизацию можно навесить позже — навык от способа запуска не зависит.

## Почему безопасно на расписании

- **Read-only:** трекер читается без записи, Backlog только читается. Единственная запись — снапшот сводки в PULSE (git-tracked, неизменяемый на прогон).
- **Идемпотентно:** каждый прогон пересобирает состояние из источников заново. Два прогона подряд не портят данные — только пишут два датированных снапшота (историю даёт git).
- **Деградация:** нет трекера/VPN в момент прогона → сводка соберётся из vault с пометкой `[НЕДОСТУПНО]`, а не упадёт.

## Гигиена

- Окно по умолчанию — «со вчера» (от последнего снапшота). На cron это ровно «что произошло за сутки», без ручной настройки дат.
- Лог cron складывать в `brief_notes/_cron.log` (в `.gitignore`, если не нужен в истории).
- Токены/доступ к трекеру — только через MCP окружения. В crontab секретов не хранить.
