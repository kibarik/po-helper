# Формат brief.md — сводка по людям для дейлика

`brief.md` — **copy-ready дайджест** активности команды за окно. Копируется «как есть» в рабочий чат перед дейликом или читается PO с экрана. Это главный человеко-facing артефакт навыка. Формат повторяет эталон-скриншот PO: строка на человека с эмодзи-маркером, `@username`, что сделал, и открытый вопрос/нюдж к дейлику.

## Жёсткие правила

- **Эмодзи-маркеры разрешены** (в отличие от `blockers.md`/`report.md`) — это осознанное исключение под человеко-facing чат, как на эталоне. Маркер — один в начале строки, из фиксированного словаря ниже. Декоративных эмодзи в прозе описаний нет.
- **Строка на человека, не на карточку.** Один участник = один блок (1–3 строки максимум). Если у человека несколько карточек — свести в одну строку через `·`, не плодить.
- **Группировка по Sprint Goal / KR отгрузки.** Заголовок-строка = цель/KR; под ним — участники, чья активность служит этой цели. Активность «вне плана» — в отдельный блок в конце.
- **Каждая строка — с вопросом/нюджем к дейлику.** Не «приговор» («не сделал»), а вопрос («вчера метил закрыть ABC-12 — вышло?»). Это разминка синка, а не отчёт надзора.
- **Метаданные** — единственная строка HTML-комментария сверху: `<!-- brief: ГГГГ-ММ-ДД · окно: <со вчера / 3d / ...> · источники: <jira+pulse+backlog / vault-only> -->`. При вставке в чат отбрасывается. Никакого frontmatter.
- **Якорь на каждую строку.** Активность ← ключ JIRA / строка PULSE / задача Backlog / коммит. Нет источника → плейсхолдер (`[КОД?]`, `[КТО?]`), не выдумка.

## Словарь эмодзи-маркеров

Ровно эти семь, по одному в начале строки:

- 🎯 — **движется к цели** — закрыл значимое / карточка ушла дальше по цели спринта.
- ✅ — **вчерашнее обязательство выполнено** — то, что обещал на прошлом дейлике, сделано.
- 📌 — **в работе, нужен догляд** — карточка в процессе, статус актуален, вопрос уточняющий.
- ⚠️ — **застрял / обязательство поехало** — N дней без движения или вчерашнее обещание не выполнено.
- 🚧 — **блокатор** — участник заблокирован (внешняя команда / зависимость / ожидание).
- 🆕 — **вне плана** — активность есть, но карточки не было в плане спринта.
- 💤 — **тишина** — за окно ни одного апдейта (нет активности в источниках).

## Структура

```
<!-- brief: ГГГГ-ММ-ДД · окно: со вчера · источники: jira+pulse+backlog -->

Сводка к дейлику — ГГГГ-ММ-ДД (за <окно>)
Sprint Goal: <повтор из плана спринта>

<KR / цель A>
🎯 @username — <что сделал за окно, результат> · <вопрос/нюдж к дейлику>
📌 @username — <карточка в работе, статус> · <уточняющий вопрос>
✅ @username — <вчерашнее обязательство закрыто> · <что дальше?>

<KR / цель B>
⚠️ @username — <N дней без движения по КОД / обязательство поехало> · <в чём затык?>
🚧 @username — <чем заблокирован, кто вовлечён> · <нужно решение PO?>

Вне плана:
🆕 @username — <что делал вне плана> · <это в приоритете сейчас?>

На кого смотреть на дейлике:
- @username — застрял N дней по КОД (<чем>).
- @username — вчера обещал <что>, не закрыто → спросить причину.
- @username — блокатор <тема>, нужна помощь.

План PO на дейлик:
- <личное действие PO: спросить / эскалировать / согласовать / решить> — <к кому: People Graph, если релевантно>.
```

## Правила ведения

- **«На кого смотреть на дейлике»** — сжатая выжимка из строк выше: только застрявшие (⚠️), поехавшие обязательства и блокаторы (🚧). Кто идёт по цели (🎯/✅) сюда не попадает. Пустой блок не выводить.
- **«План PO на дейлик»** — только личные действия PO (спросить, эскалировать, согласовать), не задачи команды. «К кому идти» тянуть из People Graph (`/people-map`), если вопрос требует навигации. Пусто → не выводить.
- **Стейлнесс «N дней без движения»** — считается от последнего апдейта карточки/участника в источниках (Context Ripeness decay). Порог — `planner.reminder_horizon_days` (дефолт 2). Ниже порога — без пометки.
- **Тишина (💤)** — участник без единого апдейта за окно. Не обвинение: возможно, работа не в трекере (дизайн/аналитика) или отпуск. Формулировать нейтрально («за сутки не видно апдейтов — что в работе?») и сверять доступность по `capacity`/диалогу.
- **Деградация источника.** Трекер недоступен → строки собираются из прошлого снапшота PULSE + Backlog, в шапке `источники: vault-only`, и явная строка: `Трекер [НЕДОСТУПНО] — сводка по последнему дейлику и обязательствам`. Не выдумывать перемещения.
- **Имена/`@username`** сверять с `GROUND/NEXUS/team`; не сматченного участника не додумывать (`[КТО?]`).
