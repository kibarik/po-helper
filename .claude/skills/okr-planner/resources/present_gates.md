# present_gates — Светофор консистентности презентации (`engine/gates.py`)

Гейты презентации квартала — отдельная, более лёгкая проверка, чем `hard_gates.md` (13 гейтов OKR-пайплайна). `engine.gates.audit(spec: DeckSpec) -> Report` прогоняется над уже загруженным `deck-spec` **до** рендера. Бинарного пер-гейтового статуса нет — только агрегированный уровень 🔴/🟡/🟢 (`Report.level`) плюс списки конкретных нарушений (`Report.red`, `Report.yellow`).

Печатает вывод `engine.gates.format_report(rep)`:
```
Светофор консистентности: {level}
  🔴 {сообщение}
  ...
  🟡 {сообщение}
  ...
```
(если ни `red`, ни `yellow` не пусты — строка «всё покрыто»).

---

## 🔴 Красный — блокирует рендер, вернуть PO

Проверяется на каждом KR каждого OBJ каждого направления:

| Условие | Сообщение |
|---------|-----------|
| `kr.now` пусто (`None`/`""`) | `KR {id}: нет СЕЙЧАС (now)` |
| `kr.becomes` пусто | `KR {id}: нет СТАНЕТ (becomes)` |
| `kr.sprint_cells` пуст (`{}`) | `KR {id}: нет ни одной sprint-ячейки (нет исполнителя/цикла)` |

Плюс отдельно по каждому элементу `authored.market[]`:

| Условие | Сообщение |
|---------|-----------|
| `market[i].src` отсутствует/пусто | `Рыночное число '{value}' без источника` |

Наличие хотя бы одного 🔴-нарушения → `Report.level = "🔴"`, независимо от количества 🟡.

## 🟡 Жёлтый — можно рендерить с явным `[УТОЧНИТЬ]`

| Условие | Сообщение |
|---------|-----------|
| `kr.risks` пуст (`[]`) | `KR {id}: без рисков` |
| в `kr.sprint_cells[sp]` есть строка-чип без символа `·` (нет стека, только роль) | `KR {id} {sp}: чип '{c}' без стека` |
| `spec.authored.glossary` пуст (`[]`) | `Пустой глоссарий` |

Если 🔴 нет, но есть хотя бы одно 🟡 → `Report.level = "🟡"`.

## 🟢 Зелёный

Ни одного 🔴, ни одного 🟡 → `Report.level = "🟢"`. Полное покрытие, рендер без замечаний.

---

## Когда печатается

Согласно workflow `/okr-present` (`docs/superpowers/plans/2026-07-08-okr-present.md`): Светофор печатается **дважды**:

1. **Фаза A (Сбор + аудит)** — сразу после сборки `deck-spec.yaml` из vault-фактов, перед STOP на утверждение PO. 🔴 на этой стадии → СТОП, вернуть PO на доработку фактов/vault.
2. **Перед D2 (Финализация)** — повторный прогон после цикла коррекций (фаза E, «на HTML») перед тем, как `render_pptx.py`/`render_roadmap.py` зафиксируют финальные `.pptx`/`.xlsx`. Гарантирует, что правки, внесённые по ходу выравнивания на HTML, не оставили новых 🔴/🟡.

Гейты презентации не заменяют `hard_gates.md`: `deck-spec` строится уже из утверждённого OKR (стадия 6 `/okr-deliver` пройдена), `present_gates.md` проверяет **полноту раскладки для слайдов** (образ результата, исполнители спринта, источник рыночных чисел), а не стратегическую валидность OKR.
