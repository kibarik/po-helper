# Hard Gates — валидация OKR (12 гейтов Фазы 1 + гейты груминга Фазы 2)

Каждый гейт — бинарный: ✅ пройден / 🔴 нарушен. При одном 🔴 — финальный статус 🔴 (блокирует `/okr-deliver`). Гейты груминга (G-G1…G-G3) применяются отдельно — при ревью груминг-степа Фазы 2 (см. секцию ниже).

## Светофор

| Статус | Условие | Действие |
|--------|---------|----------|
| 🟢 Зелёный | Все 12 гейтов ✅ | `/okr-deliver` |
| 🟡 Жёлтый | Нарушены гейты 7–12 (некритичные) | `/okr-deliver` с явными `[УТОЧНИТЬ]`, PO принимает риск |
| 🔴 Красный | Нарушен хотя бы один гейт 1–6 (критичные) | Вернуться на `/okr-enrich` или `/okr-key-results` |

## Критичные гейты (1–6) — блокирующие

### Гейт 1: Количество OBJ
**Правило:** 3 ≤ OBJ ≤ 5.
**Нарушение:** < 3 (нет стратегии) или > 5 (нет фокуса).
**Исключение:** ACTIVITY-OBJ не считается стратегическим, он всегда допустим сверх лимита.

### Гейт 2: Каждый OBJ имеет "Почему важно"
**Правило:** Для каждого OBJ задокументировано: `Проблема (боли)` + `Бизнес-эффект (что станет лучше)`.
**Нарушение:** Пустые поля или формулировка в стиле "потому что важно".

### Гейт 3: Каждый KR имеет IMP
**Правило:** Поле IMP заполнено числом 1–9. Нет пустых, нет `TBD`, нет диапазонов.
**Нарушение:** Любой KR без IMP.

### Гейт 4: Образ результата Critical/High KR (IMP ≥ 7) заполнен полностью
**Правило:** Ни одного `TBD` / пустого поля в "Образ результата" для KR с IMP ≥ 7.
**Нарушение:** `TBD` или пустота в образе результата Critical/High KR.

### Гейт 5: Технические KR имеют БЫЛО→СТАЛО
**Правило:** Для KR с изменением dataflow (интеграция, переключение, миграция) описан `БЫЛО: [dataflow]` и `СТАЛО: [dataflow]`.
**Нарушение:** Dataflow-KR без БЫЛО/СТАЛО.

### Гейт 6: Нет факта без источника
**Правило:** Ключевые утверждения (коммитменты, оценки сроков, стейкхолдеры) имеют якорь: JIRA-ключ, PO-решение, Confluence-ссылку, встречу, roadmap.
**Нарушение:** Утверждение «сделаем к августу» без источника (кто сказал, когда).

## Некритичные гейты (7–12) — 🟡 допустимы

### Гейт 7: Количество KR на OBJ
**Правило:** 2 ≤ KR ≤ 6 на каждый OBJ.
**Нарушение:** OBJ с 1 KR (нет распределения) или > 6 KR (нет фокуса).

### Гейт 8: Образ действия заполнен для Critical/High KR
**Правило:** Для IMP ≥ 7 — образ действия содержит хотя бы 3 шага с ролями.
**Нарушение:** Пустой или однострочный образ действия для Critical/High KR.

### Гейт 9: Зависимости между KR явные
**Правило:** Если KR X блокирует KR Y — в Y указано `Зависит от KR X.N` или `[BLOCKER]`.
**Нарушение:** Найдены неявные зависимости (через БЫЛО/СТАЛО — один KR ссылается на систему другого без аннотации).

### Гейт 10: Critical KR имеют дедлайн/коммитмент
**Правило:** Для IMP = 9 — в "Образ результата" или "Метрики&риски" указан дедлайн или коммитмент перед стейкхолдером.
**Нарушение:** Critical KR без явного коммитмента.

### Гейт 11: [RESEARCH] KR имеют конкретный образ результата
**Правило:** Тег `[RESEARCH]` ≠ оправдание `TBD`. Образ результата = «план / декомпозиция / согласование с N стейкхолдерами к дате».
**Нарушение:** `[RESEARCH]` KR с образом результата `TBD`.

### Гейт 12: Итоговое число KR ≤ 20
**Правило:** Сумма всех KR (без ACTIVITY) ≤ 20. ACTIVITY-KR — не лимитируются.
**Нарушение:** Более 20 активных KR = нет фокуса квартала.

## Гейты груминга (Фаза 2)

Применяются при ревью груминг-степа (`okr_step_{n}.md`), не на `/okr-deliver` Фазы 1. G-G1 и G-G2 — блокирующие: нарушение → груминг невалиден.

### G-G1: Якорный артефакт зафиксирован (блокирующий) 🔴
**Правило:** для каждого груминг-степа в `artifacts/` есть `smart-agreement.md` со `status: ЗАФИКСИРОВАН` ИЛИ `anchor_type: БФТ` со ссылкой на финализированный БФТ в `{bft_store}`.
**Нарушение:** груминг проведён без якоря, или якорь в статусе `ЧЕРНОВИК`. → `/okr-grooming` не должен был стартовать.

### G-G2: Каждый вопрос исследования ← артефакт (блокирующий) 🔴
**Правило:** каждый вопрос `ADR#/SA#/BA#/DEL#` опирается на якорь / БФТ / context-pack. Вопросы без артефакта-основы вынесены в блок «⚠ Требуют артефакта».
**Нарушение:** вопрос-выдумка без основы вписан как обычный — нарушение принципа нулевого допуска к галлюцинациям.

### G-G3: Полнота US-декомпозиции и [EXT] 🟡
**Правило:** каждый US-блок завершается `[QA] Отладка по US{k}` и `[RELEASE] Отгрузка US{k}`; работа внешних команд помечена `[EXT]`.
**Нарушение:** US без QA/RELEASE, либо внешний шаг без `[EXT]`. Для `[RESEARCH]`/`[POC]` без US-декомпозиции — гейт неприменим.

```
G-G1  Якорный артефакт ЗАФИКСИРОВАН            ✅ / 🔴
G-G2  Каждый вопрос ← артефакт                 ✅ / 🔴
G-G3  US закрыты [QA]+[RELEASE], внешн.→[EXT]  ✅ / 🟡
```
