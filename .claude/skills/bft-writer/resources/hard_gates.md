# Hard Gates БФТ + чек-лист самовалидации

## Hard Gates — бинарные 🔴 (автоматический отказ)

Любой сработавший гейт = документ не готов. Исправить до сборки.

| # | Гейт | Условие 🔴 |
|---|------|-----------|
| 1 | **Hallucination** | Требование без источника (трекер / PO / ТЗ / СА) и без маркера `[УТОЧНИТЬ]` |
| 2 | **Traceability** | Пустое «Связанные требования» в любой строке требования |
| 3 | **Identifier** | Нарушение схемы `{БТ|ПТ|ИТ|ФТ|НФТ}-{EPIC}-{N}` |
| 4 | **Boundary** | Отсутствует раздел «Границы системы» (явное ВНУТРИ / СНАРУЖИ) |
| 5 | **NFR** | Оценочная НФТ («быстро/надёжно/масштабируемо») вместо измеримой (P95/%/RPS/срок) |
| 6 | **Negative-Flow** | Интерфейс/сценарий описан только happy path, без `alt`-веток |
| 7 | **Open-Questions** | Отсутствует раздел «Открытые вопросы» (тихий пропуск) |
| 8 | **Priority** | ФТ без приоритета (Высокий/Средний/Низкий) |
| 9 | **Success-Criteria** | Критерии успеха не оцифрованы (нет «метрика → цель → срок») И не вынесены отдельным разделом. Обязательно для фичей/миграций; для чисто технических API-БФТ — либо продуктовые метрики, либо явная пометка «метрики в НФТ» с обоснованием |
| 10 | **Source/Path** | Внешние ссылки без указания источника (URL / ключ трекера / pageId wiki) |

## Чек-лист самовалидации

### 1. Структура и идентификаторы
- [ ] Frontmatter заполнен (source / space / version / synced / jira)
- [ ] H1 содержит EPIC-код и название
- [ ] Все обязательные разделы присутствуют
- [ ] Идентификаторы единообразны `{TYPE}-{EPIC}-{N}`
- [ ] Раздел «Границы системы» явно разделяет ВНУТРИ / СНАРУЖИ

### 2. Трассировка
- [ ] Нет пустых «Связанные требования»
- [ ] Каждая строка ссылается вверх (к БТ) + на эпик в трекере
- [ ] Каскад `БТ ← ПТ ← ИТ` и `БТ ← ФТ ← НФТ` соблюдён

### 3. Качество требований
- [ ] БТ и каждый ФТ имеют приоритет
- [ ] НФТ только измеримые (число / % / RPS / срок)
- [ ] ПТ в формате Когда / Я хочу / Чтобы
- [ ] Каждый ФТ ≈ атомарная задача (1–2 дня), со ссылкой на трекер

### 4. Сценарии и визуализация
- [ ] PlantUML содержит ≥1 ветку `alt` (negative flow)
- [ ] Есть таблица акторов
- [ ] Атрибутивный состав ключевых сообщений (Параметр | Обяз | Тип | Описание)
- [ ] Mermaid не используется
- [ ] Надписи на стрелках на языке документа, технические имена в скобках

### 5. Полнота и неизвестное
- [ ] Раздел «Открытые вопросы» присутствует
- [ ] Закрытые вопросы → «Ключевые решения» (с «кто ответил»)
- [ ] Все пробелы знаний помечены `[УТОЧНИТЬ]` и собраны в начале документа
- [ ] Ни одного выдуманного ключа эпика / статуса / оценки — только из трекера / PO

### 6. Контекст и риски
- [ ] Зависимости между командами вашего проекта со статусом согласования
- [ ] Риски имеют митигацию
- [ ] Стейкхолдеры указаны с контактами
- [ ] Измеримые критерии успеха (метрика → цель → срок)

### 7. Якоря истины
- [ ] Есть финальный раздел «Якоря истины» (факт → источник)
- [ ] Неподтверждённые факты помечены `[УТОЧНИТЬ]`
