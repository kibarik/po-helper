# Примеры и шаблоны Карты реализации историй

Реестр эталонов для калибровки стиля и шаблонов для рисования. Автор метода — Андрей Шапиро / Byndyusoft; первоисточники под лицензией **AGPL-3.0**. Здесь — **ссылки и пересказ своими словами**, не копии оригинала.

## Разобранный пример (наш, для новичка)

- **Онлайн-запись в парикмахерскую** — сквозной разбор всех 4 шагов метода (рамка → 4 рабочие истории → матрица → формы реализации → требования): `docs/SIM/primer/skvoznoy-primer.md`.

Именно на этом примере построен рабочий JSON в `resources/render_excalidraw.md` (3 истории из 4: «Выбор услуги», «Выбор времени», «Подтверждение»).

## Учебные истории из основ

Три коротких эталонных истории по 5 элементам — `docs/SIM/osnovy/rabochaya-istoriya.md`:

| # | История | Чем полезна как образец |
|---|---|---|
| 1 | Умный будильник по фазам сна | Ц смотрит вверх; способ (Д) без гаджета |
| 2 | Сортировщик овощей на конвейере | носитель-машина («манипулятор с камерой»); чистая пара О→О′ |
| 3 | Спикер раздаёт контакты аудитории | одна суть → несколько форм реализации (QR / NFC / ссылка / бумага) |

## Инструмент автора и шаблоны для рисования

- **Официальный инструмент метода** — среда **Социотех** (`sociotech.center`): единственный инструмент от автора, полностью поддерживает КРИ на уровне сценариев и структуры данных.
- **Шаблоны для электронных досок** (fallback, если Excalidraw MCP недоступен): **Numbers**, **Excel**, **Holst.so**, **Figma** (community), **Miro** (miroverse). В них удобно вести матрицу «истории × формы реализации».

> При калибровке стиля вывода: карта должна читаться без знания нотации — столбцы историй по процессу, черта отделяет суть от форм. Меньше украшений, больше смысла. Основной рендер — `resources/render_excalidraw.md`.
