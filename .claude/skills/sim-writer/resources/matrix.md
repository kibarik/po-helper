# Структура матрицы КРИ (для `/sim-matrix`)

Как набор рабочих историй превращается в наглядную карту. Матрица ориентирует элементы **в двух направлениях сразу**, и каждое несёт смысл. Автор метода — Андрей Шапиро, Byndyusoft. Человеческая версия — `docs/SIM/postroenie/shag-3-matrica.md`.

---

## Две оси

| Ось | Направление | Что означает | Простыми словами |
|---|---|---|---|
| **Следование процесса** | слева → направо | порядок шагов во времени | «что за чем идёт» |
| **Подчинение функции** | сверху → вниз | отношение «часть — целое» и порядок элементов истории | «состоит из» / «зачем → чем» |

- **Горизонталь — это «потом».** Истории, идущие одна за другой, ставим слева направо в порядке выполнения: «Выбор услуги» → «Выбор времени» → «Подтверждение».
- **Вертикаль — это «состоит из».** Крупный шаг («Оформление записи») ставим **выше** его под-шагов (A/B/C) — так на карте видно подчинение функции (целое над частями). Внутри истории вертикаль задаёт строгий порядок карточек: Ц → Н → С → Д → О→О′.

> ⚠️ Частый промах: под-шаги («части целого») выстроили в ряд, как будто по времени. Части целого стоят **друг под другом**, а не рядом. См. гейт G9 в `hard_gates.md`.

## Столбец истории — пять карточек

Одна рабочая история = **столбец из пяти карточек**, по одной на элемент, сверху вниз в строгом порядке:

```
┌───────────────┐
│  Ц — цель      │  ← зачем во всём деле        (держит историю сверху)
├───────────────┤
│  Н — носитель  │  ← кто действует
├───────────────┤
│  С — ситуация  │  ← когда
├───────────────┤
│  Д — способ    │  ← как по сути
├───────────────┤
│  О → О′        │  ← с чем работает → что получается
└───────────────┘
```

Сколько историй — столько столбцов рядом. Порядок карточек внутри столбца всегда одинаков, поэтому карту легко **читать по строкам**: пробежался по строке носителей — увидел всех действующих; по строке ситуаций — все обстоятельства.

## Черта и формы реализации

Под каждым столбцом-историей, **после горизонтальной черты**, добавляются карточки **форм реализации** — конкретных способов поддержать историю в жизни (устройство, приложение, экран, документ, услуга). У одной истории их может быть несколько — это варианты, из которых потом выбирают.

```
        История C · Подтверждение
      ┌────────────────────┐
  Ц   │ закрепить место    │
  Н   │ Клиент             │
  С   │ услуга/время выбр. │      ВЫШЕ ЧЕРТЫ — суть («что и зачем»)
  Д   │ подтверждает       │
  О→О′│ черновик → запись  │
      ╞════════════════════╡  ← черта
 форма│ кнопка «Записаться»│      НИЖЕ ЧЕРТЫ — воплощение («чем»)
реали-│ SMS/пуш с подтвер- │
 зации│ ждением            │
      └────────────────────┘
```

> ⚠️ Черта разделяет **«что и зачем»** (сверху) и **«чем»** (снизу). Всё про гаджеты и экраны живёт **только под чертой**. Форма, поднятая выше черты в саму историю, — 🔴 (гейт G7): суть слипается с решением, теряется выбор.

## Формы реализации → требования

Формы реализации — мост от модели поведения к требованиям. Как только форма выбрана, из неё **прямо вытекают требования** — то, что решение должно уметь или из чего состоять. Это главный «продукт» карты.

Трассировка обязательна: **требование ← форма реализации ← история**. Разрыв цепочки → `[УТОЧНИТЬ]` (гейт G8). Требования формулируются на стадии `/sim-requirements`.

Пример (история «Выбор времени», форма «виджет-календарь на сайте»):
- показывать только **свободные** слоты выбранного мастера;
- слоты обновляются в реальном времени, чтобы двое не заняли одно время;
- виджет работает с телефона.

> 📌 Хорошие требования вырастают **прямо из Ц и С** истории — форма обязана служить сути, а не наоборот. Ровно ради этого форму выбирают в последнюю очередь.

## Как выглядит матрица целиком

```
        ← следование процесса (ход дела во времени) →

           A. Выбор услуги   B. Выбор времени   C. Подтверждение
         ┌─────────────────┬──────────────────┬──────────────────┐
   Ц     │ не ошибиться    │ удобное время,   │ закрепить место, │
   Н     │ Клиент          │ Клиент           │ Клиент           │
   С     │ решил постричься│ услуга выбрана   │ услуга/время выбр│
   Д     │ подбирает услугу│ выбирает время   │ подтверждает     │
  О→О′   │ хочу → услуга   │ слоты → слот     │ черновик → запись│
         ╞═════════════════╪══════════════════╪══════════════════╡
  формы  │ каталог услуг   │ виджет-календарь │ кнопка + SMS     │
реализ.  │ звонок админу   │ чат-бот          │ пуш              │
         └─────────────────┴──────────────────┴──────────────────┘
```

## Промахи раскладки (→ возврат)

| Промах | Возврат |
|---|---|
| Под-шаги поставили в ряд вместо «друг под другом» | `/sim-matrix` |
| Сломан порядок карточек Ц·Н·С·Д·О→О′ в столбце | `/sim-matrix` |
| Слишком много столбцов (десятки) — дело слишком широкое | `/sim-frame` (сузить) |
| Под столбцами сразу гаджеты, пока карта не собрана | `/sim-matrix` |

> Рендер матрицы в холст — `render_excalidraw.md`. Проверка перед отгрузкой — `hard_gates.md`.
