# Глоссарий Карты реализации историй

Термины метода (автор — Андрей Шапиро, Byndyusoft). Пересказ своими словами; человеческая версия с бытовыми примерами — `docs/SIM/osnovy/slovar.md`.

## Главные понятия

| Термин | Значение |
|---|---|
| **Карта реализации историй (КРИ / SIM)** | Таблица-карта, в которую рабочие истории раскладываются: слева направо — ход дела, сверху вниз — от целого к частям; под каждой историей после черты — формы реализации и вытекающие требования. Инструмент, чтобы **людям вместе думать и договариваться**, а не запуск для компьютера. |
| **Рабочая история** | Базовый «кирпичик»: короткое, строго устроенное описание **одного шага дела** по пяти элементам (Ц·Н·С·Д·О→О′). Спецификация — `working_story.md`. |

## Пять элементов истории

| Термин | Значение |
|---|---|
| **Цель действия / деятельности (Ц)** | Ради чего шаг нужен во **всём деле целиком** (а не только в самом шаге) и почему именно так. Смотрит **вверх**. «Зачем?» |
| **Носитель действия (Н)** | Кто выполняет шаг: человек, роль или **конкретная часть** машины. Не «система». «Кто действует?» |
| **Ситуация (С)** | Обстоятельства шага: момент, условие, повод. «Когда, при каких обстоятельствах?» |
| **Способ действия (Д)** | Каким именно образом выполняется шаг — его **суть**, а не гаджет. «Как?» |
| **Объект оперирования (О → О′)** | То, с чем работает носитель: О — вход шага, О′ — выход. Пара «до → после» и есть смысл шага. «С чем работает и что получается?» |

## Про воплощение

| Термин | Значение |
|---|---|
| **Форма реализации** | Конкретный способ **поддержать** историю в жизни: инструмент, устройство, приложение, экран, документ, услуга. Появляется **после** описания истории, живёт **под чертой**. Одну суть можно поддержать несколькими формами — это варианты для сравнения. |
| **Требование** | То, что решение должно уметь или из чего состоять; вырастает из выбранной формы реализации. Трассировка: требование ← форма ← история. |
| **Черта** | Горизонтальный разделитель под столбцом истории: **выше — суть («что и зачем»), ниже — воплощение («чем»)**. Ключевой методический приём: не даёт форме-решению протечь в модель поведения. |

## Про устройство карты

| Термин | Значение |
|---|---|
| **Шаг деятельности (шаг дела)** | Одно осмысленное действие внутри дела, которому соответствует **ровно одна** рабочая история. |
| **Следование процесса** | Порядок шагов во времени, «что за чем идёт». Разворачивается **слева направо**. |
| **Подчинение функции** | Отношение «часть — целое»: крупная задача разбита на подзадачи. Разворачивается **сверху вниз** (целое над частями; внутри истории — порядок Ц→Н→С→Д→О→О′). |
| **Столбец истории** | Изображение одной истории на карте — пять карточек-элементов сверху вниз в строгом порядке Ц·Н·С·Д·О→О′. |
| **Модальность** | Режим карты: «как есть» (как дело идёт сейчас) / «как будет» (как хотим после улучшений) / гибрид. Не смешивать в одной карте. |
| **Рамка деятельности** | Шапка карты из `/sim-frame`: какое дело, его цель деятельности, модальность. Держит всех в одном русле. |

## Методические тезисы

| Тезис | Суть |
|---|---|
| **Сначала суть, потом решение** | Сначала «что и зачем» (истории), только потом «чем и как» (формы). Карта не даёт перепрыгнуть через первое ко второму. |
| **Одна история = один шаг** | Проходит через всю работу. Слышно «сначала А, потом Б» — это две истории, режь. |
| **Форму выбирают в последнюю очередь** | И всегда под конкретную суть шага — чтобы честно сравнить варианты, а не хвататься за первый пришедший в голову. |
| **Общий язык команды** | Метод даёт дизайнеру, заказчику и разработчику одни и те же истории и форму для общего разговора. |

> Родственный метод — Карта процесса-опыта (XPM): отвечает на «как устроен процесс», тогда как КРИ — на «что именно разрабатывать». См. `docs/XPM/README.md` и `docs/SIM/README.md`.
