# Примеры досок Связанного SWOT (golden references)

Эталоны для калибровки стиля и проверки «на что похоже хорошо». Автор метода — **Александр Бындю** ([byndyu.ru](https://byndyu.ru)); материалы-первоисточники под AGPL-3.0 — здесь ссылки и пересказ, не копии. Для коммерческого использования метода обращайтесь к автору.

## Разобранный пример (наш, для новичка)

- **Репетитор Анна** — сквозной разбор всех трёх шагов (факторы → ставки со связями и отметками → переход в Карту гипотез): `docs/LINKED-SWOT/primer/skvoznoy-primer.md`. Именно этот кейс отрисован в JSON в `resources/render_excalidraw.md`.

## Учебные материалы метода (docs/LINKED-SWOT)

| Файл | Чем полезен как образец |
|---|---|
| `docs/LINKED-SWOT/osnovy/oblasti.md` | эталон формулировок карточек по областям + цвета |
| `docs/LINKED-SWOT/osnovy/stavki.md` | эталон формата ставки, связей многие-ко-многим, баланса |
| `docs/LINKED-SWOT/postroenie/shag-1-faktory.md` | заполненные четыре области (репетитор), приоритет позицией |
| `docs/LINKED-SWOT/postroenie/shag-2-stavki.md` | две ставки со связями и отметками, непокрытые факторы |
| `docs/LINKED-SWOT/postroenie/shag-3-svyaz-s-kartoy-gipotez.md` | ставка → гипотеза / ставка → цель, обратный ход |
| `docs/LINKED-SWOT/kachestvo/tipovye-oshibki.md` | «как не надо → как надо» — источник hard-гейтов |

## База знаний автора

- **Репозиторий метода** — [github.com/Byndyusoft/linkedswot](https://github.com/Byndyusoft/linkedswot): описание нотации, примеры досок, шаблоны.
- **Соседний метод (куда уходят ставки)** — [github.com/Byndyusoft/hypothesismapping](https://github.com/Byndyusoft/hypothesismapping) (Карта гипотез); навык `hypothesis-map`.
- **Сайт автора** — [byndyu.ru](https://byndyu.ru).

## Шаблоны и инструменты для рисования

| Инструмент | Где | Для чего |
|---|---|---|
| **Холст** | `app.holst.so` | ручной шаблон доски (3 колонки, карточки, связи) — основной fallback |
| **Социотех** | `sociotech.center` | официальный инструмент семейства методов Byndyusoft |
| **Excalidraw MCP** | `create_view` | автоматический рендер из навыка — рецепт в `resources/render_excalidraw.md` |

> При калибровке стиля вывода: доска должна читаться без знания нотации — три зоны по цвету, ставки в центре, ниточки-связи слева/справа, приоритет высотой. Ценность не в списках факторов, а в **связях**: ставка, соединяющая внутренний факт с внешним и дающая шаг развития.
