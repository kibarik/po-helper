# Глоссарий Карты гипотез

Термины метода (автор — Александр Бындю, byndyu.ru). Пересказ своими словами; человеческая версия с бытовыми примерами — `docs/HYPOTHESIS-MAP/osnovy/slovar.md`.

| Термин | Значение |
|---|---|
| **Карта гипотез** | Визуальная нотация стратегии: связка Цель → Субъект → Гипотеза → Задача со стрелками влияния. Делает стратегию видимой и прослеживаемой от цели до задачи. Не программа — инструмент, чтобы людям вместе думать и договариваться. |
| **Стратегия** | Набор **явных гипотез** о том, как из «сейчас» попасть в «цель». Цель + список задач без проговорённых идей — ещё не стратегия. |
| **Цель** | Куда идём, описанное словами (без цифр — цифры в метриках), часто со сроком. |
| **Метрика** | Параметр с числом, показывающий движение к цели; всегда текущее **и** желаемое значение (виден размер шага). Не «галочка». |
| **Опережающая метрика** | Меряется постоянно по ходу дела; заранее подсказывает, идём ли к цели. Приборная панель. |
| **Запаздывающая метрика** | Итог, виден только в конце периода. Фото на финише. |
| **Балансирующая метрика** | То, что не двигаем, а **удерживаем в рамках**, чтобы, гонясь за целью, не сломать важное (CAC, маржа, ФОТ). |
| **Субъект** | Живой человек (или группа) с самодвижением, чьё поведение мы меняем ради цели. Сам решает, как себя вести, нам не подчиняется. **Не** исполнитель нашей задачи. |
| **Мотивация (боли и желания)** | Что субъекта мучает и чего он хочет. На неё опирается «потому что» гипотезы. Копают до корня («5 раз ну и что?»). |
| **Негативный субъект** | Тот, кто намеренно против: переманивает ваших субъектов (конкурент) или перехватывает управление («крот»). Против него — гипотезы, снижающие его влияние. |
| **Гипотеза** | Формализованная идея по шаблону «Если… то… потому что… тогда…»: каким действием и почему поменяем поведение субъекта и качнём цель. Ставка «предложить и проверить», не финальная истина. |
| **Ветка** | Связь гипотезы с задачами (и с субъектом/целью). Задача без ветки в работу не берётся. |
| **Задача** | Конкретное дело, которым проверяем гипотезу. Масштаб — от «добавить кнопку» до «построить завод». Всегда прицеплена к ветке. |
| **Блокер** | Что мешает двигаться: причина · что сделать · ответственный · дедлайн. |
| **Заметка** | Свободная запись фасилитатора по ходу сессии, чтобы не потерять деталь. |
| **Приоритизация** | Расстановка «что первым»: вес влияния × стоимость проверки. Показывается цветом и толщиной стрелки. |
| **Приоритетный путь** | Заметная «красная дорожка» из самых важных веток; по ней идут первыми. |
| **Топология** | «Готовая расстановка» элементов под ситуацию (классическая, многоуровневая, метрики/мотивация на уровне и т. д.). См. `resources/topologies.md`. |
| **Набегающая волна** | Способ собирать карту не за один раз, а в несколько заходов, каждый раз уточняя (грубый черновик → детали → ещё круг). |
| **Каденция (страткаденция)** | Ритм управления стратегией: регулярные встречи, где сверяют прогресс, проверяют гипотезы (сработала/нет и почему) и правят карту и приоритеты. Отвечает Владелец карты. |
| **Владелец карты (Заказчик)** | Ради кого всё: подтверждает цели, сверяет со стратегией компании, инвестирует, **отвечает за реализацию**. |
| **Фасилитатор-технолог** | Ведущий сессии, держит метод и нотацию. Роль этого навыка. |
| **Технолог** | Хранитель правил метода (теория не ниже практика 2 ступени). |
| **Эксперт** | Знаток предметной области; без него не сформулировать хорошие гипотезы. |
| **Исполнитель** | Тот, кто потом проверяет гипотезы. Вовлечение в обсуждение — половина пользы карты. |
