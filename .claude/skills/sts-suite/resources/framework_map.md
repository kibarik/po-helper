# Карта фреймворка проектирования социотехнических систем (Byndyusoft)

Как методы связаны в единый Discovery-конвейер. Авторы: Александр Бындю (Связанный SWOT, Карта гипотез) и Андрей Шапиро (XPM, SIM). Пересказ своими словами с атрибуцией.

---

## Конвейер: результат одного = вход другого

```
┌ АНАЛИЗ ───────────┐   ┌ СТРАТЕГИЯ ────────┐   ┌ ПРОЕКТИРОВАНИЕ ───────────────┐
│ Связанный SWOT    │──▶│ Карта гипотез     │──▶│ Карта процесса-опыта (XPM)    │
│ /linked-swot      │   │ /hypothesis-map   │   │ /xpm-writer   (процесс+опыт)  │
│ → стратег. ставки │   │ → цели/субъекты/  │   │ Карта реализации историй (SIM)│
│                   │   │   гипотезы/задачи │   │ /sim-writer   (истории→треб.) │
└───────────────────┘   └───────────────────┘   └───────────────────────────────┘
        Бындю                   Бындю                       Шапиро
                                  │                              │
                                  ▼                              ▼
                          дорожная карта / HADI          детальное проектирование
```

## Роль каждого метода

| Метод | Отвечает на вопрос | Главный элемент | Скилл | Человеческая методичка |
|---|---|---|---|---|
| Связанный SWOT | «Какие у нас факторы и за счёт чего расти?» | стратегическая ставка | `linked-swot` | `docs/LINKED-SWOT/` |
| Карта гипотез | «Зачем и какими идеями?» | гипотеза (Если/то/потому что/тогда) | `hypothesis-map` | `docs/HYPOTHESIS-MAP/` |
| XPM | «Как устроен процесс и опыт?» | ключевая точка | `xpm-writer` | `docs/XPM/` |
| SIM | «Кто как действует → что разрабатывать?» | рабочая история (Ц/Н/С/Д/О→О′) | `sim-writer` | `docs/SIM/` |

---

## Петли обратной связи

- **Нет гипотезы?** → назад в Связанный SWOT: заполнить области, поискать ставку на пересечении.
- **Нет инструмента под точку XPM?** → детальное проектирование / рабочие истории SIM.
- **Стратегические каденции** (Карта гипотез) → регулярно сверять реальность с картой, обновлять.

## Общий инструмент-реализация

Все четыре метода поддержаны в сервисе **Социотех** (`sociotech.center`) — единый инструмент автора. Рендер схем в этом наборе скиллов — через **Excalidraw MCP** (см. `render_excalidraw.md` каждого скилла).

---

## Сквозные принципы набора (зеркало bft-writer)

Ноль галлюцинаций (каждый элемент ← источник) · модальность «как есть/как будет» · метод «набегающей волны» · каталог типовых ошибок = hard-гейты · STOP-паузы · соединение воедино · атрибуция + AGPL (пересказ, не копирование).
