# /diagram-view — диаграмма прямо в чате

Берёт PlantUML-исходник и рендерит его как inline-SVG в чат через `mcp__visualize__show_widget`.
Не нужен внешний plantUML-редактор — диаграмма видна рядом с Summary.

## 1. Найти исходник
Источник PlantUML (в порядке приоритета):
1. Явно переданный пользователем текст / путь к `.puml`.
2. `@startuml … @enduml` блок в открытом/упомянутом `.md` (в т.ч. внутри ```` ```plantuml ```` фенса).
3. Если в документе **несколько** блоков — отрендерь каждый отдельным виджетом, в порядке появления; озаглавь по `title`/первому имени.

Нет блока `@startuml` → СТОП, спроси у пользователя путь/текст. Не выдумывай диаграмму.

## 2. Рендер в SVG (локально)
Запиши исходник как есть в `/tmp/diagram-view-<n>.puml` и прогони:
```bash
plantuml -tsvg -charset UTF-8 /tmp/diagram-view-<n>.puml 2>&1
```
- Успех → создаётся `/tmp/diagram-view-<n>.svg`. Прочитай его (Read).
- `plantuml: command not found` → разовая установка `brew install plantuml` (нужна Java). Сообщи и СТОП — это окружение, не ошибка диаграммы.
- Вывод содержит `Error` / `syntax error` → это ошибка **синтаксиса PlantUML**. Покажи строку из лога, предложи правку, не рендерь битый виджет.

PlantUML-SVG самодостаточен (свои цвета, viewBox) — переписывать под CSS-переменные виджета не нужно.

## 3. Показать в чате
Вызови `mcp__visualize__show_widget`:
- `widget_code` = **содержимое SVG целиком** (начинается с `<svg …>`; авто-детект SVG-режима).
- `title` = snake_case по смыслу диаграммы (напр. `components_vecher_pod_kluch`).
- `loading_messages` = 1–2 коротких.

Несколько диаграмм → несколько вызовов `show_widget`, по одному на блок.

## 4. Текст-ответ
Виджет несёт картинку. В текст вынеси: 1 строку что это за диаграмма + (если есть) 2–3 пункта ключевых связей/нот из исходника. Не дублируй весь PlantUML.

## Guardrails
- Только рендер существующего исходника — не достраивай и не «улучшай» диаграмму без явной просьбы.
- Ошибка синтаксиса ≠ недоступность рендерера: не выдавай одно за другое (см. §2).
- Исходник пишется в `/tmp` дословно (UTF-8), кириллица сохраняется через `-charset UTF-8`.
- Большие диаграммы (>~30 узлов) рендерятся, но предупреди что виджет будет крупным; при желании пользователя — разбей на под-диаграммы.
