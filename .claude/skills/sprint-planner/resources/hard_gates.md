# Hard Gates — валидация плана спринта (8 бинарных гейтов)

Прогоняются в `/sprint-deliver` перед сухим прогоном. Каждый гейт — бинарный: ✅ / 🔴. При одном 🔴 в критичных — финал 🔴, запись блокируется.

## Светофор

| Статус | Условие | Действие |
|--------|---------|----------|
| 🟢 Зелёный | Все 8 гейтов ✅ | сухой прогон → запись после «ок PO» |
| 🟡 Жёлтый | Нарушены только гейты 6–8 (некритичные) | запись с явными `[УТОЧНИТЬ]`, PO принимает риск |
| 🔴 Красный | Нарушен хотя бы один гейт 1–5 | вернуться на `/sprint-load` или `/sprint-decompose` |

---

## Критичные гейты (1–5) — блокирующие

### Гейт 1: У каждой цели есть Sprint Goal
**Правило:** Каждая OBJ-секция плана содержит «🎯 Образ результата команды» — доказуемое состояние к концу спринта, привязанное к KR.
**Нарушение:** Пустой Sprint Goal или формулировка-активность («поработаем над…»).

### Гейт 2: Каждая история ← KR или помечена внеплановой
**Правило:** История либо наследует KR из SPRINT-ROADMAP, либо стоит в секции «Внеплановые работы» с тегом (`[TECH-DEBT]`/`[BUG]`/`[UNPLANNED]`/`[SPIKE]`/`[CARRYOVER]`).
**Нарушение:** История без KR и без тега внеплановой — «сирота».

### Гейт 3: Нет TBD-исполнителя в Must
**Правило:** Все Must-истории имеют исполнителя из ростера. `TBD` допустим только в Should/Could.
**Нарушение:** Must-история с `Исполнитель: TBD`.

### Гейт 4: Никто не перегружен
**Правило:** Сумма SP каждого исполнителя ≤ его ёмкости спринта (velocity × focus factor − отпуска).
**Нарушение:** Хотя бы один исполнитель 🔴 перегружен. Действие: снять Should/Could или перераспределить.

### Гейт 5: Образ результата заполнен у каждой Must-истории
**Правило:** Ни одного `TBD`/пустого в «Образ результата» для Must-историй. Для технических — `БЫЛО→СТАЛО`.
**Нарушение:** Must-история без доказуемого образа результата.

---

## Некритичные гейты (6–8) — 🟡 допустимы

### Гейт 6: Никто не недогружен
**Правило:** Загрузка каждого исполнителя ≥ 70% ёмкости.
**Нарушение:** Исполнитель 🟡 недогружен → добрать историю из бэклога/N+1/внеплановых.

### Гейт 7: Carryover учтён
**Правило:** Незакрытые истории прошлого спринта (ФАКТ) либо перенесены с тегом `[CARRYOVER]`, либо явно закрыты/сняты с обоснованием.
**Нарушение:** ФАКТ показывает незакрытые истории, не отражённые в плане.

### Гейт 8: Блок N+1 присутствует
**Правило:** В документе есть секция «🔭 Спринт N+1 (предварительно)» с образом результата и крупными историями.
**Нарушение:** Нет горизонта следующего спринта.
