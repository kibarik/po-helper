# Build Gates — гейты BUILD-прохода (запись в JIRA, необратимо)

Прогоняются в `/sprint-build` и `/sprint-activate`. BUILD необратим → каждый гейт блокирует запись до явного решения PO. Принцип: **человек — на развилках, автономия — внутри стадии**.

## Предусловия BUILD

| # | Условие | Нарушение → действие |
|---|---------|----------------------|
| P1 | `tracker.access: build` в профиле | `read-only` → BUILD запрещён, остаться в PLAN |
| P2 | MCP трекера поднят | нет MCP → СТОП, поднять MCP/VPN (не искать обход) |
| P3 | PLAN-проход завершён (есть vault-ПЛАН + матрица покрытия) | нет → вернуться в `/sprint-deliver` |
| P4 | Метаданные проекта резолвлены (issuetype id, epic_link_field, назначаемые учётки) | пусто → discovery в `/sprint-sync`, не гадать id |

## Гейты записи

### 🚪 G1 — reuse-before-create
**Правило:** ни одной NEW-задачи, пока не проверен беклог (`text ~`, все `tracker.projects`). Создаём NEW только если активного дубля нет.
**Нарушение:** create без reuse-матрицы из `/sprint-decompose`.

### 🚪 G2 — slot-safety
**Правило:** писать только в ПУСТОЙ или явно выбранный PO слот. Не репойнтить непустой слот.
**Нарушение:** запись в слот с существующими задачами без явного «да, в этот» от PO → СТОП.

### 🚪 G3 — cascade-preview
**Правило:** перед добавлением эпика/родителя в спринт — показать, сколько детей подтянется, и получить ок.
**Нарушение:** массовое добавление без предпросмотра каскада.

### 🚪 G4 — dry-run → approve → write → verify
**Правило:** каждый батч записи предваряется обозримым сухим прогоном и подтверждением PO; после записи — верификация чтением.
**Нарушение:** запись без сухого прогона или без верификации батча.

### 🚪 G5 — zero-orphans
**Правило:** после группировки — 0 орфанов. Орфан = не подзадача, `Epic Link` пуст, не эпик. Каждая задача под историей/эпиком и под измеримой поставкой.
**Нарушение:** остались задачи без Epic Link и не подзадачи.

### 🚪 G6 — activation-gate
**Правило:** `Start sprint` — **отдельным явным подтверждением**, не «заодно» с записью.
**Нарушение:** активация спринта в том же батче, что и наполнение.

## Антипаттерны (стоп-лист)

- Репойнт непустого слота.
- Слепое доверие путям из CLAUDE.md (проверять реальную папку под git).
- Запись артефактов в вычищаемый путь (worktree/tmp).
- `create` до reuse-check.
- Гадание issuetype-id вместо discovery.
- Назначение исполнителя без валидации учётки/роли.
- `Start sprint` «заодно».
- Paste токена в чат.
- Дрейф имени/дат спринта.
- Конвертация существующей истории в подзадачу через REST (нельзя — только Move-визард UI; проектировать через Epic Link).
