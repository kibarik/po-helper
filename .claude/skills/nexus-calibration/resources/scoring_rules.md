# Правила разметки и гейт калибровки

## Оракул — PO

Org-реальность не лежит в коде или трекере; её знает PO. Поэтому разметку ставит PO, а не LLM. LLM-выборка — то, что проверяется; разметка PO — истина.

## Бинарная разметка

По каждому вопросу PO ставит ровно одну метку:

| Метка | Когда |
|---|---|
| **✅ совпало** | Ответ LLM (кто) совпал с реальным адресатом, и «почему» не противоречит реальности |
| **❌ расходится** | Адресат не тот; **или** адресат тот, но по неверной причине; **или** LLM сказал `[нексус молчит]`, а адресат в реальности есть; **или** LLM назвал человека, а в реальности `[нексус молчит]` (нет такого адресата) |

Полутонов нет. «В целом верно, но порядок не такой / забыл подключить X / причина натянута» = **❌**: нексус неточен, и это надо чинить. Частичный зачёт маскирует дыру.

### Что считать совпадением по типам

- **Тип A (один адресат):** node_id совпал. Если вопрос допускает двух равноправных — PO засчитывает `✅`, если LLM назвал любого корректного и обосновал.
- **Тип B (порядок действий):** `✅` только если и состав адресатов, и порядок/эскалация совпали с тем, как реально пошёл бы запрос. Пропущен ключевой человек или перепутана эскалация → `❌`.
- **Негативный кейс:** `[нексус молчит]` = `✅`, если адресата в реальности правда нет. Если LLM промолчал, а человек есть → `❌` (пробел графа).

При `❌` PO **обязан** одной фразой дать правду — это вход для `/radar-review`:

```
| 7 | ❌ | на самом деле → Сидорова: платёжки с Q1 ведёт она, у Петрова только инфра |
```

Без фразы-правды `❌` нельзя разобрать. Команда `/radar-calibrate` явно просит PO дописать правду к каждому `❌`.

## Гейт: N/N, без частичного зачёта

Калибровка **пройдена** только при всех совпадениях:

- Короткая: **10/10**.
- Длинная: **50/50**.

`9/10`, `49/50` — **не пройдена**. Никакого «90% качества». Один `❌` → цикл продолжается: `/radar-review` → correction-Prompt → правка узлов → новый прогон.

## Светофор прогона

Метка статуса прогона (не замена гейту, а сводка для PO):

| Светофор | Условие | Действие |
|---|---|---|
| 🟢 | `N/N` совпало | КАЛИБРОВКА ПРОЙДЕНА. Цикл закрыт. |
| 🟡 | 1–2 `❌` (для short) / ≤5 `❌` (для long) | близко; `/radar-review`, точечные правки, повтор |
| 🔴 | больше | граф системно расходится; `/radar-review` + ревизия наполнения (возможно, нужен `/paf-onboard`, а не точечные правки) |

🟡/🔴 — оба «не пройдено», разница лишь в масштабе правок: 🟡 чинится correction-Prompt'ом, 🔴 может потребовать пере-наполнения нексуса.

## Анти-паттерны разметки

1. **Самооценка LLM.** LLM не ставит себе `✅`. Метку ставит только PO. Если PO не разметил — прогон не засчитан, не додумывай за него.
2. **«Зачёт по духу».** Адресат «почти тот» = `❌`. Роутинг либо попадает, либо нет.
3. **Молчаливое исправление.** `/radar-calibrate` не правит вопросы и не пересдаёт их за PO. Разметка → STOP → `/radar-review`.
