## Вариант с линзой `both` — Тип C и routable-gap

При `/radar-calibrate short --lens both` часть слотов — вопросы Типа C (context/RAG), с колонками `Линза` и `routable?`:

| # | Вопрос | Тип · линза | Ответ LLM (кто · почему) | routable? | PO |
|---|---|---|---|---|---|
| C1 | «Кто помнит, почему два года назад выбрали именно этого эквайера?» | C · context | team-sidorova-anna · `context_holds` (история платёжных интеграций) | **нет** — в `contact_for`/`expertise_topics` этого нет | ✅ |

`routable? = нет` → **routable-gap**: знание в графе есть (в `context_holds`), но ИИ-роутер его не адресует. Под линзой `both` gap считается как `❌` для гейта; фикс — перенести факт в `contact_for`/`expertise_topics` (см. `correction_prompt.md` §routable-gap). Это ровно паттерн расхождений #18 (аудит PCI) и #44 (замещение PM) из длинного прогона.
