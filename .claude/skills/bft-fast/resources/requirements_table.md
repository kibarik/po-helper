# Вложение — таблица требований на согласование границ (requirements_table.md)

Второй артефакт `/bft-fast` (после письма): **сводная таблица ключевых требований** в структуре БФТ (БТ/ПТ/ИТ/ФТ/НФТ). Назначение — согласование границ задач.

**Главное отличие от полного БФТ:** таблица содержит **только то, что сказали люди на встрече**. Каждая строка формируется из **цитаты** участника. Никакого обогащения, порогов, приоритетов, выдуманных параметров — это скелет на цитатах; детальная проработка и обогащение происходят на следующем этапе (deep `bft-*`).

## Колонки

| Колонка | Что | Правило |
|---|---|---|
| Тип | `БТ` / `ПТ` / `ИТ` / `ФТ` / `НФТ` | по различению типов (как в bft-writer/bft_standards) |
| Идентификатор | `{ТИП}-{N}` (локальный) | `БТ-1`, `ПТ-1`, … — якорь внутри таблицы |
| Название | 2-4 слова | суть требования |
| Требование | одна фраза | что именно, из цитаты; без обогащения |
| Связанные | ID других строк | каскад `БТ ← ПТ ← ИТ`, `БТ ← ФТ ← НФТ`; только где связь видна из встречи |
| Источник (цитата) | транскрипт → `ФИО: «дословно или близко к тексту»`; `#summary`-заметка → `Summary: «строка заметки»` | обязателен; без цитаты/строки-источника строки нет |

## Различение типов (кратко)

- **БТ** — ценность/зачем (бизнес-требование).
- **ПТ** — что хочет пользователь (story «когда/хочу/чтобы» можно сжать до фразы).
- **ИТ** — что видит/делает актор на экране (интерфейс приёмки), не API.
- **ФТ** — поведение системы на уровне результата.
- **НФТ** — измеримое -ility, **но только если озвучено** (не выдумывать пороги вроде «≤2000ms»).

## Правила грунтования (zero hallucination)

1. **Строка ← цитата.** Нет реплики участника, из которой прямо следует требование — строки нет. Колонка «Источник» обязательна и непуста.
2. **Без обогащения.** Не добавлять пороги, приоритеты, параметры, негативные сценарии, если их не проговорили. Это работа следующего этапа.
3. **Тип по факту.** Включать тип, только если под него есть цитаты. Часто НФТ/ИТ на discovery-встрече пусты — тогда их в таблице нет, не добивать.
4. **Цитата верна.** Дословно или максимально близко к транскрипту; ASR-искажения чинить осторожно, смысл не менять. Атрибуция — по говорящему.
5. **out-of-scope не в таблицу.** Что отнесли к «не делаем»/следующему кварталу — не требование этой итерации (уходит в «Границы» письма, не в таблицу).

## Форма вывода

Отдельным блоком после письма, с заголовком-подписью:

```
Вложение — таблица требований на согласование границ:

| Тип | ID | Название | Требование | Связанные | Источник (цитата) |
|---|---|---|---|---|---|
| БТ | БТ-1 | … | … | … | ФИО: «…» |
| ПТ | ПТ-1 | … | … | БТ-1 | ФИО: «…» |
…

Только зафиксированное на встрече. Пороги/приоритеты/детализация — на этапе глубокой проработки.
```

Таблица — часть вложения, отдельно от тела письма (письмо остаётся самодостаточным для пересылки).
