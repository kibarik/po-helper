# Схема извлечения из Summary (extraction_schema.md)

Конвейер `/bft-fast` извлекает из Summary набор сущностей. Каждая сущность несёт `source` — цитату/строку из Summary. Сущность без источника не создаётся: неподтверждённое уходит в «Открытые вопросы» (No Silent Skip).

## Поля

| Поле | Что это | Источник в Summary | Если пусто |
|---|---|---|---|
| `проблема` | бизнес-разрыв, который закрываем | блок «Тема»/«Проблема» | вопрос «в чём проблема?» |
| `для_кого` | пользователь/роль, для кого результат | «Тема», участники | вопрос «для кого?» |
| `что_делаем` | список действий/поведения системы | блок «Что делаем» | письмо тонкое → всё в вопросы |
| `участники[]` | ФИО + роль (владельцы вопросов) | frontmatter `участники`, упоминания | `[кому?]` при атрибуции |
| `договорённости[]` | зафиксированные решения, сроки, границы | блок «Договорённости» | нет заметок/границ |
| `затронутые_сущности[]` | объекты, которых касается работа (карточки, страницы, блоки) | «Что делаем» | демо не строится → вопрос |
| `out_of_scope_hints[]` | что явно вне («не входит», «не делаем») | «Договорённости»/«Границы» | «Границы» без out-of-scope |
| `открытые_точки[]` | нерешённые развилки/вопросы, уже перечисленные во встрече | блок «Открытые точки»/нерешённые пункты | вопросы выводятся только из фактов без источника (гейт) |
| `дизамбигуации[]` | явные семантические границы факта: квалификатор «X, не Y», момент/условие, референс-стандарт | «оплачен, не создан»; «возврат завершён, не подан»; «как передают распы по АИС» | шаг/факт без уточнения — не выдумывать границу |
| `optional_scope_hints[]` | что делаем «дополнительно, если возможно» — nice-to-have, не must-have | «дополнительно», «если получится», «опционально», «желательно» | нет тира «если возможно» |

## Точность (дизамбигуации)

Главный рычаг точности письма. Если в Summary факт несёт **явную семантическую границу** — уточняющий квалификатор, точный момент/условие срабатывания или референс-стандарт — она извлекается вместе с фактом и не теряется при синтезе:

- Квалификатор «X, не Y»: «публикуется, когда заказ **оплачен** (не создан)».
- Точный момент/условие: «возврат — когда **подтверждён и завершён**, не когда клиент подал заявку».
- Референс-стандарт: «маскированные данные, **как передают распы по АИС**».

Дизамбигуация цепляется к своему факту (демо-шаг или строка границ), НЕ выносится отдельным пунктом. Нет дизамбигуации в Summary — не выдумывать (гейт галлюцинаций).

## Правила якорения (ранги как в bft-writer/anchor_rules)

- Факт As-Is/поведение ← строка Summary (R2). Решение PO ← пометка в Summary (R3).
- Имя участника сверяется с ростером `capacity.roster_source` (`GROUND/NEXUS/team`) для полного ФИО/роли; не сматченное — как в Summary, не додумывать.
- Транскрипт (не `#summary`): сначала внутренняя выжимка до эквивалента `#summary` (логика навыка `summary`), затем извлечение по этой схеме.

## Атрибуция владельца вопроса

Открытая точка → вопрос → владелец = участник, чья роль соответствует теме точки (роутинг тема↔роль). Если в Summary точка уже адресована («уточнить у X») — владелец = X. Неоднозначно → префикс `[кому?]`. Владелец — префикс строки вопроса (`* <ФИО> — <вопрос>`), не заголовок группы (см. letter_format.md).
