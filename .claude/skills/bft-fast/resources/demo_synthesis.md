# Синтез блока «How to demo» (demo_synthesis.md)

`How to demo` — нумерованный E2E-сценарий демонстрации от лица ключевой роли, принимающей результат. Он же ограничитель scope: что демонстрируем, за то и отвечаем.

## Правила

1. **От лица ключевой роли, операционные глаголы:** шаги пишутся от лица условной роли проверяющего/принимающего демо (напр. «PO площадок», «проверяющий» — берётся из `участники`/контекста Summary как тот, кто логично принимает результат). Операционные шаги — от 1-го лица этой роли, где это естественно: «открываю», «добавляю», «вижу», «проверяю». Если так яснее, кто действует, роль можно назвать в начале демо. Не «система отображает» как формулировка всего сценария — но см. пункт 4: наблюдательные шаги для крайних случаев допустимы и грамматического «я» не требуют.
2. **Порядок — путь принимающего демо:** вход в интерфейс → действие → наблюдаемый результат → крайние случаи (пустое состояние) → внешняя верификация.
3. **Каждый шаг ← факт** из `что_делаем`/`договорённости`/`затронутые_сущности` (extraction_schema.md). Шаг без источника не выдумывается.
4. **Крайние случаи как шаги:** пустое состояние, скрытые элементы, «работает как обычно» — если есть в Summary. Такие шаги допустимо формулировать наблюдательно (без грамматического «я» в каждом слове) — это не нарушение правила 1, а естественная форма для констатации состояния системы, а не действия роли.
5. **Последний шаг — верификация результата** (внешняя: поиск, индексация, интеграция), если она есть в Summary. Нет данных о верификации → шаг опускается, не выдумывается.
6. **Только in-scope:** сущности из `out_of_scope_hints` в демо не попадают — они уходят в блок «Границы».

## Форма

```
[Роль (опционально, если проясняет кто действует):]
How to demo:
1. <вход в интерфейс> → <что вижу>.
2. <действие(я)>.
3. <публичный/итоговый результат>.
4. <крайний случай / пустое состояние>.
5. <внешняя верификация, если есть>.
```

## Анти-паттерны

- Демо-шаг про то, чего нет в Summary (выдумка) — запрещено.
- Раздутое демо «на всякий случай» — теряется scope-эффект. Только реально согласованное.
- Технические детали реализации (API, БД, код) — не сюда (высота приёмки, не СА).
