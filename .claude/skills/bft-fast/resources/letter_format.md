# Формат письма (letter_format.md)

Результат `/bft-fast` — одно письмо в чат, копируемое в мессенджер как есть. Три блока строго в порядке ниже.

## Шаблон

```
How to demo:
1. …
2. …
(нумерованный E2E — см. demo_synthesis.md)

Открытые вопросы:
<ФИО стейкхолдера>:
- [ ] <конкретный вопрос-развилка>
- [ ] …
<ФИО второго стейкхолдера>:
- [ ] …
[кому?]:
- [ ] <вопрос без явного владельца>

Границы: <in-scope сущности> (<out-of-scope> — не входят в зону БФТ).
<операционная заметка: сроки/эффекты>
```

## Правила сборки

- **Порядок блоков фиксирован:** `How to demo` → `Открытые вопросы` → `Границы`.
- **В теле письма нет frontmatter, меток пайплайна, идентификаторов `СТ-*`/`ФТ-*`.** Только человекочитаемый текст, копируемый в мессенджер как есть.
- **Вопросы — чекбоксы `- [ ]`**, сгруппированы по ФИО; блок `[кому?]` — только если есть неатрибутированные вопросы (когда владелец вопроса не определён).
- **«Границы»:** одна строка in-scope сущностей + явное «не входят в зону БФТ» для out-of-scope; операционные заметки (сроки, эффекты, интеграции) — отдельными строками. Если в Summary нет явного out-of-scope — только in-scope + заметки, без пустых скобок.
- **Голос** — по `../../bft-writer/resources/writing_style.md`: деловой русский, без эмодзи, без раздутого bold, без тире-связок, без стоп-слов, без AI-примет. Регистр совпадает с золотыми референсами.

## Гейт перед выводом (hallucination gate — СТ-23)

Пройти по каждому утверждению письма:
- Есть источник в Summary (факт, договорённость, затронутая сущность) → оставить.
- Нет источника → переоформить в вопрос (`[кому?]: - [ ] <переформулированное как вопрос>`).

**Правило:** ни одного факта-выдумки. Если что-то уверенно звучит, но нет цитаты в Summary — это не опускается молча, а становится вопросом.

## Канал вывода

- **Основной канал:** текст письма в чат PO (копируется как есть, без служебных маркеров).
- **Опциональная копия:** файл `{bft_store}/{epic_slug}/letter.md` (создать папку при отсутствии). Если в профиле нет `bft_store` → файл рядом с источником с пометкой `[УТОЧНИТЬ путь в bft_store]`.
- **После письма:** короткая строка PO: «Ответы получены — запусти глубокую проработку отдельно». Сам deep-запуск команда не делает.
