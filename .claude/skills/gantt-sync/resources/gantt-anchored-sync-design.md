# GANTT-anchored синхронизация GANTT ↔ OKR ↔ JIRA — механика

> Проектно-нейтральное описание read-model. Конкретные пути/проекты/якоря — из `domain-profile.md`.

## 1. Цель и проблема

GANTT в GanttPRO (крупные треки квартала, владеет один человек) — стратегический источник истины. OKR (KR) и JIRA-эпики должны быть **производны от GANTT и сверяться с ним**.

Типовая проблема: GANTT (крупные треки) и JIRA (детальные эпики) не мапятся 1:1, планировщик путается, расхождения не видны. Решение формализует правило соответствия и регулярную сверку.

Решение — **read-model**: GANTT остаётся скелетом (правит только владелец в GanttPRO), автоген джойнит поверх него KR и live-статусы JIRA и подсвечивает дрейф покрытия. Ничего не пишем в GANTT и (в этой итерации) в JIRA — только читаем и сверяем.

## 2. Модель высот (GANTT = корень)

| Высота | GANTT | JIRA | OKR | Сверяем? |
|--------|-------|------|-----|----------|
| Верх | **свимлейн** | Инициатива (правило); крупный Эпик (исключение) | Цель | да |
| Середина | **карточка** (строка) | **Эпик** | **KR** | да — якорь |
| Низ | опц. / незаметно | История / Баг | — | нет |

- Якорь сверки — **карточка GANTT** (строка xlsx). Всё вешается на неё.
- Середина стремится к 1:1: `карточка ≈ KR ≈ Эпик`. Карточка может тянуть несколько эпиков (mapping хранит список).
- Свимлейн → Инициатива JIRA как правило; изредка крупный Эпик без инициативы-обёртки (mapping допускает оба).
- История/Баг — нижний уровень; в GANTT может отсутствовать. Для сверки не используется.

## 3. Ключ связи — курируемый mapping

Строки GANTT не имеют ID → нужен явный якорь. Эвристику по именам отвергаем: формулировки GanttPRO и Epic Name расходятся.

Ключ связи ведётся в `kr_epic_map_doc` (KR-EPIC-MAP): колонка-якорь **gantt-card-slug** на каждой строке KR + раздел «GANTT свимлейн → Инициатива».

Формат строки KR-таблицы:

```
| KR | Название (≈ карточка GANTT) | GANTT-цель | Эпики JIRA | gantt-slug |
```

- `gantt-card-slug` — стабильный id карточки, присваивается PO один раз, не меняется при переформулировке в GanttPRO.
- `KR` — номер KR (или пусто → дрейф: карточка без целеполагания).
- `Эпики JIRA` — список ключей (или пусто → дрейф). Суффикс `(?)` = связь под вопросом.

Раздел свимлейнов:

```
## GANTT свимлейн → Инициатива
| Свимлейн (xlsx) | Инициатива JIRA |
```

## 4. Входы и конвейер

1. **GANTT** — `gantt_source_xlsx` (экспорт GanttPRO). Единственный источник карточек + свимлейнов + фаз-спринтов. Владелец кидает свежий экспорт — файл заменяется.
2. **mapping** — `kr_epic_map_doc` (карточка → Цель/Инициатива → KR → эпики).
3. **JIRA live** — открытые Инициативы + Эпики проектов `tracker.projects`, их типы/статусы; собираются через MCP.

Конвейер:

```
parse xlsx        → cards[] (по строкам: свимлейн/карточка, спринты, фазы)
load mapping      → card→{initiative, kr, epics, slug}
load issues.json  → живые инициативы + эпики (ключ, тип, статус)
join + diff       → покрытие на двух высотах
render            → GANTT-SYNC-STATUS.md
```

Парсер xlsx хрупок к смене шаблона GanttPRO-экспорта. Митигация: проверяет ожидаемые опорные строки (`gantt.anchors`) и падает с внятной ошибкой, если структура уехала, а не выдаёт тихо мусор.

## 5. Правила дрейфа (только покрытие)

Подсвечиваются разрывы связей на двух высотах. Статус↔фаза (тайминг) и rollup-прогресс — вне скоупа.

**Верх (свимлейн ↔ Инициатива):**
- ⚠ свимлейн GANTT без Инициативы JIRA
- ⚠ открытая Инициатива JIRA без свимлейна GANTT

**Середина (карточка ↔ KR ↔ Эпик):**
- ⚠ карточка GANTT без Эпика JIRA (пробел планирования)
- ⚠ карточка GANTT без KR (нет целеполагания)
- ⚠ открытый Эпик JIRA без карточки GANTT (scope creep / вне стратегии)
- ⚠ KR OKR без карточки GANTT (не в плане квартала)

Эпик, привязанный к KR в mapping (даже без карточки GANTT), сиротой не считается.

**Низ (История):** не сверяем.

## 6. Формат зеркала

Дерево по GANTT сверху вниз (свимлейн → карточки со спринтами, KR, эпиками), затем плоский список «## Дрейф» (чеклист для разбора) и «## Сводка». Флаги: ✅ покрыто · ⚠ разрыв. Образец — `examples/GANTT-SYNC-STATUS.example.md`.

## 7. Владение и кадэнс

- GANTT правит владелец в GanttPRO; сверка — на планировщике/PO.
- Рутина: владелец обновил xlsx → прогон `/gantt-sync` → разбор `## Дрейф` → правки в JIRA/OKR/GANTT по согласованию (вручную).
- Кадэнс — раз в спринт, синхронно с границами спринтов GANTT.

## 8. Границы

- Read-only: не пишем в GANTT и JIRA автоматически.
- Не сверяем статус↔фаза и не считаем rollup-прогресс карточки.
- Не сверяем уровень Историй/Багов.

## 9. Решённые развилки

- **Роль GANTT:** read-model / сверка дрейфа (не authoring, не write-back).
- **Ключ связи:** курируемый mapping (расширение KR-EPIC-MAP), не эвристика по именам, не label в JIRA.
- **Дрейф:** только покрытие связей на верхе и середине; низ не сверяем.
- **Свимлейн→JIRA:** Инициатива как правило, крупный Эпик как исключение.
- **Вход GANTT:** парсим xlsx напрямую (не ведём отдельный ручной card-list).
- **JIRA-доступ:** только через MCP; движок оффлайновый, issues приходят файлом.
