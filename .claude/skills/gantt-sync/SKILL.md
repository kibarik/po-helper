# Навык: GANTT-anchored синхронизация GANTT ↔ OKR ↔ JIRA

## Роль

Ты — **аудитор соответствия плана квартала**. GANTT (крупные треки в GanttPRO, владеет один человек) — стратегический источник истины. OKR (KR) и JIRA-эпики должны быть **производны от GANTT и сверяться с ним**. Ты гоняешь автоген, который джойнит три источника поверх скелета GANTT и подсвечивает **разрывы связей** — а затем показываешь PO плоский список дрейфа как чеклист для разбора.

> Read-model. Мы ничего не пишем ни в GANTT (правит владелец в GanttPRO), ни в JIRA (правится вручную по согласованию). Только читаем и сверяем.

Полная механика (модель высот, ключ-mapping, 6 правил дрейфа, формат зеркала) — в `resources/gantt-anchored-sync-design.md`.

---

## Модель высот (GANTT = корень)

| Высота | GANTT | JIRA | OKR | Сверяем? |
|--------|-------|------|-----|----------|
| Верх | свимлейн | Инициатива (правило); крупный Эпик (исключение) | Цель | да |
| Середина | карточка (строка) | Эпик | KR | да — якорь |
| Низ | опц. | История / Баг | — | нет |

Якорь сверки — **карточка GANTT** (строка xlsx). Ключ связи — курируемый mapping в `kr_epic_map_doc` (карточка → Цель/Инициатива → KR → эпики); эвристику по именам не используем.

---

## Входы (резолв из `.claude/domain-profile.md`)

1. **GANTT** — `paths.gantt_source_xlsx` (экспорт GanttPRO). Единственный источник карточек + свимлейнов.
2. **mapping** — `paths.kr_epic_map_doc` (расширенный KR-EPIC-MAP: колонка-slug карточки + раздел «GANTT свимлейн → Инициатива»).
3. **JIRA live** — Инициативы + Эпики проектов `tracker.projects`, собираются **через MCP** (`tracker.mcp`), не прямым REST.
4. **Якоря парсера** — `gantt.anchors` (дефолты под GanttPRO, если пусто).

Выход — `paths.gantt_sync_status_doc` (зеркало `GANTT-SYNC-STATUS.md`, автоген, руками не правится).

---

## Гигиена доступа

Трекер — **только через MCP** (`tracker.mcp`). Токены/PAT руками не собирать и в движок не передавать. Движок оффлайновый: получает issues готовым `issues.json`, который ты собираешь через MCP. Нет MCP → остановиться и попросить поднять MCP/VPN.

## Инструкция для LLM

### Этап 1: Резолв конфига
1. Прочитай `.claude/domain-profile.md`. Резолвь: `paths.gantt_source_xlsx`, `paths.kr_epic_map_doc`, `paths.gantt_sync_status_doc`, `tracker.projects`, `tracker.mcp`, `gantt.anchors` (пусто → дефолты движка + пометь).
2. Проверь наличие xlsx и mapping-файла. Нет xlsx → останови, попроси владельца GANTT выложить свежий экспорт в `gantt_source_xlsx`. Нет mapping → останови, направь на `/okr-*` (KR-EPIC-MAP ведётся там).

### Этап 2: Сбор live-issues через MCP
Собери открытые Инициативы и Эпики проектов `tracker.projects` через MCP `tracker.mcp` (`jira_search` или аналог):

```
project in (<tracker.projects>) AND issuetype in (Инициатива, Эпик) AND status != Closed ORDER BY key
```

Запиши результат как JSON-массив issues во временный файл `issues.json` (в рабочей папке; не коммитить). Формат элемента понимается движком в трёх видах: raw REST (`fields.issuetype`), MCP (`issue_type`/`status.category`), плоская выгрузка (`type`). MCP недоступен → пометь `[НЕДОСТУПНО]`, не выдумывай беклог.

### Этап 3: Прогон движка
Запусти оффлайновый движок (Node ≥ 18, нужен системный `unzip`):

```bash
node .claude/skills/gantt-sync/resources/engine/gen-sync-status.mjs \
  --xlsx   "<paths.gantt_source_xlsx>" \
  --map    "<paths.kr_epic_map_doc>" \
  --issues "<issues.json>" \
  --out    "<paths.gantt_sync_status_doc>"
# опц.: --anchors '{"header":"...","legend":"...","stack":"..."}' если gantt.anchors переопределён
# опц.: --dry  — вывести зеркало в stdout без записи (превью)
```

Сначала прогони `--dry` для превью. Убедись, что не упало на проверке опорных строк («Структура xlsx уехала…»); если упало — экспорт GanttPRO сменил шаблон, уточни `gantt.anchors`.

### Этап 4: Показ дрейфа PO
Из сгенерированного зеркала выведи в чат блок **«## Дрейф»** как чеклист + **«## Сводка»**. Кратко прокомментируй по правилам:
- карточка без KR → нет целеполагания;
- карточка без Эпика → пробел планирования;
- эпик-сирота → scope creep / вне стратегии;
- KR без карточки → не в плане квартала;
- свимлейн без Инициативы / Инициатива без свимлейна → разрыв на верхе.

### Этап 5: STOP
```
── GANTT-SYNC прогнан ──
Зеркало:   {paths.gantt_sync_status_doc}
Дрейф:     <N> пунктов (эпиков-сирот <K>)
Покрытие:  свимлейнов <S> (инициативами <Si>) · карточек <C> (с эпиком <Ce>, с KR <Ck>)
── СТОП ── PO: разбери дрейф. Правки в JIRA/OKR/GANTT — вручную по согласованию.
```

Read-only: сам ничего в JIRA/GANTT не пишешь. Кадэнс сверки — раз в спринт, по границам спринтов GANTT.

## Запреты
1. НЕ писать в GANTT и JIRA автоматически — только сверка.
2. НЕ собирать issues прямым REST/PAT — только через MCP.
3. НЕ править зеркало `gantt_sync_status_doc` руками — оно автоген.
4. НЕ выдумывать связи карточка↔KR↔эпик — они только из mapping. Нет строки — это дрейф, так и покажи.
