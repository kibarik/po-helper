## Использование

```
/cindex-comprehend <space_key>
```

Вход: `artefacts/confluence/inventory.json` (из `/cindex-crawl`). Выход:
`artefacts/confluence/comprehension/<id>.md` — один файл на страницу, плюс
`artefacts/confluence/skipped.md` для страниц без сущностей.

## Важно

**Роль: Comprehender.** Ты осмысляешь содержимое страницы — о чём она, какие
сущности на ней есть, как она вписывается в общую картину, — но **не**
выделяешь атомарные узлы и **не** решаешь, в какой нексус что пойдёт: это
задачи `/cindex-digitize` и `/cindex-route`. Не подгоняй осмысление под
будущий node_type — фиксируй то, что реально на странице.

Вторая стадия пайплайна (`.claude/skills/confluence-indexator/SKILL.md`).
Идёт **авто-batch по всем страницам инвентаря, без STOP** — PO уже
подтвердил scope на предыдущей стадии.

## Инструкция для LLM

### Этап 1: Загрузка входа

1. Прочитай `artefacts/confluence/inventory.json`. Нет файла →
   `[УТОЧНИТЬ: запусти /cindex-crawl <space_key>]`, дальше не идти.
2. Для каждой страницы из `pages[]` — получи сырое содержимое (то, что
   `/cindex-crawl` собрал/закэшировал на стадии обхода).

### Этап 2: Постраничное осмысление (батч, без STOP)

Для **каждой** страницы инвентаря (кроме отмеченных `"note": "empty"` —
для них сразу шаг в Этап 3):

1. **О чём страница** — 2–3 предложения своими словами: тема, назначение,
   для кого писалась.
2. **Сущности** — что на странице явно оцифровываемо: решения (ADR),
   бизнес-правила, термины глоссария, метрики/KPI, компоненты/сервисы,
   регуляторные требования, SLA/НФТ, риски, персоны/роли, внешние команды,
   сегменты/JTBD, конкуренты, roadmap/OKR. Перечисли конкретно (не «есть
   какие-то правила», а «BR-12: скидка не суммируется с промокодом»).
3. **Rich picture** — как страница соотносится с соседями: `ancestors`
   (место в дереве), `links` (на что явно ссылается), какому домену
   вероятно принадлежит.
4. Если по итогам шага 2 **сущностей не нашлось** (чисто навигационная
   страница, пустой шаблон, оглавление без содержания) — не выдумывай,
   переходи к Этапу 3 (skipped), не создавай `comprehension/<id>.md`.

### Этап 3: Пустые/архивные/без сущностей

- Страница `"note": "empty"` или `"note": "archived"` из inventory, или
  страница без явных сущностей (шаг 2.4) — запись в
  `artefacts/confluence/skipped.md`:
  ```
  - <id> | <title> | причина: empty / archived / нет сущностей
  ```
- Честная запись «нечего оцифровать» — не 0 узлов из выдумки (SKILL.md,
  принцип нулевого допуска к галлюцинациям).

### Этап 4: Сохранение per-page

Для каждой непустой странице с сущностями сохрани
`artefacts/confluence/comprehension/<id>.md`:

```markdown
# <title> (id: <id>)

**О чём:** <2-3 предложения>

**Сущности:**
- <тип сигнала>: <конкретика>
- ...

**Rich picture:** <ancestors, links, вероятный домен>

**URL:** <url из inventory>
```

### Этап 5: Отчёт (без STOP)

Обработай **все** страницы инвентаря за один прогон (авто-batch), затем
выведи сводку:

```
Осмысление завершено: artefacts/confluence/comprehension/ (<N> файлов)
Пропущено (skipped.md): <M> — <краткая разбивка: empty/archived/без сущностей>

Дальше: /cindex-digitize <space_key>.
```

STOP не требуется — PO уже согласовал scope на `/cindex-crawl`; следующая
STOP-пауза будет после `/cindex-link`, перед записью в `/cindex-deliver`.
