## Использование

```
/bft-gen <epic_code> <краткое_описание>
```

**Параметры:**
- `<epic_code>` — короткий код и название (напр. `<EPIC>: <Название>`).
- Дополнительно приложи: эпик в трекере, ТЗ, решения PO, смежные СА/БФТ, стейкхолдеров.

## Примеры

```
/bft-gen SEARCH-1: Полнотекстовый поиск по каталогу
  Эпик: PROJECTKEY-123
  ТЗ: <URL wiki>
  Решения PO: синхронный поиск; реиндексация ≤ N минут
  Стейкхолдеры: <PO>, <архитектор>
```

## Важно

**Приоритет:** СТРОГОЕ соблюдение шаблона `skills/bft-writer/examples/ideal_bft.md` и стандартов `skills/bft-writer/resources/bft_standards.md`.

**Ожидаемый результат:** документ сохраняется рядом с другими БФТ проекта с wiki-frontmatter.

**Принцип:** нулевой допуск к галлюцинациям. Каждый факт → источник (трекер / PO / ТЗ). Неизвестное → `[УТОЧНИТЬ]`.

## Инструкция для LLM

### Этап 0: Загрузка роли и стандартов

1. Прочитай `skills/bft-writer/SKILL.md` — персона и принципы.
2. Загрузи стандарты: `resources/bft_standards.md`.
3. Загрузи эталоны: `examples/ideal_bft.md` (шаблон), `examples/golden_bft_example.md` (golden).
4. Загрузи гейты и протоколы: `resources/hard_gates.md`, `resources/debate_rules.md`.

### Этап 1: Якоря (Truth Anchors)

- Найди и зафиксируй для каждого факта источник: эпик трекера, ТЗ, решения PO, смежные СА/БФТ, стейкхолдеры.
- **GATE 0:** минимум {эпик трекера + постановка/ТЗ}. Иначе — запроси у PO, не выдумывай.

### Этап 2: Каркас

- Бизнес описание; Общая информация (проект/PO/отв.за документ/эпик); Стейкхолдеры; История изменений.
- **Границы системы:** явное ВНУТРИ / СНАРУЖИ.
- As-Is → Gap → To-Be.
- **Измеримые критерии успеха** (метрика → цель → срок) — вывести **ОТДЕЛЬНЫМ разделом** в документе (обязательно для фичей/миграций; для чисто технических API-БФТ — продуктовые метрики или явная пометка «метрики в НФТ» с обоснованием).
- **GATE 1:** границы и критерии успеха оцифрованы и выведены разделом.

### Этап 3: Открытые вопросы (No Silent Skip)

- Выпиши ВСЕ неясности/блокеры/противоречия в «Открытые вопросы».
- Закрытые → «Ключевые решения» (вопрос → решение → кто ответил → источник).
- **GATE 2:** ни один неясный момент не утаен; `[УТОЧНИТЬ]` проставлены.

### Этап 4: Каскад требований (с трассировкой)

- БТ (ценность, приоритет) → ПТ (story) → ИТ (контракты) → ФТ (поведение, приоритет) → НФТ (только измеримые).
- Идентификаторы `{БТ|ПТ|ИТ|ФТ|НФТ}-{EPIC}-{N}`.
- КАЖДАЯ строка: «Связанные требования» заполнены вверх + эпик.
- Каждый ФТ ≈ атомарная задача (1–2 дня).
- **GATE 3:** трассировка сплошная, приоритеты проставлены, НФТ измеримы.

### Этап 5: Диаграммы и схемы данных

- PlantUML sequence: happy path + ВСЕ `alt`-ветки (ошибка/недоступно/отмена/таймаут).
- Таблица акторов. Атрибутивный состав ключевых сообщений (Параметр | Обяз | Тип | Описание).
- Mermaid запрещён.
- **GATE 4:** показаны negative flows.

### Этап 6: Зависимости, риски, blast radius

- Прямые/косвенные зависимости между командами/системами проекта со статусом согласования.
- Защищённые компоненты (backward compat). Риски с митигацией.
- **GATE 5:** зависимости имеют статус; риски покрыты.

### Этап 7: Якоря истины

- Заполни финальный раздел «Якоря истины» (факт → источник: трекер / решение PO / ТЗ / смежный СА).
- Неподтверждённое → `[УТОЧНИТЬ у {кого}]`. Собери все `[УТОЧНИТЬ]` в блок **Требует уточнения у PO** в начале документа.

### Этап 8: Adversarial Review

- Проведи 3-раундовый red-team по `resources/debate_rules.md` (Аналитик vs Адвокат Дьявола).
- Обязательные линзы атаки: нагрузка, отказ зависимости, идемпотентность/гонки, изменение требований, безопасность/ПДн, скрытые зависимости, трассировка.
- Внеси найденные negative flows / НФТ / правки в черновик. Допиши вердикт в блок `## Adversarial Review`.

### Этап 9: Самовалидация (Hard Gates + Светофор)

- Прогон 10 бинарных 🔴-гейтов из `resources/hard_gates.md`.
- Прогон чек-листа самовалидации.
- Сформируй отчёт «Светофор» по слоям.
- Любой 🔴 → исправить и пересдать. Только 🟢/🟡 → сборка финала.

### Этап 10: Сборка и сохранение

- Frontmatter (source/space/version/synced/jira) + H1 `# [БФТ] {EPIC}: {Название}` + все разделы по `ideal_bft.md`.
- Сохрани документ.
- Выведи: «Черновик БФТ `{EPIC}` готов. Светофор: <итог>. Все `[УТОЧНИТЬ]` собраны в начале. Следующий шаг: ревью (Продакт/Архитектор/Аналитик/Разработка/Тестирование).»
