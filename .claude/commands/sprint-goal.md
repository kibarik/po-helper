## Использование

```
/sprint-goal <sprint>
```

Вход: `sprint-context.md`. Выход: `{sprint_workspace}/sprint-goal.md`.

## Важно

**Роль: Outcome Designer. Это вопрос WHY из Scrum-планирования.** Сформулировать верхнеуровневый **образ результата команды** — что команда должна показать руководителю к концу спринта, строго с позиции стратегических целей (OKR). Это даёт команде однозначный ответ: «что наш руководитель хочет увидеть».

> Здесь только стратегия — никакой тактики (истории/исполнители/SP появятся позже). Sprint Goal не должен загрязняться деталями.

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `skills/sprint-planner/SKILL.md` (принципы 1, 6).
2. Прочитай `{sprint_workspace}/sprint-context.md` — актуальные KR спринта, коммитменты.
3. Прочитай `{okr_output_doc}` — образы результата соответствующих KR (квартальный контекст).

### Этап 2: Диалог по образу результата (по одному вопросу за раз)
Для каждой активной OBJ/KR спринта уточни видение PO. Образ результата формируется диалогом, а не монологом.

**Q (на каждый OBJ):** «По OBJ {N} ({название}) в этом спринте двигаем KR {список}. Что конкретно команда должна **показать** к концу спринта, чтобы это был виден прогресс к квартальной цели? Опиши доказуемое состояние — что я как руководитель увижу на демо.»

Уточняй, пока формулировка не станет доказуемой (не «поработаем над», а «X работает и продемонстрировано»).

### Этап 3: Сборка sprint-goal.md
```markdown
# Sprint Goal: {sprint}

## OBJ {N}: {название}
**KR:** {список с ссылками}
**🎯 Образ результата команды:** к концу спринта команда показывает {доказуемое состояние}.
**Почему это важно для квартала:** {связь с образом результата KR}.
**Как проверим (демо):** {что показываем на ревью}.

[повторить для каждой активной OBJ]

## Сводный Sprint Goal спринта (1–2 предложения)
{интегральный образ результата всего спринта}
```

Правила:
- 1–3 Sprint Goal (по числу активных OBJ). Если их больше 3 — спринт расфокусирован, предупредить PO.
- Каждый образ результата ← KR из context. Без KR → это внеплановое, не Sprint Goal.
- Никаких историй/исполнителей/SP на этой стадии.

### Этап 4: Сохранение + STOP
- Сохрани в `{sprint_workspace}/sprint-goal.md`.
- **Выведи и ОСТАНОВИСЬ:**
  ```
  Sprint Goal готов: {sprint_workspace}/sprint-goal.md
  Целей спринта: N. Сводный goal: «{одна строка}».
  ── СТОП ── PO: подтвердите образ результата команды.
  Дальше → /sprint-decompose {sprint}
  ```

## Запреты
1. НЕ декомпозируй на истории — это `/sprint-decompose`.
2. НЕ назначай исполнителей и SP.
3. Образ результата = доказуемое состояние, не активность. Не «улучшить/поработать».
4. Более 3 Sprint Goal → предупредить о расфокусировке, не молчать.
