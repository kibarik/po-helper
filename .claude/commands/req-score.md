## Использование

```
/req-score <request_id>
```

Вход: `smart-task.md` (черновик) + все artefacts. Выход: `artefacts/scoring.md` + **routing-решение**.

## Важно

**Роль: Validator + Router.** Совмещаешь две функции: проверяешь зрелость запроса (hard gates + Светофор) и **выносишь routing-решение** — что с запросом делать. Adversarial-проверка против «придуманной ценности» встроена в гейты (отдельной стадии дебатов в v1 нет).

**Свежий взгляд:** проверяешь критически, как будто видишь запрос впервые. Главный вопрос — ценность подтверждена словами запрашивающего (R1), а не додумана?

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `domain-profile.md`, `skills/request-intake/resources/hard_gates.md`, `resources/anchor_rules.md`.
2. Прочитай `smart-task.md` и все `artefacts/` (context, interview, impact-map, gap).

### Этап 2: Hard Gates (часть A)
Прогони 8 бинарных гейтов из `hard_gates.md` (Hallucination, Imagined-Value, No-Acceptance, Solution-as-Problem, Altitude, Open-Questions, Scope-Owner, Smart-Incomplete). Любой 🔴 → зафиксируй и определи возврат (см. «Возвраты» в hard_gates.md).

### Этап 3: Routing-решение (часть B)
Классифицируй запрос — ровно одно решение. Грунтуй решение релевантными GROUND-Нексусами (матрица — `sa_documentation/nexus_process_map.md`, колонка `/req-score`):
- **Консультация** — это вопрос, ответ есть в домене → ответь grounded (со ссылками на NEXUS/CORTEX) и закрой.
- **Отклонить** — вне scope / дубль / ценность не подтверждена (гейт 2) → краткое обоснование. Прецедент отказа сверь с `precedents` (уже отклоняли похожее — с той же причиной?); нарушение стандартов — с `compliance`.
- **Редирект** — не наша команда (гейт 7) → целевая команда + почему. Владельца целевого домена возьми из `ownership`.
- **→ БФТ** — валидный change-request, все гейты 🟢/🟡 → на `/req-handoff`. Перед этим сверь: соотносится ли с квартальными приоритетами (`strategy`) и по силам ли команде (`capacity`) — если явно против OKR или вне мощности, отметь в обосновании как риск приоритизации.

### Этап 4: Светофор (часть C)
Заполни таблицу Светофора из `hard_gates.md`. Итог: 🟢 / 🟡 / 🔴.

### Этап 5: Сохранение + STOP
- Сохрани `artefacts/scoring.md` (гейты + routing-решение + Светофор + обоснование).
```
Скоринг: <intake_workspace>/<request_id>/artefacts/scoring.md
Светофор: <🟢/🟡/🔴>. Гейты: <пройдены / 🔴 N>.
РЕШЕНИЕ: <Консультация | Отклонить | Редирект → команда | → БФТ>
Обоснование: <кратко>.
── СТОП ──
<если → БФТ>  Дальше — /req-handoff <request_id>.
<если 🔴>     Возврат — /req-interview | /req-gap | /req-smart (см. scoring.md).
<если Консультация/Отклонить/Редирект>  Пайплайн завершён (early exit). Ответ/обоснование выше.
```

## Запреты
1. НЕ пропускать 🔴-гейт ради «протолкнуть в БФТ».
2. НЕ выносить больше одного routing-решения.
3. Ценность без цитаты R1 → 🔴 (гейт 1/2), не «додумать за стейкхолдера».
4. Стиль scoring.md — деловая проза; статусные эмодзи (🟢/🟡/🔴) допустимы только как метки Светофора.
