## Использование

```
/req-handoff <request_id>
```

Вход: `smart-task.md` + `artefacts/scoring.md` (решение «→ БФТ», Светофор 🟢/🟡). Выход: финальный `smart-task.md` + подсказка `/bft-context-gen`.

## Важно

**Роль: Deliverer (local).** Финализируешь SMART-задачу и готовишь хэндофф в БФТ-пайплайн. **JIRA не пишем** (решение v1): PO заводит эпик сам — меньше прав/MCP-зависимостей. Задача стадии — отдать PO готовый seed и точную следующую команду.

Запускается **только** если `/req-score` дал решение «→ БФТ». Для Консультация/Отклонить/Редирект эта стадия не нужна (пайплайн завершён на скоринге).

## Инструкция для LLM

### Этап 1: Загрузка и проверка
1. Прочитай `domain-profile.md`, `smart-task.md`, `artefacts/scoring.md`.
2. Проверь: решение скоринга = «→ БФТ» и Светофор 🟢/🟡. Иначе → стоп, сообщи что хэндофф не применим (см. решение в scoring.md).

### Этап 2: Финализация smart-task.md
- Сними пометку «черновик»; убедись, что критерии приёмки проверяемы, якоря на месте, SMART-чек закрыт (🟡-пометки `[срок: УТОЧНИТЬ]` допустимы — перенести в «Открытые вопросы»).
- Добавь блок **Хэндофф** в конец `smart-task.md`:
  - предлагаемый код эпика и проект (из `domain-profile.md` `tracker.projects` / `bft.default_project`);
  - что PO нужно занести в JIRA (заголовок + суть + критерии приёмки);
  - следующая команда БФТ-пайплайна.

### Этап 3: Отчёт + STOP
```
SMART-задача финализирована: <intake_workspace>/<request_id>/smart-task.md
Заголовок: «<…>». Светофор: <🟢/🟡>.

Хэндофф в bft-writer (JIRA заводит PO):
1. Заведи эпик в <проект>: «<заголовок>» (суть + критерии приёмки — в smart-task.md).
2. Запусти: /bft-context-gen <epic_code> <jira_key>

── СТОП ── Intake завершён. Дальше — БФТ-пайплайн по созданному эпику.
```

## Запреты
1. НЕ писать в JIRA/Confluence автоматически (решение v1 — только локальный артефакт).
2. НЕ запускать, если решение скоринга ≠ «→ БФТ».
3. НЕ декомпозировать задачу — это `/bft-*` ниже по потоку.
4. Стиль — `skills/bft-writer/resources/writing_style.md`.
