## Использование

```
/sprint-activate <sprint>
```

Вход: наполненный спринт в JIRA (после `/sprint-build`) + `{sprint_output_doc}`. Выход: неактуальное закрыто, спринт стартован (по явному ок PO), артефакты закоммичены, цикл замкнут на `/sprint-fact`.

## Важно

**Роль: Finalizer. Финал BUILD-прохода — актуализация и активация.** `Start sprint` — отдельным явным подтверждением, **не «заодно»** (🚪 G6 activation-gate). Закрытие задач — только по согласованию PO, корректным transition. Хендофф чистый: без секретов, без сырого транскрипта.

> Разрешено только при `tracker.access: build` и пройденном `/sprint-build`. Нет MCP → СТОП.

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `skills/sprint-planner/SKILL.md` (принципы), `resources/build_gates.md` (G6 + антипаттерны), `resources/jira_build.md` (transitions, верификация).
2. Прочитай `.claude/domain-profile.md` секцию `tracker` (`access: build`, MCP) и `{sprint_output_doc}`.
3. Предусловия: `access: build` · MCP поднят · `/sprint-build` пройден (орфанов 0). Нарушение → СТОП.

### Этап 2: Актуализация (кандидаты на закрытие)
Собери подзадачи/истории под закрытие по критериям: родитель закрыт · работа superseded · протухло по дате. Выведи **список кандидатов** → согласование PO (по одному кластеру за раз, не скопом):
```
Кандидаты на закрытие ({sprint}):
- {ключ} — {причина: родитель закрыт / superseded / протухло}
── PO: подтвердите закрытие / уберите из списка.
```
После «ок» — закрыть корректным transition, верифицировать чтением. (Опц.) убрать закрытые легаси-деревья из спринта.

### Этап 3: 🚪 Activation-gate (отдельное подтверждение)
Спринт наполнен и вычищен → **отдельно** предложи старт:
```
Готов к активации: {sprint} · слот {имя} ({даты})
Истории: N · Орфанов: 0 · Неактуальное закрыто: K
⚠️ Start sprint необратим и стартует каденцию команды.
── СТОП ── PO: «стартуй спринт» (явно) — или «пока не стартуй».
```
Активировать **только** по явной команде PO. Без явного «стартуй» — оставить в future, зафиксировать «не активирован».

### Этап 4: Отчёт и финализация ПЛАНа
1. Заполни `{sprint_output_doc}` реальными ключами + ссылками JIRA на истории и подзадачи (не «на словах»), проставь статус (Активен/Готов к старту).
2. Сохрани через `obsidian.vault_write`.

### Этап 5: Коммит артефактов + гигиена
1. Убедись, что артефакты под git и **не в вычищаемом пути** (worktree/tmp). В вычищаемом → перенести в `{execution_root}`.
2. Закоммить артефакты спринта (не оставлять незакоммиченными).
3. Чистый хендофф: без секретов, без сырого транскрипта. Секреты сессии — ротировать.

### Этап 6: Манифест + замыкание цикла
```markdown
## Delivery Manifest: BUILD {sprint}
**Дата:** {дата} · **Слот:** {имя} · **Статус:** {Активен / Готов к старту}
### Итог
- Истории в спринте: N · NEW: K · Орфанов: 0 · Закрыто неактуального: M
- Артефакты: {sprint_output_doc} (реальные ключи+ссылки) — ✅, закоммичены — ✅
### Замыкание цикла
- В конце спринта → /sprint-fact {sprint} (ФАКТ = вход для /sprint-sync следующего)
```
```
BUILD {sprint} завершён. Спринт: {Активен / Готов к старту}.
── Планирование и материализация спринта завершены ──
```

## Запреты
1. `Start sprint` — только по отдельному явному «стартуй» PO, НЕ «заодно» (G6).
2. Закрытие задач — только по согласованию PO, корректным transition. НЕ закрывать молча.
3. НЕ коммитить артефакты в вычищаемый путь. НЕ оставлять незакоммиченными.
4. Секреты в чат/хендофф — запрещены; сессионные — ротировать.
5. Нет MCP / `read-only` → СТОП.
