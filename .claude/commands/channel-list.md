## Использование

```
/channel-list [фильтр]
```

- `[фильтр]` — опц.: по `channel_type` (telegram/email/…), по стейкхолдеру (node_id), по теме или участку системы.

Выход: таблица каналов + матрица покрытия в чат (read-only, узлы не меняет).

## Важно

**Роль: Inventory Reporter.** Показать честную картину: какие каналы размечены, что покрыто, где пробелы. Не выдумывать каналы, которых нет в реестре.

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `.claude/domain-profile.md` — резолвь `paths.channels_nexus` (дефолт `GROUND/NEXUS/channels`).
2. Прочитай `skills/info-channels/SKILL.md` и `resources/routing_rules.md` (§D Светофор покрытия).
3. Реестр пуст / папки нет → сообщи «каналов пока нет, добавьте через /channel-map», STOP.

### Этап 2: Сбор
Прочитай все `chan-*.md` в `<channels_nexus>` (кроме `_index.md`, `_template.md`). Для каждого извлеки frontmatter: `platform`/`handle`, `channel_type`, `purpose`, `topics`, `signal_types`, `stakeholders`, `system_areas`, `goals`, `ingest_action`, `confidence`, `ripeness`. Применяй `[фильтр]`, если задан.

### Этап 3: Таблица каналов
```
| Канал | Тип | Назначение | Темы | Стейкхолдеры | Участки | Роутинг | CP/спелость |
```
Пустые обязательные поля показывай как `[УТОЧНИТЬ]`. `ripeness: wilting` помечай — узел протух, нужна актуализация.

### Этап 4: Матрица покрытия (Светофор §D)
Для каждого канала — 🟢/🟡/🔴 по обязательным полям (`purpose`/`topics`/`stakeholders`) + опциональным (`system_areas`/`goals`/`ingest_action`). Сводка снизу:
- Каналов всего: N. Полностью размечены (🟢): X. С пробелами (🟡): Y. Не размечены (🔴): Z.
- **Пробелы покрытия:** какие стейкхолдеры из `NEXUS/team` не привязаны ни к одному каналу; какие участки системы / цели без канала-источника.

### Этап 5: Отчёт + STOP
```
Каналов: <N> (🟢<X> / 🟡<Y> / 🔴<Z>). Протухших (wilting): <k>.
Пробелы: <кратко — кто/что без канала>.
── СТОП ── Доработать разметку: /channel-map <slug>. Разметить входящее: /channel-route.
```

## Запреты
1. Read-only: НЕ менять узлы.
2. НЕ выдумывать каналы/связи — только то, что в реестре. Нет данных → `[УТОЧНИТЬ]`.
