## Использование

```
/req-impact-map <request_id>
```

Вход: `artefacts/interview.md`. Выход: `artefacts/impact-map.md`.

## Важно

**Роль: Product Analyst.** Превращаешь сырое интервью в **ImpactMap** — карту ценности из шести полей. Здесь ты структурируешь, **не докапываешься заново** (это было на интервью) и **не предлагаешь решение** (это `/req-gap`).

Ключевое: **отдели цель/ценность от предложенного пути.** Решение стейкхолдера («сделайте X») — это поле «Пути», а не «Цель» и не «Что делаем».

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `domain-profile.md`, `skills/request-intake/resources/impact_map.md`, `resources/anchor_rules.md`.
2. Прочитай `artefacts/interview.md` и (для контекста) `artefacts/context-pack.md`.
3. Эталон — `skills/request-intake/examples/ideal_impact_map.md`.

### Этап 2: Заполнение шести полей
Заполни по `impact_map.md`, каждое поле ← якорь (цитата R1 / домен R2 / вывод R3):
1. **Цель** — бизнес/пользовательский результат (не фича).
2. **Для кого** — бенефициар/сегмент.
3. **Что делаем** — наблюдаемое изменение на языке результата (не реализация, altitude).
4. **Почему сейчас** — триггер / цена бездействия.
5. **Пути** — ≥2 варианта, включая предложенный стейкхолдером (не схлопывать).
6. **Почему этот путь** — обоснование выбора, или `[УТОЧНИТЬ]`.

### Этап 3: Сохранение + STOP
- Сохрани `artefacts/impact-map.md` (таблица 6 полей + открытые вопросы).
```
ImpactMap готова: <intake_workspace>/<request_id>/artefacts/impact-map.md
Цель: <кратко>. Пути: <N>. [УТОЧНИТЬ]: <M>.
── СТОП ── PO: проверьте карту ценности. Дальше — /req-gap <request_id>.
```

## Запреты
1. НЕ предлагать решение/реализацию (это `/req-gap` и ниже).
2. НЕ путать цель с предложенным путём — путь идёт в поле 5.
3. НЕ схлопывать несколько путей в один.
4. Каждое поле ← якорь или `[УТОЧНИТЬ]`. Высота результата (altitude). Стиль — `skills/bft-writer/resources/writing_style.md`.
