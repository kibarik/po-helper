## Использование

```
/radar-graph [run_id]
```

- `[run_id]` — опц. слаг прогона калибровки; если задан, исходник сохраняется в его папку.

Выход: PlantUML-исходник People Graph + inline-SVG в чате (через `/diagram-view`).

## Важно

**Роль: Cartographer.** Строишь **диаграмму связей в команде** из реальных person-узлов `team`-нексуса — три слоя (org / social / expertise) с легендой. Это визуальная проверка графа перед калибровкой и после правок, не сам прогон.

**Только существующие узлы.** Рисуешь то, что есть в `GROUND/NEXUS/team/`. Не достраиваешь связи и людей, которых в нексусе нет.

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `skills/nexus-calibration/resources/graph_render.md` (спецификация рендера) и `SKILL.md` (3 слоя графа).
2. Прочитай `.claude/domain-profile.md`, если есть, — резолвь название команды и `paths.calibration_workspace`.
3. Прочитай все `GROUND/NEXUS/team/*.md`, **кроме** `_index.md`/`_template.md`.

### Этап 2: Проверка наполнения
- Узлов нет (только placeholder) → СТОП: «People Graph пуст, рисовать нечего. Наполни нексус: /paf-nexus-create или вручную по _template.md».
- Узлы есть → собери по каждому: `node_id`, `full_name`, `role_title`, `department`, `reports_to`, `manages`, `collaborates_with`, топ-2 `contact_for`/`expertise_topics`.

### Этап 3: Сборка PlantUML
Собери исходник по скелету из `graph_render.md`:
- один `rectangle` на человека (ID = `node_id`), подпись = ФИО + роль + 1–2 темы `contact_for`;
- группировка `package` по `department` (если заполнен);
- рёбра: `reports_to` → сплошная `-->`; `collaborates_with` → пунктир `..>`; экспертиза → нота/подпись, не ребро;
- легенда обязательна (расшифровка стилей);
- висячие `reports_to`/`manages` (на несуществующий node_id) и изолированные узлы — пометь (блок «вне графа связей»), это сигнал для калибровки.

### Этап 4: Сохранение исходника
- `run_id` задан → сохрани в `<calibration_workspace>/<run_id>/people-graph.puml`.
- Иначе → `/tmp/people-graph.puml`.

### Этап 5: Рендер через /diagram-view
Передай управление навыку `/diagram-view` для inline-рендера (он гонит `plantuml -tsvg -charset UTF-8` и показывает SVG). `plantuml` недоступен → сообщи (окружение, не ошибка графа) и отдай PO готовый `.puml`-исходник.

### Этап 6: Текст-ответ
1 строка — что за диаграмма + 2–3 наблюдения, релевантных калибровке: пересечения экспертизы (кандидаты на дизамбигуацию), висячая иерархия, люди без `contact_for` (невидимы для роутинга).

```
Диаграмма связей: <N> человек, <M> связей подчинения, <K> социальных.
Зацепки для калибровки: <пересечение экспертизы X> · <изолированный узел Y> · <висячий reports_to Z>.
Граф пуст для роутинга по: <узлы без contact_for>.
Дальше — /radar-calibrate short.
```

## Запреты
1. НЕ выдумывать связи/людей вне нексуса — рисуешь только наполненные узлы.
2. НЕ «улучшать» граф (достраивать логичные, но отсутствующие рёбра) — висячие/изолированные узлы показывай как есть, это диагностика.
3. Большой граф (>~25 чел) — предупреди и предложи разрез по `department`/только-иерархия (см. `graph_render.md`), не рендерь нечитаемое полотно молча.
4. Стиль текста-ответа — деловая проза (`skills/bft-writer/resources/writing_style.md`).
