## Использование

```
/okr-landscape <quarter>
```

Пример: `/okr-landscape 2026-Q4`

Вход: `{okr_workspace}/context-pack.md` (стадия 0). Выход: `{okr_workspace}/landscape-{quarter}.md` + обновлённые узлы `GROUND/NEXUS/landscape/`.

## Важно

**Роль: Landscape Analyst.** Собрать контекст окружающего ландшафта до формулировки OBJ/KR: «что делает наша команда» vs «что делают команды вокруг». Цель — чтобы дальше (в OKR и БФТ) точно понимать блокаторы и связи: «с PO каких команд мы связаны и почему». Минимум галлюцинаций: каждый факт ← источник.

> Стадия 0.5 OKR-пайплайна: `context-gen → **landscape** → objectives`. Отдельная LLM-роль, свой артефакт, STOP-пауза. Не подменяет `/okr-context-gen` (наш стратегический контекст) — этот шаг про команды **вокруг**.

## Инструкция для LLM

### Этап 1: Загрузка роли и стандартов
1. Прочитай `skills/okr-planner/SKILL.md`.
2. Прочитай `skills/okr-planner/resources/okr_standards.md` (секция «Снимок ландшафта»).
3. Резолвь секцию `landscape` из `.claude/domain-profile.md` (`nexus_root`, `snapshot_doc`, `ext_teams`).

### Этап 2: Чтение фона (параллельно)
4. Прочитай `{okr_workspace}/context-pack.md` — что делает наша команда в квартале (из него берётся §1 снимка).
5. Прочитай узлы `GROUND/NEXUS/landscape/` (ext-team узлы) — уже известные команды вокруг. Если Нексуса нет → предложи засеять `/paf-nexus-create landscape` из `domain-profile.landscape.ext_teams` или создай узлы по ходу опроса.
6. Прочитай C1-архитектуру (`{cortex.architecture}`, если есть) — кто владеет смежными системами (кандидаты во внешние команды).
7. Прочитай годовой roadmap `{roadmap_year_doc}` — упоминания смежных команд/направлений.

### Этап 3: JIRA cross-project (если доступен)
Через `atlassian`: активные эпики в проектах **вне** `tracker.projects` нашей команды, но пересекающиеся по компонентам/системам. Фокус: чем заняты соседние команды в квартале. Недоступно → `[НЕДОСТУПНО: JIRA — нет VPN]`.

### Этап 4: Опрос PO (один вопрос за раз)
Для каждой команды вокруг — уточняй последовательно. PO может ответить «пропустить»/«достаточно».

**Q1 (кто рядом):** «Какие команды работают в орбите на {quarter}? Кто наши upstream (мы зависим), downstream (зависят от нас), peer, platform, регуляторы?»

**Q2 (чем заняты):** «Что каждая из этих команд делает в {quarter} — их фокус/OKR, насколько знаешь? Откуда знаешь (их roadmap / встреча / слухи)?»

**Q3 (точки касания):** «Где мы стыкуемся: общие системы, API, потоки данных, общие стейкхолдеры?»

**Q4 (влияние):** «На кого мы можем влиять — прямо (общий сервис/контракт) или косвенно (через посредника)? Кто может нас блокировать?»

**Q5 (PO и контакты):** «Кто PO каждой команды и как с ним связаться?»

### Этап 5: Обновление Нексуса `landscape`
Для каждой команды — создай/обнови ext-team узел по шаблону `GROUND/NEXUS/landscape/_template.md`:
- `node_id: landscape-team-<slug>` (ascii); стабильные поля: `team_name`, `po_name`, `owned_systems`, `relationship`, `touchpoints`, `influence`.
- `sources` — реальный источник (`okr-landscape:interview` / `their-okr` / `confluence` / `jira` / `architecture-inference`); `confidence` по уровню доверия; `updated` = сегодня.
- **Не выдумывай** иерархию/системы сверх подтверждённого — неизвестное → `[УТОЧНИТЬ]`, узел без `sources[]` не создавать.
- Текущий OKR команды в узел НЕ пиши (он квартальный) — он идёт в снимок (§2 ниже).

### Этап 6: Сборка снимка `landscape-{quarter}.md`
Структура (формат — `okr_standards.md` «Снимок ландшафта»):

```markdown
# Landscape: OKR {quarter}

## 1. Наша команда в квартале (источник: context-pack.md)
[Что делаем мы — 3-5 пунктов фокуса квартала из context-pack]

## 2. Команды вокруг
| Команда | PO | OKR/фокус квартала (источник) | Связь | Релевантность к нашим OBJ/KR | Флаги |
|---|---|---|---|---|---|
[по строке на команду; Связь = upstream/downstream/peer/platform/regulator + direct/indirect; Флаги = [RISK]/[BLOCKER]/[EXT]]

## 3. Матрица покрытия
| Команда | Узел в Нексусе | Источник факта | [УТОЧНИТЬ] |

## 4. Открытые вопросы [УТОЧНИТЬ]
[Что не удалось установить из источников]
```

### Этап 7: Сохранение + STOP
- Сохрани снимок в `{okr_workspace}/landscape-{quarter}.md`.
- Обнови/создай узлы в `GROUND/NEXUS/landscape/`.
- **Выведи и ОСТАНОВИСЬ:**
  ```
  Landscape готов: {okr_workspace}/landscape-{quarter}.md
  Команд вокруг: N (upstream: X / downstream: Y / peer: Z). Узлов обновлено: M.
  Флаги: [RISK] — a, [BLOCKER] — b. [УТОЧНИТЬ]: k пунктов.
  ── СТОП ── PO: проверьте ландшафт, добавьте пропущенные команды.
  Дальше → /okr-objectives {quarter}
  ```

## Запреты
1. НЕ формулируй OBJ или KR — это `/okr-objectives`. Здесь только контекст вокруг.
2. Факт без источника → `[УТОЧНИТЬ у {кого}]`, не выдумывай чужой OKR.
3. JIRA/чужой беклог недоступен → `[НЕДОСТУПНО]`, не придумывай состав.
4. Не смешивай наши команды-исполнители (`teams`) с внешним ландшафтом (`landscape`).
5. Стиль артефакта — по `skills/bft-writer/resources/writing_style.md`: деловая проза без AI-стоп-слов; эмодзи только семантические (статус в матрице покрытия).
