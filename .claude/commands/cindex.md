## Использование

```
/cindex <space_key>
```

Вход: ключ пространства Confluence (напр. `PROD` — иллюстративный пример,
реальное пространство берётся из `.claude/domain-profile.md`). Выход: ничего
не создаёт — только диагностика статуса и рекомендация следующего шага.

## Важно

**Роль: оркестратор, не исполнитель стадии.** `/cindex` не обходит Confluence,
не осмысляет и не пишет узлы — это делают `/cindex-crawl` → `/cindex-comprehend`
→ `/cindex-digitize` → `/cindex-route` → `/cindex-link` → `/cindex-deliver`.
Команда только читает `artefacts/confluence/` и говорит, где пайплайн
остановился и что запустить дальше.

Полная методология, принципы (ноль галлюцинаций, оцифровка ≠ валидация,
роутинг только в реестр, гибрид по confidence, STOP-паузы) — в
`.claude/skills/confluence-indexator/SKILL.md`. Этот файл её не дублирует.

## Инструкция для LLM

### Этап 1: Проверка предусловий

Прочитай `.claude/skills/confluence-indexator/SKILL.md`, раздел
«Предусловия». Если Confluence MCP не настроен в `.mcp.json` (проверить через
`ToolSearch` на инструменты обхода пространства/чтения страницы) — сообщи об
этом сразу в статусе (см. Этап 3), не блокируй вывод статуса целиком: PO
может ещё не дошёл до `/cindex-crawl`.

### Этап 2: Инвентаризация артефактов

Проверь наличие (не содержимое — только факт существования и, если файл
есть, грубый размер/число записей) для `<space_key>`:

| Артефакт | Путь | Стадия |
|---|---|---|
| Inventory | `artefacts/confluence/inventory.json` | 1 Обход |
| Comprehension | `artefacts/confluence/comprehension/*.md` | 2 Осмысление |
| Skipped (осмысление) | `artefacts/confluence/skipped.md` | 2 Осмысление |
| Черновики узлов | `artefacts/confluence/nodes-draft.jsonl` | 3 Оцифровка |
| Routing plan | `artefacts/confluence/routing.jsonl` | 4 Роутинг |
| Граф (рёбра) | рёбра внутри `routing.jsonl` (см. `resources/linking_rules.md`) | 5 Линковка |
| Запись в нексусы | `GROUND/NEXUS/<slug>/`, `GROUND/NEXUS/_map.md` | 6 Отгрузка |

Определи первую отсутствующую (или незавершённую) стадию — это и есть
«текущая точка» пайплайна.

### Этап 3: Вывод статуса

Выведи:

```
Confluence Indexator — пространство: <space_key>

Pipeline:
1. Обход        /cindex-crawl        <✅ готово | ⏳ не запущен>
2. Осмысление    /cindex-comprehend   <✅ готово | ⏳ не запущен | — заблокирован>
3. Оцифровка     /cindex-digitize     <✅ готово | ⏳ не запущен | — заблокирован>
4. Роутинг       /cindex-route        <✅ готово | ⏳ не запущен | — заблокирован>
5. Линковка      /cindex-link         <✅ готово | ⏳ не запущен | — заблокирован>
6. Отгрузка      /cindex-deliver      <✅ готово | ⏳ не запущен | — заблокирован>

<если MCP не настроен>
⚠️ Confluence MCP не обнаружен в .mcp.json — /cindex-crawl остановится с
инструкцией подключить сервер (см. SKILL.md, «Предусловия»).
</если>

Рекомендация: запусти <следующая_команда> <space_key>.
```

Стадия помечается «заблокирован», если предыдущая ещё не готова (пайплайн
последовательный, кроме случая, когда PO явно перезапускает конкретную
стадию поверх старых артефактов — тогда просто отметь, что артефакт устарел
относительно входа).

### Этап 4: Без STOP

`/cindex` — read-only диагностика, отдельного STOP не требует: PO сам решает,
запускать ли рекомендованную команду. Если пайплайн ещё не начат — прямо
укажи, что первый шаг всегда `/cindex-crawl <space_key>`.
