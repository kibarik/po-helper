## Использование

```
/radar-review <run_id>
```

- `<run_id>` — слаг прогона с размеченными ❌ (напр. `short-01`). Обязательно.

Вход: `<run_id>/questions.md` с разметкой PO. Выход: `<run_id>/review.md` + `<run_id>/correction-prompt.md` + STOP.

## Важно

**Роль: Diagnostician.** По каждому `❌` отвечаешь на два вопроса: **почему расходится** (какое поле какого узла врёт/пусто/двусмысленно) и **что подправить** (точечный diff узла). Итог — `correction-prompt.md`: готовый текст, который **PO сам отправит в чат**, чтобы LLM доработал понимание структуры и поправил узлы. Узлы ты здесь **не правишь** (принцип 6 SKILL).

**Свежий взгляд.** Разбираешь расхождение от лица реальности PO, а не оправдываешь прошлую LLM-выборку. Вопрос не «почему я так ответил», а «какого факта не хватает графу, чтобы ответить как в реальности».

## Инструкция для LLM

### Этап 1: Загрузка
1. Прочитай `skills/nexus-calibration/resources/correction_prompt.md` (шаблон + анти-паттерны), `scoring_rules.md`, `calibration_loop.md`.
2. Прочитай `<calibration_workspace>/<run_id>/questions.md` с разметкой PO.
3. Прочитай затронутые `GROUND/NEXUS/team/*.md` (узлы из ответов на `❌`-вопросы).

### Этап 2: Проверка входа
- Разметки PO нет / неполная → СТОП: «Прогон не размечен. Сначала размечай ✅/❌ в questions.md».
- Все `✅` (N/N) → СТОП: «🟢 Калибровка пройдена, разбирать нечего» (review не нужен).
- К `❌` нет фразы-правды → перечисли такие # и попроси PO дописать (без правды расхождение не разобрать).

### Этап 3: Разбор каждого ❌ (review.md)
Для каждого расхождения собери блок (формат — `correction_prompt.md` §review.md):
- вопрос, ответ LLM (кто·почему), правда PO;
- **почему расходится:** какое поле какого узла врёт / пусто / двусмысленно / пересекается;
- **что подправить:** адресный diff — узел + поле + было/стало (`+ contact_for "…"`, сузить `influence_zones`, починить `reports_to`);
- слой (org/social/expertise).

Группируй родственные расхождения: один кривой узел может ронять несколько вопросов — правка одна.

**Routable-gap** (колонка `routable? = нет` у Типа C): знание есть в `context_holds`, но не в структурных полях. Стандартный фикс — перенести факт в `contact_for`/`influence_zones`/`expertise_topics`, оставив `context_holds` как RAG-текст (см. `correction_prompt.md` §routable-gap).

### Этап 4: Уточняющие вопросы (если факта нет)
Если правка зависит от факта, которого нет ни в нексусе, ни в правде PO → вынеси в раздел «Уточнить у PO» внутри `review.md`. Не угадывай (принцип 7 SKILL). При непустом разделе correction-Prompt **не выпускается** до ответов PO.

### Этап 5: Сборка correction-prompt.md
Собери готовый к отправке Prompt (шаблон `correction_prompt.md` §correction-prompt.md):
- адресные правки по узлам (узел → поля → было/стало + причина);
- инструкция проставить `sources: ["calibration:<run_id>"]`, поднять `confidence`, обновить `updated`;
- финальный блок: LLM показывает diff и формулирует словами новое понимание структуры (проверяемость);
- минимальность: меняем только то, что вызвало `❌`.

### Этап 6: Отчёт + STOP
```
Разбор <run_id>: расхождений <K>, узлов к правке <M>. Уточнить у PO: <0 | список>.
review.md и correction-prompt.md готовы: <calibration_workspace>/<run_id>/
── СТОП ── PO:
  1) <если есть «Уточнить у PO»> ответь на уточнения — перезапущу разбор;
  2) иначе скопируй блок correction-Prompt и отправь в чат → LLM поправит узлы;
  3) затем /radar-calibrate <short|long> <run_id+1> для нового прогона.
Цикл закроется на N/N (🟢).
```

## Запреты
1. НЕ править `GROUND/NEXUS/team/*` сам — только выдать correction-Prompt; узлы меняет LLM после того, как PO отправит Prompt (принцип 6).
2. НЕ выпускать correction-Prompt вида «исправь, чтобы тест прошёл» — только адресные правки полей по реальности (анти-паттерн в `correction_prompt.md`).
3. НЕ угадывать недостающий факт — «Уточнить у PO» (принцип 7).
4. НЕ переписывать узлы целиком — минимальные точечные правки под конкретные `❌`.
5. Правка без `sources`/`confidence`/`updated` запрещена (узел без источника = workslop).
6. Стиль — деловая проза (`skills/bft-writer/resources/writing_style.md`).
