## Использование

```
/bft-ext-teams <epic_code>
```

Пример: `/bft-ext-teams EPIC-10`

Вход: `bft_documentation/<epic>/artefacts/bft-context-pack.md` (из `/bft-context-gen`) + снимок ландшафта `{landscape.snapshot_doc}` + Нексус `GROUND/NEXUS/landscape/`. Выход: `bft_documentation/<epic>/artefacts/external-teams-actions.md`.

## Важно

**Роль: Dependency Analyst.** Отдельный этап аналитики: спроецировать ландшафт окружающих команд на конкретный эпик и точно ответить — «разрабатывая эту задачу, с PO каких команд я связан, по каким причинам, прямо или косвенно». Не диагноз проблемы (`/bft-problem`), не решение (`/bft-concept`) — только карта связей с внешними командами.

> Стадия 0.5 BFT-пайплайна: `context-gen → **ext-teams** → problem`. Артефакт `external-teams-actions.md` читают `/bft-problem` (blast radius, worldviews) и `/bft-draft` (зависимости, стейкхолдеры).

## Инструкция для LLM

### Этап 1: Загрузка роли и входов
1. Прочитай `skills/bft-writer/SKILL.md` (принципы, эпистемическая честность).
2. Прочитай `resources/bft_standards.md` (секция «Внешние команды — таксономия связи»).
3. Прочитай контекст-пак: `bft_documentation/<epic>/artefacts/bft-context-pack.md` — scope эпика, затрагиваемые системы.
4. Резолвь секцию `landscape` из `.claude/domain-profile.md`.
5. Прочитай снимок ландшафта `{landscape.snapshot_doc}` — `{quarter}` резолвится как `cadence.current_quarter` из профиля (текущий квартал). Нет файла → `[УТОЧНИТЬ: нет landscape-снимка, запусти /okr-landscape]`, работай от Нексуса.
6. Прочитай ext-team узлы `GROUND/NEXUS/landscape/` — стабильные факты о командах (системы, PO, тип связи).

### Этап 2: Пересечение scope эпика с ландшафтом
Сопоставь затрагиваемые эпиком системы/данные/интеграции (из context-pack) с `owned_systems` и `touchpoints` внешних команд. Отбери **релевантные эпику** команды. Нерелевантные — не тащи.

### Этап 3: Классификация причины связи (таксономия)
Для каждой релевантной команды определи **причину** по таксономии (`bft_standards.md`):
1. Общая система/компонент — обе команды меняют один сервис.
2. Зависимость по данным — потребляем/поставляем данные во владении команды.
3. Интеграционный контракт / API на стыке.
4. Последовательная зависимость roadmap — их KR/работа блокирует/включает наш эпик.
5. Общая регуляторика / compliance-поверхность.
6. Общий стейкхолдер / ресурс / люди.
7. Косвенное влияние (2-й порядок) — рябь через посредника.

И **тип влияния**: `direct` (прямой стык) / `indirect` (через посредника).

Разделяй **факт** (из context-pack / Нексуса / снимка — с якорем) и **вывод LLM** (`[вывод LLM — проверить с PO]`). Неизвестное → `[УТОЧНИТЬ у {кого}]`.

### Этап 4: Сборка `external-teams-actions.md`
Формат (`bft_standards.md`):

```markdown
# Внешние команды — карта связей: <epic_code>

> Источник ландшафта: {landscape.snapshot_doc} + GROUND/NEXUS/landscape/. Дата: <ISO>.

## Связи с внешними командами
| Команда | PO | Причина связи (кат. 1–7 + конкретика) | Тип | Чем заняты в квартале (источник) | Блокер/синергия | Рекомендованное действие | Якорь |
|---|---|---|---|---|---|---|---|
[по строке на релевантную команду; Тип = direct/indirect]

## Открытые вопросы [УТОЧНИТЬ]
[Неизвестные связи, неподтверждённые причины, отсутствующий снимок]
```

- **Рекомендованное действие** — конкретное: «согласовать контракт с PO {имя} до {срок}», «уточнить их roadmap на квартал», не «взаимодействовать».
- **Якорь** — источник факта связи (context-pack / узел Нексуса / строка снимка / JIRA).

### Этап 5: Сохранение + STOP
- Сохрани `bft_documentation/<epic>/artefacts/external-teams-actions.md`.
- **Выведи и ОСТАНОВИСЬ:**
  ```
  Карта внешних команд готова: bft_documentation/<epic>/artefacts/external-teams-actions.md
  Релевантных команд: N (direct: X / indirect: Y). Блокеры: [список].
  [УТОЧНИТЬ]: k пунктов.
  ── СТОП ── PO/СА: проверьте связи и причины. Дальше — /bft-problem <epic_code>.
  ```

## Запреты
1. НЕ ставь диагноз проблемы и НЕ предлагай решения — только карта связей с внешними командами.
2. Не тащи нерелевантные эпику команды — фильтр по пересечению scope.
3. Причина связи без якоря → `[УТОЧНИТЬ]` или `[вывод LLM]`, не выдумывать чужой OKR.
4. Нет снимка ландшафта → честно помечай, не сочиняй ландшафт с нуля.
5. Стиль — по `skills/bft-writer/resources/writing_style.md`: деловая проза без AI-стоп-слов; эмодзи только семантические.
