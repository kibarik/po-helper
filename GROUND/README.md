# GROUND Vault

**GROUND Vault** — ваш персонализированный продуктовый контекст.

Это ваша рабочая область, куда Кортекс цифровизует контекст продукта в **насыщенный Нексус** (узлы знаний) и где ведётся продуктовый процесс по PAF.

## Поток онбординга

1. `/paf-init` — конфигурация (`config.yaml`) + этот GROUND-скелет + дефолтный минимальный каталог Нексусов.
2. *(опционально)* `/paf-nexus-create` — кастомные Нексусы под специфику вашего продукта.
3. `/paf-onboard` — цифровизация контекста (документы + интервью) в узлы Нексусов.

## Структура

```
GROUND/
├── README.md            ← этот файл
├── config.yaml          ← (создаётся /paf-init) конфиг клиента
├── _intake/             ← документы для онбординга (ингестия в Phase A)
├── NEXUS/               ← узлы знаний, насыщаемые Кортексом
│   ├── _index.md        ← MOC всех Нексусов клиента
│   ├── _registry.yaml   ← реестр Нексусов (дефолт + кастом)
│   ├── market/          ← дефолтные Нексусы
│   ├── customer/
│   ├── product/
│   └── growth/
├── PULSE/               ← Progress Pulse (динамика продукта)
├── BUNCH/               ← Банчи (связки инициатив/экспериментов)
└── RESULTS/             ← Harvesting (урожай — результаты и инсайты)
```

## Правила

- **Методология PAF — read-only.** Справочник PAF (`docs/`/, мастер-каталог Нексусов) правке не подлежит.
- **Работайте только в `GROUND/`.** Это ваш контекст; всё остальное — дистрибутив.
- `GROUND/` отслеживается в git (контекст клиента). `.claude/memory.db`, `.swarm/` — gitignored.

## ATLAS (навигация) и задачи

- **Навигация:** `python3 sa_documentation/atlas_index.py --atlas` генерирует
  `ATLAS/manifest.json` (LLM) + ген-блок `ATLAS/INDEX.md` (человек). Запускать
  после изменения узлов. Не требует ruflo/`.swarm`.
- **RAG-прогрев (опц.):** на свежем clone/worktree `.swarm/` пуст (gitignored).
  Семантический поиск агентов заработает только после
  `python3 sa_documentation/nexus_index.py` в рабочей CWD. Без прогрева навигация
  работает на `ATLAS/manifest.json`.
- **Задачи/Банчи:** Backlog.md (`backlog init`, затем `backlog board`/MCP).
  Связь задача→узел: поле `nexus_nodes: [node_id]` во frontmatter задачи.
